# T076 — Make daily progress and caps per deck

## 2026-07-20

**Summary:** `DailyStats` was keyed by `day` only, so studying ANY deck's daily new-card/review
cap suppressed every other deck's allowance — a deck with a full unlearned backlog could show "All
caught up" the moment another deck (or itself) hit the shared global cap. Made `DailyStats` keyed
per `(day, deckKey)`, threaded the studied deck through the write path, and updated every Home read
path to resolve per-deck stats — while keeping the top "Written today" stat and the day streak
whole-app as required.

**What I did:**
- `Sources/Models/DailyStats.swift`: added `var deckKey: String?` (the studied `Deck.sourceDeckName`
  — the stable identifier the importer already uses to find/reuse a deck across re-imports; `nil`
  means a legacy pre-migration row or the whole-app aggregate).
- `Sources/App/ModelSchema.swift` (new): `KakitoriSchemaV1` (nests the OLD shape of `DailyStats`,
  no `deckKey`, alongside the unchanged `Deck`/`Section`/`Note`/`CardSchedule`), `KakitoriSchemaV2`
  (the current models), and `KakitoriMigrationPlan` (`MigrationStage.lightweight` V1→V2) — since
  `KakitoriApp.makeModelContainer` previously built from a bare `Schema([...])` with no versioning,
  adding `deckKey` needed an explicit migration path so it doesn't crash opening an existing
  on-device store.
- `Sources/App/KakitoriApp.swift`: `makeModelContainer` now builds `Schema(versionedSchema:
  KakitoriSchemaV2.self)` and passes `migrationPlan: KakitoriMigrationPlan.self` to `ModelContainer`.
- `Sources/Models/StatsRecorder.swift`: `recordGrade`/`recordStudySeconds` gained a `deckKey: String?
  = nil` parameter; `fetchOrCreateDailyStats` now fetch-or-creates by `(day, deckKey)`.
- `Sources/Features/Session/SessionViewModel.swift`: stores `deckKey = deck.sourceDeckName`; passes
  it to both `StatsRecorder` calls and to the `fetchDailyStats` helper used to build the session's
  `SessionQueue` (so the queue's own caps are already per-deck, not just the Home display).
- `Sources/Features/Home/HomeView.swift`: `todayStats` became `todayStats(for deck:)`, matching
  `(day, deckKey: deck.sourceDeckName)`; `dueCount(for:)` and both `deckList` branches (compact +
  grid) now pass each deck's own stats into `DailyAllowance.forDeck` / `DeckCardView`.
- `Sources/Features/Home/TodayBannerView.swift`: `calculateAllowance` now computes each deck's
  `DailyAllowance.forDeck` from ITS OWN `(day, deckKey)` row, then sums via `DailyAllowance.aggregate`
  — previously it applied one global stats row's counts uniformly to every deck via `forDecks`.
- `Sources/Features/Home/StatsRowView.swift`: "Written today" / "Minutes studied" now SUM across all
  of today's rows (`allStats.filter { $0.day == today }`) instead of taking `.first` — with one row
  per deck per day, `.first` would silently drop every deck but one. The streak's `activeDays` set
  (`Set(allStats.map(\.day))`) already unions across rows correctly and needed no change — a day is
  "active" if ANY deck has a row for it.

**Tests added/extended:**
- `Tests/UnitTests/StatsRecorderTests.swift`: `testRecordGradeIsolatesDeckKeys`,
  `testRecordGradeAndStudySecondsForDifferentDecksStayIsolated` (two decks studied same day get two
  separate rows, each accumulating only its own grades/seconds),
  `testDailyAllowanceForOneDeckIsUnaffectedByAnotherDecksRecordedProgress` (deck A driven to its
  new-card cap via `StatsRecorder`; deck B's `DailyAllowance.forDeck` — read from ITS OWN row —
  stays uncapped, `isAllCaughtUp == false`). Extended the `fetchDailyStats` test helper to take an
  optional `deckKey`.
- `Tests/UnitTests/DailyStatsTests.swift`: `testPreDeckKeyRowSurvivesAdditiveSchemaChange` — writes a
  row via a real file-backed store on `KakitoriSchemaV1` (no `deckKey`), then reopens the SAME file
  with `KakitoriSchemaV2` + `KakitoriMigrationPlan`; asserts no crash and the row survives with all
  original fields intact and `deckKey == nil`.
- `Tests/UnitTests/DailyAllowanceTests.swift`: updated `testBannerAllowanceIsZeroWhenDailyCapsAreHit`
  to tag its `DailyStats` fixture with the deck's `sourceDeckName` (previously untagged/global,
  which would no longer match under the new per-deck lookup).
