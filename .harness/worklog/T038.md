# T038 — Trace session end-to-end: Home → setup → session → summary

**Date:** 2026-07-18

## Summary

Wired the full Trace loop together in `HomeView`: deck card → `DeckSetupSheet` → full-screen
`SessionView` → `SummaryView` → back to Home, with mid-session close preserving grades and
"Study another deck" correctly re-presenting the setup sheet.

## What was done

1. **`Sources/Features/Home/HomeView.swift`:**
   - `DeckCardView`'s `onStudy` now sets `setupDeck`, which drives a `.sheet` presenting
     `DeckSetupSheet` fed with the deck's jpTitle/name and a live due count (reusing
     `TodayBannerView.calculateDueCards`). `presentationDetents` is `.medium` on compact
     width (iPhone) and `.large` on regular width (iPad formSheet-style).
   - `DeckSetupSheet`'s `onStart` builds a `SessionViewModel` for (deck, mode) on the app's
     `ModelContext` with `AppClock.system` and a random session seed, dismisses the sheet, and
     presents it via `.fullScreenCover`.
   - `SessionView`'s `onClose` calls `viewModel.close()` (records study seconds; each grade
     was already persisted at grade time) then dismisses the cover.
   - When the view model reaches `.finished`, the cover swaps to `SummaryView` fed the
     summary numbers and the live streak (`DailyStats.currentStreak`). "Back to decks"
     dismisses to Home; "Study another deck" dismisses the cover and re-presents the setup
     sheet for the same deck.
2. **Bug found + fixed during visual verification:** setting `setupDeck` in the same action
   that dismisses the `fullScreenCover` ("Study another deck") silently dropped the sheet
   presentation — SwiftUI won't reliably start a new presentation transition while another is
   still dismissing. Fixed by staging the deck in `deckPendingRestudy` and presenting the
   setup sheet from the cover's `onDismiss` callback instead, so it opens once the dismiss
   animation actually completes.
3. **`Tests/UnitTests/SessionFlowTests.swift` (new):**
   - `testFullSessionGraduatesAllCardsAndRecordsStats`: seeds 1 deck + 4 `new` notes, builds
     the view model, loops `showAnswer()`/`grade(.good)` until `.finished`. Asserts every
     `CardSchedule` left `.new` with a non-nil `dueAt`, readable from a **fresh** `ModelContext`
     on the same `ModelContainer` (required an explicit `modelContext.save()` — SwiftData
     doesn't share uncommitted context state across contexts), and the day's `DailyStats.cardsWritten`
     equals the number of grades given.
   - `testMidSessionCloseWithoutExtraTeardownPreservesGrades`: seeds 4 `new` notes, grades
     exactly one, discards the view model with **no** `close()` call, and asserts (via a fresh
     context) that the graded note's schedule advanced while the other three are still
     untouched `.new`.
