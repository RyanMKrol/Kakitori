# T058 — SM2Scheduler: learning + relearning transitions (re-attempt of T023)

## 2026-07-18

Added `Sources/Scheduler/SM2Scheduler.swift`: `struct SM2Scheduler` (pure Foundation, no
`Scheduler` conformance yet — that's T024) with `init()` removed (SwiftLint flagged the
explicit memberwise-equivalent init as unneeded_synthesized_initializer) and
`func apply(_:to:now:rng:) -> ScheduleSnapshot`. `.new` is treated as `.learning` at step 0;
`.learning` walks `SRSConstants.learningStepsSeconds` ([60, 600]s) with Again resetting to
step 0, Hard repeating the current step, Good advancing (graduating to `.review`
interval 1 day off the last step), and Easy graduating immediately at 4 days.
`.relearning` (10-min step) has Again/Hard repeat the step and Good/Easy re-graduate to
`.review` at `max(1, round(intervalDays × 0.5))`, with Easy floored at the 4-day easy
graduating interval so it's never worse than Good. `.review` is a pass-through placeholder
per spec (T024 fills it in). No fuzz, no force-unwraps, bounds-safe step indexing via
`min(max(...))`. Split into `applyLearning` / `applyLearningGood` / `applyRelearning` /
`graduate` / `relapsedIntervalDays` helpers to keep each under SwiftLint's 50-line
function-body-length limit.
