# T028 — StatsRecorder: grades + study time into DailyStats

## 2026-07-17 — Implementation & Tests

### What was built

Created `Sources/Models/StatsRecorder.swift` — a small persistence helper that records grades and study time into DailyStats.

**Key implementation details:**
- `recordGrade(previousState:now:in:)` — increments `cardsWritten`, and conditionally increments `newIntroduced` (only when `previousState == .new`) and `reviewsDone` (only when `previousState == .review`).
- `recordStudySeconds(_:now:in:)` — accumulates seconds into `secondsStudied`.
- Both methods fetch or create the DailyStats row for the adjusted 4 AM day key, using `AppClock.system.adjustedDay(for:)`.
- No force-unwraps, no `Date()` calls (time is injected), proper error handling with `throws`.

Created `Tests/UnitTests/StatsRecorderTests.swift` — comprehensive test suite with 11 tests covering:
- Basic grade recording with .new state (increments both cardsWritten and newIntroduced).
- 3:59 AM boundary (lands on previous day's adjusted-day key).
- 4:00 AM boundary (lands on new day).
- All four card states (.new, .learning, .review, .relearning) and their effects on counters.
- Study seconds accumulation (30 + 45 = 75).
- Multiple grades and seconds on the same day all accumulating correctly.
- No duplicate rows across different adjusted days.
- Zero seconds recorded correctly.

All tests use an in-memory SwiftData ModelContainer with fixed local calendar dates. Test setup wrapped in try-catch to avoid force-tries.
