# T042 тАФ Listen & Write mode (шБЮуБНхПЦуВК)

**Date:** 2026-07-18

## What was done

Implemented the Listen & Write session mode with autoplay, prompt UI, empty guide boxes, and enabled play button in answer block for all modes.

### Changes made:

1. **SessionViewModel** (`Sources/Features/Session/SessionViewModel.swift`):
   - Added `audio: any AudioPlaying` injection parameter (defaults to `AudioService()`)
   - Added `autoplayEnabled: Bool` property reading from UserDefaults (default: true)
   - Added `replayAudio()` function calling `audio.play(target:audioFilename:deckID:)`
   - Added `autoplayOnCardEntry()` which triggers replay when mode is `.listen`, autoplay is enabled, and card changes
   - Autoplay is reset to `false` on each card entry and triggered once on card load in `.listen` mode
   - Added `hasAutoplayed` tracking to prevent replaying on every property change

2. **PromptPaneView** (`Sources/Features/Session/PromptPaneView.swift`):
   - Refactored `traceModePrompt` to render only Trace mode
   - Added `listenModePrompt` with accent "LISTEN & WRITE" caption, large play button (play.circle.fill 64pt in accent color), and instruction text
   - Used `@ViewBuilder` on `promptView` to handle conditional branching
   - Enabled the answer block's play button by wrapping it in a Button that calls `viewModel.replayAudio()` with proper closure naming for SwiftLint compliance
   - Accessibility identifier: `play-audio` on prompt button, `play-audio-answer` on answer button

3. **CanvasPaneView** (`Sources/Features/Session/CanvasPaneView.swift`):
   - Added hint text for Listen mode: "Write the word you heard"
   - Existing `TraceGuideLayer` visibility logic (only shows when `viewModel.mode == .trace`) already handles empty guide boxes in Listen mode
