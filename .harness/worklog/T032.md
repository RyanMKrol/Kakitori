3. Added `Tests/UnitTests/CanvasPaneRenderTests.swift`:
   - In-memory SwiftData container, fixed clock, one review note ("ありがとう").
   - Renders `CanvasPaneView` at iPad size (1194×834) and writes `screenshots/T032-canvas.png`.
   - **Deviation from the literal `ImageRenderer` recipe used by `SessionScaffoldRenderTests`:**
     `ImageRenderer` cannot flatten `WritingCanvas`'s `PKCanvasView`
     (`UIViewRepresentable`) on this SDK (iOS 26.5) — it logs "Unable to render flattened
     version of PlatformViewRepresentableAdaptor<WritingCanvas>" and substitutes an opaque
     "unsupported" glyph that fills the canvas frame, obscuring the guide boxes underneath
     (confirmed with a throwaway scratch test, since removed). The codebase already has a
     proven working alternative for this exact `TraceGuideLayer` + `WritingCanvas` combination
     in `WritingCanvasPreviewSnapshotTests.swift` — a `UIHostingController` rendered via
     `drawHierarchy(in:afterScreenUpdates:)` composites it correctly (verified: produces a
     screenshot with visible boxes/glyphs, no red icon). `CanvasPaneRenderTests` uses that
     technique instead, while keeping everything else from the requested recipe (in-memory
     container, fixed clock, `#filePath`-derived screenshot path, 1194×834 iPad size).
