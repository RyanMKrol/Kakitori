# T050 — Dark mode palette

## 2026-07-18

Made `KakitoriTheme` adaptive to `userInterfaceStyle` per docs/06-ui-design.md §1, and purged
hardcoded light-only colors from every Feature view.

### What changed

- `Sources/Support/Theme.swift`:
  - `paper` and `ink` are now built from `Color(uiColor: UIColor { trait in ... })`, switching
    on `trait.userInterfaceStyle`. `paper`: light `#FAF8F4` → dark `#17150F` (warm near-black).
    `ink`: light `#1A1A1A` → dark warm off-white `#F0EDE6` (invented per the task — kept warm,
    not pure white).
  - `accent` stays `#C43B2E` in both modes, unchanged.
  - `inkFaint` (12%) and `boxLine` (8%) stay derived as opacities of the adaptive `ink` token,
    so they invert automatically.
  - Added `inkUIColor` (a `UIColor`, not `Color`) so non-SwiftUI call sites (PencilKit) can use
    the same adaptive ink value.
  - Added a new `surface` token: an elevated card-surface color, distinct from `paper` for
    light-mode elevation contrast (`.white` in light, `#211E17` in dark) — this backs every
    `Color.white` card background found in the Feature views (no design-doc token existed for
    this, since the docs only specify `paper`/`ink`/`accent`/`inkFaint`/`boxLine`/chips; kept it
    a light adaptive surface rather than collapsing cards into `paper`, otherwise the cards would
    become invisible against the page background).
  - `chipDue` fill/text already derived from `ink`/`paper`, so it inverts correctly; verified
    `chipDueForeground = paper` still gives label-on-ink-fill contrast in dark (paper token is
    now the near-black-inverted-to-off-white text, ink token is now the fill, so the label reads
    dark-on-light — checked visually, still legible, matches the "paper stays the readable body
    color against an ink fill" intent in both modes).
- Audited every view under `Sources/Features/` and `Sources/Support/` for hardcoded light values:
  - `Sources/Features/Home/DeckCardView.swift` (compact + regular card backgrounds),
    `Sources/Features/Home/StatsRowView.swift` (`StatCard` background),
    `Sources/Features/Summary/SummaryView.swift` (`GradeCard` background + "Study another deck"
    button background): `Color.white` → `KakitoriTheme.surface`.
  - `Sources/Features/Session/Canvas/GuideBox.swift`: trace-glyph tint was
    `KakitoriTheme.ink.opacity(0.12)` (a magic-number duplicate of `inkFaint`) → replaced with
    `KakitoriTheme.inkFaint` directly.
  - Left `Sources/Features/Home/HomeView.swift`'s `Color.black.opacity(0.3)` import-progress
    scrim as-is: it's a translucent dimming overlay over the whole screen, not a themed surface,
    and reads correctly in both light and dark (a black scrim over dimmed content is standard
    regardless of appearance) — not a "light-only" color.
- `Sources/Features/Session/Canvas/WritingCanvas.swift`: the `PKInkingTool` stroke color was
  hardcoded `.black` — switched to `KakitoriTheme.inkUIColor` so pencil/finger strokes stay
  visible against dark paper (canvas background is already `paper`/`clear`-composited, guide-box
  borders/crosses were already `boxLine`, trace guides were already `inkFaint`/now confirmed via
  the `GuideBox.swift` fix above — this was the one gap, item 3 in the task spec).
