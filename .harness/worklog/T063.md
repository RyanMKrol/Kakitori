# T063: Deck card — never render the deck name twice / a Latin string in the JP serif slot

**Completed 2026-07-20**

## What was done
Fixed DeckCardView to conditionally render the Japanese-display Text line only when `deck.jpTitle` is a real, non-empty value distinct from `deck.name`. Previously both layouts (compact ~L55/56 and regular ~L149/150) used `Text(deck.jpTitle ?? deck.name)`, causing duplicates when jpTitle was nil/empty or identical to name.

### Changes
- **compactRow** (lines 55–60): Wrapped `Text(jpTitle)` in conditional `if let jpTitle = deck.jpTitle, !jpTitle.isEmpty, jpTitle != deck.name`
- **regularCard** (lines 151–156): Applied same conditional to `Text(jpTitle)` at size 32
- Both layouts now omit the JP-display line when jpTitle is nil, empty, or equal to deck.name
- Preserved all existing fonts, spacing, lineLimit(1), accessibilityIdentifier, and other elements
