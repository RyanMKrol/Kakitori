# T045 Worklog

## 2026-07-18

### Build Summary

**Objective:** Implement mode availability rules per deck and card to show/hide practice modes in the deck setup sheet and filter cards for each session mode.

### What was done

1. **Created `Sources/Scheduler/ModeAvailability.swift`** — Pure enum with two static functions:
   - `deckModes(scripts:)` — Returns available modes for a deck based on its script composition. Kana-only decks get `[.trace, .listen, .recall, .mixed]`; kanji/mixed decks get `[.trace, .listen, .translate, .recall, .mixed]`.
   - `cardQualifies(_:hasAudio:ttsAvailable:english:)` — Pure predicate per mode:
     - `.listen` → requires audio OR TTS
     - `.translate` → requires non-empty English (after trimming)
     - `.trace`, `.recall`, `.mixed` → always true

2. **Updated `Sources/Features/DeckSetup/DeckSetupSheet.swift`**:
   - Added `availableModes: [PracticeMode]` parameter to constructor
   - Changed `modeList` to iterate over `availableModes` instead of `PracticeMode.allCases`
   - Updated preview and call sites

3. **Updated `Sources/Features/Home/HomeView.swift`**:
   - Computed available modes from deck's non-deleted notes' scripts
   - Passed `availableModes` to `DeckSetupSheet`

4. **Updated `Sources/Features/Session/SessionViewModel.swift`**:
   - Added filtering during session queue construction: cards now skip if they don't qualify for the chosen mode
   - Uses `ModeAvailability.cardQualifies()` with `hasAudio: note.audioFilename != nil`, `ttsAvailable: audio.isAvailable`, `english: note.english`

5. **Created `Tests/UnitTests/ModeAvailabilityTests.swift`** with 17 test cases:
   - Deck modes correctness for kana and kanji scripts
   - Card qualification for all modes with various audio/TTS/English combinations
   - Edge cases: nil, empty, whitespace-only, newline-only English fields
