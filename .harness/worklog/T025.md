# T025 — SM2Scheduler: grade previews match applied results

## Outcome
done

## Work performed (2026-07-18)

Implemented `preview(for:now:)` on SM2Scheduler to provide exact grade-button labels:

1. **Added Scheduler protocol conformance** — extended SM2Scheduler to implement the Scheduler protocol's preview method.

2. **Preview computation** — for each grade (again/hard/good/easy), the preview:
   - Calls `apply()` with fuzz disabled to ensure stable, deterministic results
   - Uses a seeded `SplitMix64(seed: 0)` RNG (unused when fuzz is off)
   - Extracts the resulting `dueAt` and `intervalDays` from the scheduled snapshot

3. **Label formatting** — formats labels by state:
   - **Learning/relearning** (sub-day): computes minutes from now, returns `"<1m"` (≤1 min) or `"\(minutes)m"`
   - **Review** (day-based): returns `"\(Int(intervalDays))d"` (e.g., `"10d"`, `"4d"`)

4. **Test coverage** — `SM2PreviewTests.swift` (12 tests) validates:
   - Preview contains exactly 4 entries (one per Grade)
   - Preview `dueAt` and `intervalDays` match fuzz-free `apply()` output (matrix: new, learning step 0/1, review, relearning × all 4 grades)
   - Pinned labels: new card / learning step 0 → `"<1m"`, `"<1m"`, `"10m"`, `"4d"`
   - Learning step 1 good → `"1d"`; review I 10 → `"10m"`, `"12d"`, `"25d"`, `"34d"`; relearning good → `"5d"`
