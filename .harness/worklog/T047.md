# T047 — Settings screen: 4 settings + persistence + privacy manifest

**Date:** 2026-07-18

## Summary

Implemented the complete Settings screen for Kakitori v1 with four user-configurable settings (new cards per day, max reviews per day, audio autoplay toggle, show romaji toggle), persistent storage via AppSettings + @AppStorage, and privacy manifest for UserDefaults API access.

## What was built

### 1. AppSettings struct (`Sources/Support/AppSettings.swift`)
- Single non-view read point for four settings keys
- Computed properties with getters/setters for round-trip storage via UserDefaults
- Default values: newCardsPerDay=10, maxReviewsPerDay=100, audioAutoplay=true, showRomaji=true
- Uses `object(forKey:) as? Type ?? default` pattern to distinguish unset (nil) from zero/false

### 2. SettingsView (`Sources/Features/Settings/SettingsView.swift`)
- SwiftUI View with @AppStorage binding for all four settings
- Two Stepper controls for numeric settings (ranges 1...50 and 10...500)
- Two Toggle controls for boolean settings
- Accessibility identifiers on all controls as specified
- Paper background matching KakitoriTheme
- Wrapped in NavigationStack-compatible layout

### 3. Home settings button (`Sources/Features/Home/HomeView.swift`)
- Added NavigationStack with navigationPath state
- Gear icon button (SF Symbol `gearshape`) in top-right header
- Accessibility identifier "settings-button"
- Routes to SettingsView via navigationDestination

### 4. Privacy manifest (`Sources/Support/PrivacyInfo.xcprivacy`)
- XML plist with NSPrivacyTracking=false
- Empty NSPrivacyTrackingDomains and NSPrivacyCollectedDataTypes
- NSPrivacyAccessedAPITypes array with UserDefaults entry using reason CA92.1

### 5. Project configuration (`project.yml`)
- Added resources section to bundle PrivacyInfo.xcprivacy with buildPhase: resources

## Notable decisions

- **AppSettings struct over direct @AppStorage in views**: Centralizes defaults logic and enables testing without UserDefaults.standard pollution. SessionQueue.build() call sites (T029+) will use `AppSettings().newCardsPerDay / .maxReviewsPerDay` to read user overrides.
- **VStack layout instead of Form**: SwiftUI's Form takes a non-trailing-closure parameter, and Section name conflicts with the SwiftData @Model in the same target. VStack with padding provides the visual grouping without conflicts.
- **Privacy manifest CA92.1 (User Defaults API)**: Covers NSUserDefaults read/write without needing NSUbiquitousUserDefaults (iCloud sync not in v1 scope).
