# T067 — Fix Show-answer reveal jank (background flash + layout jump)

## 2026-07-20

### What I did
Two scoped changes to the session reveal transition:

1. **`PromptPaneView.swift`** — replaced the two independently-fading `.transition(.opacity)`
   siblings over `KakitoriTheme.paper` (which let the paper bleed through mid-crossfade — the
   "flash") with a stable-layer approach: `promptView` stays **fully rendered and opaque**
   underneath the whole time (each layer given its own `.background(KakitoriTheme.paper)`), and
   only the `answerBlock`'s opacity animates on top. Because there is always a fully-opaque
   foreground over the paper, the background never shows through the reveal. It is a pure opacity
   change (no scale/slide), so Reduce Motion is honored. Added `.accessibilityHidden` so the
   hidden layer isn't read by VoiceOver. Prompt/answer copy and styling unchanged.

2. **`ActionRowView.swift`** — wrapped the show-answer button / grade-row swap in a `ZStack` with
   an always-laid-out but `.hidden()` grade row as a **sizer**. The grade row is the taller of the
   two (it stacks four buttons vertically at accessibility Dynamic Type sizes), so the action area
   now reserves its footprint in BOTH phases. The visible button ↔ grade row then swaps in place,
   so the reveal never changes the action-area height and never reflows the canvas above it (fixes
   the "jump" in both `compactLayout` and `regularLayout`). Kept the existing
   `emphasisTransition(reduceMotion:)` on the visible content (degrades to `.opacity` under Reduce
   Motion).

No changes to the scheduler, viewModel phase logic, or the card-to-card transition. Theme.swift
untouched.
