# T052 — Reduce Motion support

## 2026-07-18

### What was done

Implemented centralized Reduce Motion support per docs/06-ui-design.md §4 ("Reduce Motion: disable slide, keep crossfades").

**Implementation:**
1. Added two motion helper functions in `Sources/Support/Theme.swift`:
   - `KakitoriTheme.cardTransition(reduceMotion: Bool) -> AnyTransition` — returns asymmetric insertion/removal with slide when motion enabled, plain opacity when reduce motion is on
   - `KakitoriTheme.emphasisTransition(reduceMotion: Bool) -> AnyTransition` — returns scale + opacity normally, plain opacity under reduce motion

2. Updated `Sources/Features/Session/ActionRowView.swift`:
   - Added `@Environment(\.accessibilityReduceMotion)` binding
   - Grade row and Show Answer button now use `emphasisTransition` instead of inline scale+opacity transition

3. Updated `Sources/Features/Summary/SummaryView.swift`:
   - Added `@Environment(\.accessibilityReduceMotion)` binding
   - Added `discOpacity` state to support opacity-based fade-in under reduce motion
   - Disc scale-in animation now branches: spring scale-in when motion enabled, easeInOut opacity fade-in when reduce motion is on

4. Added `Tests/UnitTests/ThemeMotionTests.swift`:
   - Tests verify both motion helpers return valid transitions in all conditions

**Motion inventory per docs/06-ui-design.md §3:**
- ✅ Answer reveal: prompt→answer crossfade kept as-is (opacity, no reduce-motion change needed)
- ✅ Grade row: spring-up becomes crossfade under reduce motion (via `emphasisTransition` in ActionRowView)
- ✅ Summary disc: scale-in becomes opacity fade-in under reduce motion (in SummaryView)
- ℹ️ Card-to-card and canvas clear: motion helpers are available for future session card transitions

### Design notes

- PromptPaneView opacity transitions (prompt↔answer reveal) remain unchanged — they are content crossfades, not motion per the spec, so reduce motion does not affect them
- Crossfades are preserved in all motion settings per spec requirement "Reduce Motion: disable slide, keep crossfades"
- Motion helpers are pure (no environment reads inside); views read `accessibilityReduceMotion` and pass the bool explicitly
