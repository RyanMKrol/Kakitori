# T019 — Worklog

## 2026-07-17 — Built and tested

**What:** Implemented `MediaStore.swift` to handle copying `.apkg` media payloads into the app's per-deck media directory, renamed per the media JSON manifest.

**Implementation:**
- `struct MediaStore: Sendable` with injectable `baseURL` and static `defaultBaseURL` pointing to `Application Support/Kakitori`
- `mediaDirectory(for: UUID) -> URL` returns `baseURL/Media/<deckID.uuidString>/`
- `enum MediaStoreError: Error, Equatable` with `.badManifest` case
- `copyMedia(manifestURL:payloadDirectory:deckID:) throws -> [String]`:
  1. Parses JSON manifest (catches JSON parsing errors and converts to `.badManifest`)
  2. Creates media directory with intermediate directories
  3. Iterates manifest entries: skips missing source files, removes existing destination, copies file by name (overwrite by name)
  4. Returns list of real filenames actually copied

**Test coverage:** `Tests/UnitTests/MediaStoreTests.swift` with 4 tests:
- `testCopyMediaFromFixture`: End-to-end with actual fixture, verifies MP3 files present, size > 0, idempotent re-copy (overwrite, no duplication)
- `testBadManifestThrows`: Non-JSON file throws `.badManifest`
- `testMissingPayloadFileIsSkipped`: Manifest with missing payload files succeeds, returns empty list
- `testDefaultBaseURL`: Verifies `defaultBaseURL` points to Application Support/Kakitori
