# T037 — Deck setup sheet: practice mode picker

**Date:** 2026-07-18

## Summary

Built the DeckSetupSheet component with all five practice modes, mode picker logic, and visual verification via render test.

## What was done

1. **Created `Sources/Features/DeckSetup/DeckSetupSheet.swift`:**
   - Defined `PracticeMode` enum with all five cases (trace, listen, translate, recall, mixed)
   - Each case carries glyph, label, description, and accessibility identifier
   - Implemented DeckSetupSheet view with decoupled init (no live Deck dependency)
   - Header band: ink background with paper text, jpTitle in japaneseDisplayFont ~28pt, secondary line with enTitle and dueCount, close button (✕)
   - Small-caps "PRACTICE MODE" caption
   - Mode rows: five total, each with ~44pt rounded glyph tile, bold label, faint one-line description, selection ring (hairline circle with accent-filled dot when selected)
   - Default selection: Trace mode
   - Full-width "Start writing" button at bottom in accent red, calls onStart(selectedMode)

2. **Created `Tests/UnitTests/DeckSetupRenderTests.swift`:**
   - Render test using ImageRenderer at scale 2, frame 560×700
   - Constructs DeckSetupSheet with test data (ひらがな, Hiragana, 10 due)
   - Writes rendered output to `screenshots/T037-decksetup.png` (path derived from #filePath)

3. **Visual verification:**
   - Screenshot shows correct dark header with Japanese title, English subtitle, and close button
   - All five mode rows render with correct glyphs (書/聞/訳/思/混) and descriptions
   - Trace mode displays selected (red accent dot in ring)
   - Other modes show empty selection rings
   - Red "Start writing" button pinned at bottom
