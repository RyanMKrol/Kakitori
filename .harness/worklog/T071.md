# T071 — Fix low-contrast ink on the writing canvas

## 2026-07-20

**Summary:** The user's drawn ink rendered mid-grey against the paper in both light and dark mode.
Root cause: `WritingCanvas.makeUIView` set `PKInkingTool`'s color from the dynamic
`KakitoriTheme.inkUIColor` once, and `updateUIView` was empty — PencilKit resolved the dynamic color
against a base trait collection at tool-creation time and never refreshed it.

**What I did:**
- Added `KakitoriTheme.resolvedInkColor(for:)` to `Sources/Support/Theme.swift` — resolves
  `inkUIColor` to a concrete `UIColor` against a given `UITraitCollection`, without touching
  `inkUIColor`'s dynamic definition, `inkLight`/`inkDark`, `inkFaint`, or any guide/box colors.
- In `Sources/Features/Session/Canvas/WritingCanvas.swift`:
  - `makeUIView` now builds the `PKInkingTool` from a color resolved against the canvas view's
    actual `traitCollection` (via a new `inkingTool(for:)` helper), instead of the raw dynamic color.
  - Registered a trait-change handler via `canvasView.registerForTraitChanges([UITraitUserInterfaceStyle.self])`
    that rebuilds the tool with a freshly-resolved color whenever the appearance changes.
  - Populated `updateUIView(_:context:)` (previously empty) to also re-resolve and re-apply the tool
    color, so SwiftUI-driven appearance updates keep the ink in sync too.
- Did not touch `GuideBox.swift`, `TraceGuideLayer`, or any `inkFaint` usage — the faded trace guides
  are unchanged by design.
