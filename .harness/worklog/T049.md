# T049 — iPhone compact Home, deck setup, and summary layouts

## 2026-07-18

Built the compact (iPhone) variants for Home, deck setup, and session summary, switching
purely on `@Environment(\.horizontalSizeClass) == .compact` (never device checks).

### What changed

- `Sources/Features/Home/DeckCardView.swift`: added a `compactRow` layout used when
  `horizontalSizeClass == .compact` (regular iPad grid card untouched as `regularCard`).
  The compact row leads with a 52pt representative-glyph tile (`representativeGlyph`,
  computed from the deck's aggregate `Script` set: pure hiragana → あ, pure katakana → ア,
  everything else including kanji/mixed → 語), keeps the `deck-row-<name>` identifier, and
  the whole row is one `Button` (fully tappable). Chip row / "All caught up" reused as-is.
- `Sources/Features/Home/HomeView.swift`: `deckList` now branches — compact renders a
  single-column `VStack` of `DeckCardView`s, regular keeps the existing 2-up `LazyVGrid`.
  (The deck-setup sheet's `.presentationDetents(horizontalSizeClass == .compact ? [.medium]
  : [.large])` was already wired from an earlier task — verified it still behaves
  correctly, no change needed here.)
- `Sources/Features/Home/StatsRowView.swift`: captions compress to "Streak" / "Written" /
  "Studied" in compact (regular keeps "Day streak" / "Written today" / "Minutes studied");
  `StatCard` shrinks value/prefix font sizes and corner radius/padding to fit the narrower
  width, matching `reference/Kaki - iPhone.dc.html`'s stat card sizing. Cards stay in a
  single row (matching the prototype, which also keeps them in a row, not vertically
  stacked) — just resized to fit compact width.
- `Sources/Features/Summary/SummaryView.swift`: added `isCompact`. Streak footer drops
  " · keep it going tomorrow" in compact. Buttons keep their existing actions/identifiers
  (`back-to-decks` → `onBackToDecks`, `study-another` → `onStudyAnother`); only the label
  text and per-device copy change: compact shows "Study another script" / "Back home",
  regular keeps "Back to decks" / "Study another deck" — exactly per the task's mapping.
- `Sources/Features/DeckSetup/DeckSetupSheet.swift`: no changes — the medium-detent-on-
  iPhone / formSheet-on-iPad behavior (task item 3) was already implemented at the
  `HomeView` call site by a prior task; confirmed via visual verification below.
