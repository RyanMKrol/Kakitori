# T018 — Note field mapping: name-based fields, HTML strip, sound refs

## 2026-07-17

**Implementation:** Created `NoteFieldMapper` enum to convert Anki note positional fields into named, cleaned Kakitori values.

**Key implementation details:**
- `struct MappedNote` with fields: `ankiGUID`, `ankiDeckID`, `target` (required), and optionals `pronunciation`, `english`, `category`, `hint`, `image`, `audioFilename`
- `enum NoteFieldMapperError` with case `emptyTarget(noteID: Int64)`
- `NoteFieldMapper.map(_:using:)` function that:
  1. Zips `model.fieldNames` with `note.fields` to create a `[String: String]` lookup by name
  2. Strips HTML defensively: replaces `<br>`, `<br/>`, `<br />`, `</div>`, `</p>` with space; removes `<[^>]+>` tags; decodes HTML entities (`&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`, `&nbsp;`); trims whitespace
  3. Extracts audio filename from `[sound:...]` regex in the Audio field
  4. Throws `emptyTarget` if Target cleans to empty
  5. Gracefully degrades missing optional fields to `nil`

**Test coverage:** 13 tests in `NoteFieldMapperTests.swift` covering:
- Canonical field order mapping
- Reordered field order (proves order independence)
- HTML entity decoding (`<b>ね</b>&amp;<br>next` → `ね& next`)
- Empty Audio field → `audioFilename = nil`
- Empty Target cleans to empty → throws `emptyTarget`
- Missing fields graceful degradation
- HTML entity decoding with multiple entity types
- Multiple `<br>` tag replacement
- Audio filename extraction from sound tags
- Audio field without sound tag → `audioFilename = nil`
- Whitespace trimming in cleaned fields
- `<div>` and `<p>` tag replacement
- `MappedNote` Equatable conformance

**Notes:**
- Used `NSRegularExpression` instead of `Regex` literal to avoid concurrency safety issues and global initializer constraints
- Avoided force-unwraps, using defensive `try?` for regex initialization
- All field mapping is purely value-level logic with no SwiftData, file I/O, or force-unwraps
- Audio regex pattern `\[sound:([^\]]+)\]` correctly extracts capture group (e.g., `bc4e77e6407e004b.mp3`) from sound tags
