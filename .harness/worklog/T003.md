# T003 — AppClock injected time source + 4 AM adjusted-day helper

## 2026-07-17

Implemented `Sources/Support/AppClock.swift`: a `Sendable` value type wrapping an injected
`now: @Sendable () -> Date` closure, a `timeZone`, and a derived `Calendar`. `AppClock.system`
is the sole place that constructs `Date()` directly; `AppClock.fixed(_:timeZone:)` gives tests
and previews a deterministic clock. `adjustedDay(for:)` implements the Anki-style 4 AM local
day-rollover from docs/03-srs-algorithm.md §7 by subtracting 4 hours and formatting the result
as `yyyy-MM-dd` (POSIX locale, clock's time zone) with a `DateFormatter` built locally inside the
method (not a stored property) to stay Swift 6 strict-concurrency clean. `today` derives from
`adjustedDay(for: now())`.

Added `Tests/UnitTests/AppClockTests.swift` (XCTest) covering all four `## Done when` rollover
cases (03:59/04:00/00:30/23:30 Asia/Tokyo) plus the `today == adjustedDay(for: now())` invariant,
using dates built from `DateComponents` against a fixed `Asia/Tokyo` clock.

SwiftFormat's `redundantSendable` rule dropped the explicit `: Sendable` conformance on the
struct declaration (implicit `Sendable` conformance for an all-`Sendable`-member internal struct
is redundant per this project's `.swiftformat` config) — the type is still Sendable-clean, only
the explicit annotation from the spec text was removed by the formatter.
