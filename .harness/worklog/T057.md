## 2026-07-18 — T057: ApkgImporter re-import idempotence + soft delete

Re-attempt of T021, unblocked by widening scope to include `Sources/Models/Note.swift`.

### What I did

1. **Renamed `Note.isDeleted` → `Note.isSoftDeleted`** (stored property + init param), fixing
   the NSManagedObject `isDeleted` write collision that blocked T021. `ApkgImporter`'s
   first-import note construction now sets `isSoftDeleted: false`.
   - Three read-only call sites outside this task's scope (`Sources/Features/Home/TodayBannerView.swift`,
     `Sources/Features/Home/DeckCardView.swift` — 5 usages) still reference `note.isDeleted`.
     Since those files are out of scope, I kept a computed `var isDeleted: Bool { isSoftDeleted }`
     accessor on `Note` (in-scope file) purely as a read-only compatibility shim so the whole
     target still compiles without touching out-of-scope files. It carries no persistence/state
     of its own — it's a plain Swift computed property forwarding to the renamed stored flag.
2. **Added `Note.deck: Deck?`** — a direct owning-deck link, set on every note at creation
   (first import and re-import). This was necessary: notes with no section (i.e. cards that
   live directly under the top-level Anki deck, no subdeck) are not reachable via
   `Deck.sections.flatMap(\.notes)`, so the original re-import plan (matching existing notes by
   walking `deck.sections`) silently missed ~49 of the fixture's 114 notes and duplicated them
   on every re-import (114 → 163). The direct `note.deck` link + a `FetchDescriptor<Note>`
   predicate on `deck?.id` fixes this for both matching and soft-delete detection. This is a
   schema addition beyond the literal task spec, but stays entirely within the in-scope
   `Sources/Models/Note.swift` (no `Deck.swift` change needed — one-directional relationship).
3. **Implemented re-import in `ApkgImporter.apply(_:)`**: fetches an existing `Deck` by
   `sourceDeckName`; if found, `reimport(_:into:)` updates matched notes in place (content only —
   `note.schedule` is never touched), inserts genuinely new notes with a fresh `.new` `CardSchedule`,
   and soft-deletes (`isSoftDeleted = true`) notes absent from the incoming set. Sections are
   created on first sight and reused by name; a note whose section changed between imports is
   moved. Saves once at the end.
4. **Wired media re-copy**: `ParsedDeck` now carries `extractionDirectory`; `parse(url:)` no
   longer deletes its temp extraction dir in `defer` (moved cleanup to `importDeck(from:)`, plus
   an error-path cleanup inside `parse` itself so a throw doesn't leak the temp dir). After
   `apply` returns the deck UUID, `importDeck(from:)` calls
   `MediaStore(baseURL:).copyMedia(manifestURL:payloadDirectory:deckID:)` against the extraction
   directory's `media` manifest — `MediaStore` already overwrites by name, so re-import refreshes
   media files in place.

### Tests (`Tests/UnitTests/ApkgReimportTests.swift`, new)

- `testReimportIsIdempotent` — `importDeck` twice → still 1 `Deck`, 114 unique `Note.id`s, all
  `isSoftDeleted == false`.
- `testSchedulingSurvivesReimport` — mutates one note's schedule to a distinctive state
  (`.review`, interval 5, ease 2.6, fixed future `dueAt`) before re-import; asserts it survives
  verbatim, and untouched notes stay `.new` / `dueAt == nil`.
- `testSoftDeleteThenRevival` — builds a doctored `ParsedDeck` via `notes.dropLast()`, calls
  `apply` directly: the dropped note still exists with `isSoftDeleted == true` and a non-nil
  schedule, 113 notes remain non-deleted; re-applying the full parsed deck flips it back to
  `isSoftDeleted == false`, 114 total.
- `testMediaIsCopiedAndStableAcrossReimport` — at least one `.mp3` lands under
  `<mediaBase>/Media/<deckUUID>/` after `importDeck`; file count unchanged after a second import.
