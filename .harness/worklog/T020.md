# T020 — ApkgImporter: first-import upsert into SwiftData

## 2026-07-17

**Implementation:** Added the orchestrating `ApkgImporter` actor that ties the existing
`ZipArchive` → `AnkiCollection` → `NoteFieldMapper` → `TargetSegmenter` pipeline together and
upserts `Deck`/`Section`/`Note`/`CardSchedule` rows into SwiftData for a first import.

**Key implementation details:**
- `Sources/Importer/NoteIdentity.swift`: `enum NoteIdentity` with
  `static func uuid(forAnkiGUID:) -> UUID` — SHA-256 of the guid's UTF-8 bytes, first 16 digest
  bytes packed into a `uuid_t` → `UUID(uuid:)`. Deterministic per guid.
- `Sources/Importer/ApkgImporter.swift`:
  - `enum ImporterError`: `badZip`, `noAnkiBuilderModel`, `zeroNotes`.
  - `struct ImportedNote` / `struct ParsedDeck` — parsed intermediate representation.
  - `actor ApkgImporter { init(container: ModelContainer, mediaBaseURL: URL) }` creates its own
    `ModelContext(container)` inside the actor (never passes a `ModelContext` across the actor
    boundary). `mediaBaseURL` is stored for T021 (media copy is out of scope here).
  - `parse(url:)` (internal, not private) extracts the zip into a temp dir under
    `FileManager.default.temporaryDirectory`, opens `AnkiCollection`, maps
    `ZipArchiveError`/`AnkiCollectionError` to the matching `ImporterError`, picks the first
    model with a `Target` field, determines the Kakitori deck name as the top-level (`::`-split)
    part of the Anki deck with the most notes (ignoring an empty `"Default"` deck), and maps each
    note of that model through `NoteFieldMapper` + `TargetSegmenter` (skipping notes of other
    models and notes whose `Target` cleans to empty).
  - `apply(_:)` (internal, not private) inserts one `Deck` (`sourceDeckName == name`), one
    `Section` per distinct section name, one `Note` per parsed note with
    `id = NoteIdentity.uuid(forAnkiGUID:)`, and one `CardSchedule` per note in state `.new`, then
    saves the context and returns the deck's `id`.
  - `importDeck(from:)` = `parse` + `apply`, matching the docs/05 §6 pipeline shape.
  - Split into small private helpers (`extract`, `openCollection`, `topDeckFullName(in:)`,
    `importedNote(from:model:deckLookup:topDeckFullName:)`, `sectionName(forNoteDeck:topDeckFullName:)`)
    to keep cyclomatic complexity / function length under the SwiftLint `--strict` thresholds.

**Test coverage:** `Tests/UnitTests/ApkgImporterTests.swift` (XCTest), against an in-memory
`ModelContainer` and a temp media base directory:
- `testFirstImportUpsertsDeckAndNotes` — imports the `kana-deck-v5.apkg` fixture and asserts
  exactly 1 `Deck` with non-empty, equal `name`/`sourceDeckName`; exactly 114 `Note` rows, all
  `isDeleted == false`; every note has a non-nil `schedule` in state `.new` and non-empty `units`.
- `testNoteIdentityIsDeterministic` — `NoteIdentity.uuid(forAnkiGUID:)` is stable for the same
  guid and differs for a different guid.
- `testImportingGarbageBytesThrowsBadZip` — importing a temp file of non-zip bytes throws
  `ImporterError.badZip`.
