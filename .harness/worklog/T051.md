# T051 — Dynamic Type + VoiceOver pass

## 2026-07-18

Cross-cutting accessibility pass per docs/06-ui-design.md §4: chrome text scales with Dynamic
Type, answer glyphs / guide-box content stay fixed, hit targets are ≥44pt, and VoiceOver gets a
canvas custom action, spoken grade intervals, and hidden decorative glyphs.

### What changed

- `Sources/Support/Theme.swift`:
  - Added `japaneseDisplayFontFixed(size:bold:)` (`Font.custom(_:fixedSize:)`) for the big answer
    glyph and guide-box content — `japaneseDisplayFont(size:bold:)` (plain `Font.custom(_:size:)`)
    already scales with Dynamic Type by default, so it's kept as-is for prompt/answer metadata
    (reading, hint) which SHOULD scale.
  - Added a `ScaledFontModifier` (`@ScaledMetric`) + `View.kakitoriFont(size:weight:design:)`
    helper: replaces every literal `.font(.system(size:weight:))` / `KakitoriTheme.smallCapsLabel`
    chrome-text call site with a Dynamic-Type-scaling equivalent that matches the exact point size
    at the default (`.large`) content size category, so the baseline screenshot is unchanged.
    `smallCapsLabel(size:)` was removed (all 6 call sites switched to `.kakitoriFont`).
- Converted every chrome `Text` across `Sources/Features/` from fixed `.system(size:)` /
  `smallCapsLabel` to `.kakitoriFont(...)`: Home (header, empty state, stats, today banner, deck
  cards), DeckSetup (mode caption, start button), Session (prompt panes, canvas hint/pill buttons,
  action row, top bar chips), Summary (headline, grade grid, buttons).
- Converted the big answer glyph (`PromptPaneView.answerBlock`'s target `Text`) and the guide-box
  content glyphs (`GuideBox`'s trace overlay, `GuideBoxRow`'s inline punctuation) to
  `japaneseDisplayFontFixed` / `Font.custom(_:fixedSize:)` so they stay a constant point size
  regardless of Dynamic Type — they're the content being written, not chrome. Guide-box dimensions
  were already computed from `GeometryReader`, not `@ScaledMetric`.
- Hit targets ≥44pt: `CanvasPaneView`'s undo/clear pill buttons keep their small visual size but
  get `.frame(minWidth: 44, minHeight: 44).contentShape(Rectangle())` so the tap target meets the
  bar without changing the pill's appearance; show-answer/grade buttons switched `frame(height:)`
  to `frame(minHeight:)` so they can grow with scaled label text.
- Grade buttons (`ActionRowView`) stack into full-height `VStack` rows once
  `dynamicTypeSize.isAccessibilitySize` is true (was always an `HStack`), per docs/06 §4's "grade
  buttons full-height rows on small screens".
- VoiceOver:
  - `CanvasPaneView`'s canvas `ZStack` gets `.accessibilityAction(named: "Show answer") {
    viewModel.showAnswer() }`, invoking the same idempotent reveal as the show-answer button.
  - Grade buttons get `accessibilityLabel(grade — due in <interval-preview label>)`, e.g.
    `"Again — due in <1m"` (reuses the existing preview label already shown on the button).
  - `.accessibilityHidden(true)` added to: the 書 header mark (`HomeView` header + empty state),
    deck-row representative glyphs (`DeckCardView`'s あ/ア/語), mode-picker glyphs
    (`DeckSetupSheet`'s 書/聞/訳/思/混 — already labelled by the row's own text) and the top bar's
    mode glyph in `SessionView`, and the summary 済 disc (`SummaryView`).
  - `StatsRowView`'s `StatCard` and `SummaryView`'s `GradeCard` (previously a bare number + a
    separate caption) now report one combined `accessibilityLabel` (e.g. `"3 Day streak"`,
    `"2 Again"`) instead of reading the number and caption as separate elements.
