# T029 — SessionViewModel: session state machine

## 2026-07-18

Built `Sources/Features/Session/SessionViewModel.swift`: an `@MainActor @Observable` class that
owns a running practice session — builds a `SessionQueue` from a deck's non-deleted notes (mapping
each `CardSchedule` into a `ScheduleSnapshot`), drives `prompt → revealed → finished` via
`showAnswer()` / `grade(_:)` / `close()`, applies `SM2Scheduler` (with an injected `SplitMix64` RNG
seeded from `init`), writes results straight back onto the note's `CardSchedule`, and persists via
`StatsRecorder.recordGrade` / `recordStudySeconds`. Exposes `newCount`/`learnCount`/`dueCount`
(forwarded from the live queue) and `gradePreviews` (via `SM2Scheduler().preview`) for the T033
grading UI. All "now" reads go through the injected `AppClock`; the 4 AM `endOfToday` boundary is
computed locally (no such helper existed on `AppClock` yet, and it's out of this task's scope to add
one there) by finding the next 4:00-AM-local instant at or after `now` using the clock's calendar.

**Spec deviation:** the spec's "## Do" step 1 said to create `Sources/Features/Session/PracticeMode.swift`
because "no mode type exists yet" — that's stale; `PracticeMode` (with an extra `.mixed` case) was
already added to `Sources/Features/DeckSetup/DeckSetupSheet.swift` by an earlier task. Creating a
second `PracticeMode` would be a duplicate-declaration build error, so `SessionViewModel` reuses the
existing type instead of authoring a new one.

Test-first: wrote `Tests/UnitTests/SessionViewModelTests.swift` against an in-memory
`ModelContainer` (`Deck`/`Section`/`Note`/`CardSchedule`/`DailyStats`) and a scripted `AppClock`
before writing the implementation; confirmed the tests failed to compile/run against a stub, then
built `SessionViewModel` until all four passed:
- `testHappyPathThreeReviewCards` — 3 review cards graded Good/Good/Easy with 30s clock advances
  each → `.finished`, `cardsWritten == 3`, `gradeCounts [.good: 2, .easy: 1]`, `seconds == 90`,
  persisted intervals `[25, 25, 34]`, `DailyStats` `cardsWritten 3` / `reviewsDone 3` /
  `secondsStudied 90`.
- `testAgainReentersBeforeFinish` — Again on a review card re-enters the queue (relearning,
  lapses 1) without finishing; Good 600s later graduates to `intervalDays == 5`,
  `cardsWritten == 2`, `gradeCounts [.again: 1, .good: 1]`.
- `testClosePreservesPartialProgress` — grade first of two cards, then `close()`: first card's
  schedule updated + `DailyStats.cardsWritten == 1` + `secondsStudied` matches elapsed; second
  card's `CardSchedule` untouched.
- `testGradeBeforeShowAnswerIsNoOp` — `grade(.good)` while `.prompt` changes nothing.
