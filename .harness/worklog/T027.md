## 2026-07-17

**Implementation complete.** Added `next(now:)`, `markGraded(_:newSnapshot:now:)`, and `isFinished` to `SessionQueue`.

### Changes

1. **SessionQueue.swift:**
   - `func next(now: Date) -> QueueEntry?` — Returns the first currently-due entry (dueAt == nil or <= now), or the earliest-due entry if none are currently due (early serve). Returns nil only when empty.
   - `mutating func markGraded(_ id: UUID, newSnapshot: ScheduleSnapshot, now: Date)` — Removes the entry with `id`, then re-appends if the new state is `.learning` or `.relearning` (sub-day re-entry). No-op if id not found. Cards in `.review` state leave the session.
   - `var isFinished: Bool` — True when entries.isEmpty.

2. **SessionQueueReentryTests.swift (new file):**
   - 19 comprehensive unit tests covering:
     - Basic next() / isFinished behavior
     - Re-queueing of sub-day cards (Again, relearning)
     - Graduation removal from session
     - Early-serve when queue would otherwise idle
     - End rule (session open while any sub-day card pending, closed when all >= 1-day)
     - Re-entry ordering (position + due time matter)
     - Unknown ID no-op
     - Chip count consistency
   - All tests use hand-constructed snapshots with t0 = Date(timeIntervalSince1970: 1_000_000).
