## Overview
DeckCardView renders `Text(deck.jpTitle ?? deck.name)` in the large Japanese-display (Mincho serif) font and then `Text(deck.name)` bold beneath it. When `jpTitle` is nil the two lines are identical, and a long Latin deck name renders in the serif JP slot. Make the card robust so it never shows two identical lines and never puts a Latin fallback in the JP-display slot — a defensive fix independent of whether a friendly jpTitle is set upstream.

## Do
In both layouts (compact ~L33/37 and regular ~L115/120): render the large Japanese-display line ONLY when `deck.jpTitle` is a real, non-empty value distinct from `deck.name` (i.e. drop the `?? deck.name` fallback for that slot — when jpTitle is nil/empty, omit the JP-display Text entirely rather than duplicating the name). Keep the bold `Text(deck.name)` line always present as the deck's primary label. Preserve existing fonts, spacing, lineLimit(1), accessibilityIdentifier ('deck-row-…'), and every other element of both layouts.
