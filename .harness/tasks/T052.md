# T052 — Reduce Motion support

## Overview

Respect `@Environment(\.accessibilityReduceMotion)` per docs/06-ui-design.md §4 "Reduce Motion:
disable slide, keep crossfades", centralized in one Theme-level helper so every future view
inherits the rule instead of re-implementing it.

## Do

1. Add a small helper in `Sources/Support/Theme.swift` (or a new `Sources/Support/Motion.swift`),
   e.g.:
   - `KakitoriTheme.cardTransition(reduceMotion: Bool) -> AnyTransition` — returns the existing
     crossfade + slight horizontal slide (docs/06 §3) when `reduceMotion == false`, and a plain
     `.opacity` crossfade when `true`.
   - `KakitoriTheme.emphasisTransition(reduceMotion: Bool) -> AnyTransition` — scale-in normally,
     plain `.opacity` under Reduce Motion.
   Keep it pure (inputs → transition/animation values), no environment reads inside the helper.
2. Route ALL motion in `Sources/Features/` through it, passing each view's
   `@Environment(\.accessibilityReduceMotion)` value (docs/06 §3 is the motion inventory):
   - Session card-to-card: prompt-pane slide becomes crossfade-only; the canvas clear is not an
     animation and is unaffected.
   - Answer reveal: the prompt→answer crossfade is KEPT as-is in both settings; the grade row's
     spring-up becomes a crossfade under Reduce Motion.
   - Summary: the 済 disc scale-in becomes a plain crossfade under Reduce Motion.
3. Crossfades are never removed — Reduce Motion swaps movement/scale for opacity, it does not
   disable animation entirely.

## Done when

- `grep -rn "\.transition(\|withAnimation" Sources/Features/` shows every animated transition
  obtained from the Theme helper (no view builds its own slide/scale transition inline).
- `./build_run.sh Kakitori-Sim -seedDemoData YES` — `screenshots/latest.png` renders identically
  to before this task (static appearance must not change; this task only alters motion).
- The helper's reduce-motion mapping is covered by a small XCTest where practical (e.g. assert the
  helper picks the reduced path via an intermediate enum/case it returns); if `AnyTransition`
  opacity cannot be asserted directly, structure the helper as a two-step
  `motionCase(reduceMotion:) -> MotionCase` + `transition(for:)` so the case mapping is testable
  in `Tests/UnitTests/`.
