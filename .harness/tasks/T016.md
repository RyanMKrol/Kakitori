# T016 — ZipArchive: minimal read-only .apkg (zip) extractor

## Overview

Hand-roll a read-only zip extractor (there are NO third-party dependencies in this project — no
ZIPFoundation) so the importer can open `.apkg` files, which are plain zip archives per
docs/04-content-and-data.md §1 (`collection.anki2` + `media` manifest + numbered payload files).

## Do

Create `Sources/Importer/ZipArchive.swift` containing `struct ZipArchive` and
`enum ZipArchiveError: Error, Equatable` with cases: `notAZip`, `zip64Unsupported`,
`encryptedEntryUnsupported`, `unsupportedCompressionMethod(UInt16)`, `entryNotFound(String)`,
`decompressionFailed(String)`, `malformed`.

Public API — exactly:

```swift
init(url: URL) throws            // reads the whole file into Data and parses the central directory
var entryNames: [String]         // all entry names, central-directory order
func extract(_ name: String, to url: URL) throws     // writes one entry's bytes to `url`
func extractAll(to directory: URL) throws            // every non-directory entry into `directory`
```

Parsing rules (all multi-byte integers are little-endian; read them byte-by-byte from `Data` —
do not use unaligned load tricks):

1. **End of Central Directory (EOCD)**, signature `0x06054b50`: scan backwards from the file tail
   (it sits within the last 65,557 bytes because of the variable-length comment). File shorter
   than 22 bytes or no signature found → throw `.notAZip`. From the EOCD read: total entry count
   (UInt16 at offset +10), central-directory size (UInt32 at +12), central-directory offset
   (UInt32 at +16).
2. **Reject zip64 gracefully**: if entry count == `0xFFFF` or the CD offset == `0xFFFFFFFF` (or
   any per-entry size/offset below == `0xFFFFFFFF`) throw `.zip64Unsupported`. Our decks never
   need zip64.
3. **Central-directory entries**, signature `0x02014b50` (anything else → `.malformed`). Per
   entry, relative to the signature: general-purpose flags UInt16 at +8, compression method
   UInt16 at +10, compressed size UInt32 at +20, uncompressed size UInt32 at +24, name length
   UInt16 at +28, extra length UInt16 at +30, comment length UInt16 at +32, local-header offset
   UInt32 at +42, then the UTF-8 name at +46. Next entry begins at
   +46 + nameLen + extraLen + commentLen. If flags bit 0 is set the entry is encrypted →
   `.encryptedEntryUnsupported`.
4. **Extraction**: locate the local file header (signature `0x04034b50`) at the entry's
   local-header offset; read the LOCAL header's own name length (UInt16 at +26) and extra length
   (UInt16 at +28) — they can differ from the central copy. Entry data starts at
   offset + 30 + nameLen + extraLen. ALWAYS use the central-directory compressed/uncompressed
   sizes (local ones are zero when flags bit 3 is set).
   - Method 0 (STORED): the data slice is the payload verbatim.
   - Method 8 (DEFLATE): `import Compression` and inflate with
     `compression_decode_buffer(dst, uncompressedSize, src, compressedSize, nil, COMPRESSION_ZLIB)` —
     Apple's `COMPRESSION_ZLIB` is the raw DEFLATE stream, which is exactly what zip stores.
     A return value != uncompressedSize → `.decompressionFailed(name)`.
   - Any other method → `.unsupportedCompressionMethod(method)`.
5. `extractAll` skips directory entries (name ends in `/`) and rejects names containing `..`
   (path traversal). `.apkg` entries are flat names like `collection.anki2`, `media`, `0`, `1`, …

No force-unwraps or `try!`; every out-of-bounds read paths to `.malformed`.

Also create `Tests/UnitTests/FixtureLocator.swift`: `enum Fixture` with

## Done when

`Tests/UnitTests/ZipArchiveTests.swift` (XCTest) passes with these assertions:

- `ZipArchive(url: Fixture.kanaDeckURL)` succeeds and `entryNames` contains both
  `"collection.anki2"` and `"media"`.
- `extractAll(to:)` into `FileManager.default.temporaryDirectory` + a UUID subdirectory writes a
  `collection.anki2` whose first 16 bytes equal `"SQLite format 3\0"` (bytes
  `53 51 4C 69 74 65 20 66 6F 72 6D 61 74 20 33 00`).
- `extract("collection.anki2", to:)` on its own produces the same bytes as the extractAll copy.
- Initialising against a temp file of 100 random non-zip bytes throws `ZipArchiveError.notAZip`.
- `extract("nope.txt", to:)` throws `ZipArchiveError.entryNotFound("nope.txt")`.
