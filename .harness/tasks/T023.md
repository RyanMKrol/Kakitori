# T023 — SM2Scheduler: learning + relearning transitions

## Overview

Create `SM2Scheduler` and implement grade application for `new`, `learning`, and `relearning`
snapshots per docs/03-srs-algorithm.md §4 "Learning / relearning cards". Review-state grading is
T024; `preview` + protocol conformance is T025.

## Do

Create `Sources/Scheduler/SM2Scheduler.swift` (pure Foundation only — no SwiftData/SwiftUI imports,
no `Date()`/`.now`, no system RNG; violations fail the build). T022 already provides `Grade`,
`ScheduleSnapshot`, `SchedulePreview`, `SRSConstants`, `SplitMix64`, and `CardState` exists in
`Sources/Models/CardSchedule.swift`.

Define `struct SM2Scheduler` (do **not** declare `: Scheduler` yet — T025 adds conformance) with:

```swift
init()
func apply(_ grade: Grade, to card: ScheduleSnapshot, now: Date,
           rng: inout some RandomNumberGenerator) -> ScheduleSnapshot
```

`rng` is unused in this task (fuzz arrives in T024) but the signature is fixed now. Behavior, using
`SRSConstants` (learning steps = [60s, 600s]; relearning has a single 600s step; graduating
interval 1 day; easy graduating interval 4 days; day = 86_400s):

- **`.new`** — treat exactly as `.learning` at `stepIndex` 0 (docs/03 §1: new enters learning on
  first grade; state becomes `.learning` or `.review` per the rules below).
- **`.learning`** (steps `[60, 600]` seconds):
  - **again** → `stepIndex = 0`, `state = .learning`, `dueAt = now + 60`.
  - **hard** → repeat current step: `dueAt = now + learningStepsSeconds[stepIndex]`, no advance.
  - **good** → advance; if `stepIndex` was the last step (index 1), **graduate**:
    `state = .review`, `intervalDays = 1`, `stepIndex = 0`, `dueAt = now + 1 × 86_400`; otherwise
    `stepIndex += 1`, `dueAt = now + learningStepsSeconds[newIndex]` (600).
  - **easy** → graduate immediately: `state = .review`, `intervalDays = 4`, `stepIndex = 0`,
    `dueAt = now + 4 × 86_400`.
  - `easeFactor`, `lapses`, `intervalDays` (until graduation) are unchanged by learning grades.
- **`.relearning`** (single 600s step; `intervalDays` still holds the pre-lapse interval `I`):
  - **again** / **hard** → stay `.relearning`, `stepIndex = 0`, `dueAt = now + 600`.
  - **good** → re-graduate: `state = .review`, `intervalDays = max(1, (I × 0.5).rounded())`,
    `stepIndex = 0`, `dueAt = now + intervalDays × 86_400` (docs/03 §4 re-graduation rule).
  - **easy** → re-graduate at `intervalDays = max(4, max(1, (I × 0.5).rounded()))` — the docs/03
    "Easy graduates at 4 days" floor combined with the ×0.5 re-graduation rule, so Easy is never
    worse than Good.
- **`.review`** — placeholder until T024: return the input snapshot unchanged (no fatalError).
- **No fuzz anywhere in this task** — all dueAt offsets above are exact. Use bounds-safe indexing
  (clamp `stepIndex` into the steps array); no force-unwraps.
