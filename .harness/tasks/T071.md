## Overview
The user's handwriting/ink on the PencilKit trace canvas renders as low-contrast mid-grey against the paper in BOTH light and dark mode, hard to read. It should be near-black in light (#1A1A1A) and near-white in dark (#F0EDE6) per KakitoriTheme.inkUIColor. Root cause: WritingCanvas.swift:33 sets the PKInkingTool once in makeUIView using the DYNAMIC KakitoriTheme.inkUIColor, and updateUIView (line 45) is EMPTY — so PencilKit resolves the dynamic color against a base/default trait collection (not the canvas's real appearance) and never refreshes, and its built-in stroke light/dark adaptation washes the ink to grey. This is a CONTRAST bug on the canvas layer (simulator-verifiable), NOT the needs-human Apple-Pencil ink-FEEL M1 gate. Only the user's own ink must become high-contrast; the intentionally-faded trace guides (KakitoriTheme.inkFaint) must NOT change.

## Do
In Sources/Features/Session/Canvas/WritingCanvas.swift, make the ink tool's color a CONCRETE color resolved against the canvas view's actual traitCollection (so PencilKit gets a fixed near-black in light / near-white in dark instead of a dynamic color it resolves once against a base trait collection): resolve KakitoriTheme.inkUIColor against canvasView.traitCollection when creating the tool, and REFRESH the tool whenever the appearance changes so it tracks light/dark — populate updateUIView(_:context:) (currently empty at line 45) to re-set canvasView.tool with the freshly-resolved color, and/or observe trait changes (e.g. registerForTraitChanges / traitCollectionDidChange on a PKCanvasView subclass, or overrideUserInterfaceStyle) so the resolved ink color follows the current appearance. Prevent PencilKit from lightening/inverting the dark ink and/or apply a constant-opacity approach if the .pen tool's pressure/velocity opacity is contributing to the faintness, so a normal stroke reads at full contrast. If a reusable 'resolve inkUIColor to a concrete UIColor for a trait collection' helper is cleaner, add it to Sources/Support/Theme.swift (KakitoriTheme) rather than inlining — do NOT change inkUIColor's dynamic definition, inkLight/inkDark values, inkFaint, or any guide/box colors. Do NOT touch GuideBox.swift, TraceGuideLayer, or any inkFaint usage — the faded trace guides are intentional and must stay low-contrast. Keep the change scoped to making the USER's drawn ink high-contrast in both appearances on both iPhone and iPad.
