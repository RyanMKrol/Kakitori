## Overview
In Settings, the two daily-limit rows ('New cards per day', 'Max reviews per day') are +/- Steppers, which are tedious to nudge across their wide ranges. Replace each with a direct numeric input the user can type into (number keyboard), while preserving the Stepper's existing range as a clamp so out-of-range or non-numeric entries can't be committed. Only these two rows change; the 'Audio autoplay' and 'Show romaji' Toggles below them are untouched. This is a pure UI swap in SettingsView — the backing @AppStorage keys ('newCardsPerDay', 'maxReviewsPerDay') and their defaults (10 / 100) stay the same, so AppSettings.swift and the SessionQueue caps that read those keys keep working unchanged.

## Do
In Sources/Features/Settings/SettingsView.swift, replace the two Steppers with numeric text-entry controls:
- 'New cards per day' (@AppStorage "newCardsPerDay", default 10): currently `Stepper(value: $newCardsPerDay, in: 1 ... 50, ...)` at ~lines 18-25. Preserve the 1...50 bounds as the clamp range.
- 'Max reviews per day' (@AppStorage "maxReviewsPerDay", default 100): currently `Stepper(value: $maxReviewsPerDay, in: 10 ... 500, ...)` at ~lines 31-38. Preserve the 10...500 bounds as the clamp range.
For each row, render a TextField bound to the Int-backed @AppStorage value that: uses a number-pad keyboard (`.keyboardType(.numberPad)`); commits/validates on done or when focus leaves (blur); on commit, clamps the entered value into the row's range and, on empty/non-numeric input, reverts to the last valid value (never writes an out-of-range or NaN value to @AppStorage). Keep the row's label text and, critically, the SAME accessibilityIdentifier on the input control ('settings-new-per-day' for new cards, 'settings-max-reviews' for max reviews) so existing/future UI queries still resolve. Keep the two Toggles ('Audio autoplay', 'Show romaji') exactly as they are. Do NOT change AppSettings.swift, the @AppStorage keys, or the defaults — this is a UI-only change. Prefer a small local helper/binding (e.g. a clamped Int<->String binding, or a focus-driven commit) over per-keystroke writes so partial typing (e.g. clearing the field to retype) doesn't clobber the stored value mid-edit.

- The two Toggles ('Audio autoplay', 'Show romaji') are unchanged; AppSettings.swift, the @AppStorage keys, and defaults (10 / 100) are unchanged, so SessionQueue daily caps still read the same values.
