# T027 — SessionQueue: sub-day re-entry + session end rule

## Overview

Add in-session mutations to `SessionQueue`: a graded card scheduled sub-day re-enters the live
queue and is served early if the queue would otherwise idle; the session ends only when every
queued card has reached ≥ 1-day scheduling (docs/03-srs-algorithm.md §5).

## Do

Extend `Sources/Scheduler/SessionQueue.swift` (Foundation only; no SwiftData/SwiftUI; no
`Date()`/`.now` — time comes in as parameters; do NOT use `SM2Scheduler` here). Add to
`SessionQueue`:

```swift
/// The card to show next, or nil when the session is finished.
func next(now: Date) -> QueueEntry?
/// Record the post-grade snapshot for a card and update the live queue.
mutating func markGraded(_ id: UUID, newSnapshot: ScheduleSnapshot, now: Date)
var isFinished: Bool   // entries.isEmpty
```

Rules (docs/03 §5: "a card graded into a sub-day interval re-enters the same session queue when
its due time arrives; if the queue would otherwise be empty, it is served early. A session
therefore ends only when every queued card has reached ≥1-day scheduling"):

- `next(now:)` returns the first entry in queue order whose `snapshot.dueAt` is nil (new cards) or
  `<= now`. If no entry is currently due but entries remain, return the entry with the **smallest**
  `dueAt` (early serve — never return nil while entries exist). Returns nil only when empty.
- `markGraded(_:newSnapshot:now:)` removes the entry with `id` (unknown id → no-op, no crash).
  Then, if `newSnapshot.state` is `.learning` or `.relearning` (i.e. sub-day scheduling — those
  states' dues are minutes), append `QueueEntry(id: id, snapshot: newSnapshot)` to the back of
  `entries` so it re-enters the live queue. If `newSnapshot.state` is `.review`
  (`intervalDays >= 1` by construction), the card leaves the session for good.
- The T026 chip counts (`newCount`/`learnCount`/`dueCount`) keep reflecting the live `entries`
  after every mutation — no change needed beyond them being computed properties.

## Done when

`Tests/UnitTests/SessionQueueReentryTests.swift` (XCTest) passes, constructing all snapshots by
hand (no SM2Scheduler), `t0 = Date(timeIntervalSince1970: 1_000_000)`, queue built from two
learning cards A (dueAt t0 − 60) and B (dueAt t0 − 30):

- `next(now: t0)` returns A (earliest due).
- **Again re-queues at +1m**: `markGraded(A, learning step 0 dueAt t0 + 60, now: t0)` → queue has
  2 entries, `isFinished == false`, and `next(now: t0)` now returns B (the only currently-due
  card).
- **Graduation removes**: `markGraded(B, review intervalDays 1 dueAt t0 + 86_400, now: t0)` →
  1 entry left; B never reappears.
- **Empty-queue early serve**: with only A left (due t0 + 60) and `now: t0` (before its due time),
  `next(now: t0)` returns A anyway.
- **End rule**: `isFinished` stays false while A is sub-day; after
  `markGraded(A, learning step 1 dueAt t0 + 660, now: t0 + 60)` it is still false (sub-day card
  pending); after `markGraded(A, review intervalDays 1 dueAt t0 + 660 + 86_400, now: t0 + 660)`
  → `isFinished == true` and `next` returns nil.
- **Re-entry when due**: with re-queued A (dueAt t0 + 60) and a due learning card C in the queue,
  `next(now: t0 + 61)` prefers the earlier-positioned due entry (C, still at the front) — and once
  C is graded to review, `next(now: t0 + 61)` returns A as normally due (not early-served).
