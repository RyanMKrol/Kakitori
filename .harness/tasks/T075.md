## Overview
The Home screen cannot scroll. HomeView.body (Sources/Features/Home/HomeView.swift, ~L16-37) builds the entire page as a fixed VStack(spacing:24) { header; StatsRowView(); if !decks.isEmpty { VStack(spacing:16){ TodayBannerView(now:); deckList } } ... }.padding() with no ScrollView anywhere in the hierarchy. deckList (~L229-262) is a plain VStack(spacing:12) on compact (iPhone) and a two-column LazyVGrid on regular (iPad), neither wrapped in a ScrollView — so a LazyVGrid there neither scrolls nor lazily loads. Once header + StatsRow + TodayBanner + N deck cards exceed the viewport height, the overflow is clipped and decks below the fold are unreachable. Fix: put the populated content column inside a ScrollView so it scrolls on BOTH form factors. Per the standing dual-device rule this must be verified clean on Kakitori-Sim (iPad) AND Kakitori-Sim-Phone (iPhone); do not fix one form factor and regress the other.

## Do
In Sources/Features/Home/HomeView.swift, wrap the populated-state content so the Home screen scrolls vertically on both iPhone (compact VStack deckList) and iPad (regular LazyVGrid deckList). Introduce a vertical ScrollView around the scrolling content column (the header + StatsRowView + TodayBannerView + deckList stack) so every deck card is reachable below the fold. The default is that the WHOLE content column scrolls together (header/StatsRow scroll away with the rest) unless the pending-question answer says the header/StatsRow should stay pinned — follow that answer. Keep all changes within Sources/Features/Home/**. iOS 18 min, Swift 6 strict concurrency, SwiftUI first-party only.
