## Overview
Replace the generic per-mode kanji glyphs in the DeckSetupSheet practice-mode picker with characters FROM the current deck, using the curated per-deck showcase list + mapping decided in conv-mode-glyphs-a, and — for any uncurated/imported deck — auto-picking distinct characters from that deck's OWN notes (owner-confirmed fallback). Today each mode row shows PracticeMode.glyph (書/聞/思/混/訳) — Sources/Features/DeckSetup/DeckSetupSheet.swift:10-18, rendered as the leading glyph in modeRow (~L142-150 via Text(mode.glyph)). The sheet currently receives only derived strings (jpTitle/enTitle/dueCount/availableModes) and has no access to the deck's characters, but the CALL SITE in HomeView.swift:47-54 already holds the full Deck (setupDeck) and computes deck.sections.flatMap(\.notes)…, so the natural change is to resolve the per-mode glyphs at the call site and pass them into the sheet. deck.sections.flatMap(\.notes).map(\.target) yields all the deck's characters ([String]).
