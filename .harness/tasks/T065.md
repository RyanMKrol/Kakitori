## Overview
The mode-picker glyphs in DeckSetupSheet should become characters FROM the current deck (interesting hiragana for Hiragana, interesting katakana for Katakana) instead of the generic kanji 書/聞/思/混/訳. WHICH characters showcase each bundled deck, and HOW a character maps onto each practice-mode row, are subjective/creative choices the owner must make — the loop must NOT invent them. This is the human decision unit: the owner picks the showcase characters per bundled deck and states the mapping rule, recorded as a curated static map so the dependent render unit (conv-mode-glyphs-b) can consume it. The FALLBACK for uncurated/imported decks is already DECIDED (auto-pick distinct characters from the deck's own notes — implemented in conv-mode-glyphs-b), so it is NOT part of this decision. Bundled decks today are keyed only by sourceDeckName (BundledDeckLoader.deckResourceNames = ['Tofugu-Hiragana','Tofugu-Katakana']); there is no per-deck config struct yet, so the curated list is a NEW static constant added here.
