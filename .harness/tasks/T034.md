# T034 — Session summary screen (お疲れさま！)

## Overview

Build the session-complete screen: the 済 disc with a gentle scale-in, the お疲れさま！ headline,
the written/minutes line, a 2×2 grade-count grid, the streak line, and the two exit buttons.

## Do

- Create `Sources/Features/Summary/SummaryView.swift` (new folder). Keep it decoupled from the
  session: `init(cardsWritten: Int, minutes: Int, againCount: Int, hardCount: Int, goodCount: Int,
  easyCount: Int, streakDays: Int, onBackToDecks: @escaping () -> Void, onStudyAnother: @escaping
  () -> Void)`. `SessionViewModel`'s finished-state summary numbers feed it — but that wiring is the
  later end-to-end task, not this one.
- Centred column on `KakitoriTheme.paper`; root container identifier `"summary-done"`
  (docs/02-product-spec.md §1.4, docs/06-ui-design.md §2.4):
  - `済` in a ~100pt accent-filled circle, paper-coloured glyph in
    `KakitoriTheme.japaneseDisplayFont`. On appear: one gentle scale-in (~0.7 → 1.0 spring).
    Nothing else animates (docs/06 §3: "single gentle scale-in of the 済 disc").
  - `お疲れさま！` — bold, ~34pt.
  - `Session complete — N cards written in M min` (exact wording, e.g.
    "Session complete — 12 cards written in 6 min"; docs/06 §2.4).
  - 2×2 grid of grade-count cards (white fill, hairline border): large numeral over a small-caps
    caption `Again` / `Hard` / `Good` / `Easy`; the Again numeral in accent.
  - Streak line: `🔥 N day streak · keep it going tomorrow` (docs/06 §2.4).
  - Buttons: `Back to decks` — primary, accent fill, identifier `"back-to-decks"`;
    `Study another deck` — quiet (white fill, hairline border), identifier `"study-another"`.
    docs/06 §2.4 makes Back to decks the primary (the prototype paints Study another deck red —
    the doc wins).
- Render check: `Tests/UnitTests/SummaryRenderTests.swift` — no container needed, just construct
  `SummaryView(cardsWritten: 12, minutes: 6, againCount: 1, hardCount: 2, goodCount: 7,
  easyCount: 2, streakDays: 5, ...)` with no-op closures, frame 1194×834, render with
  `ImageRenderer` (scale 2), write to `<repoRoot>/screenshots/T034-summary.png` (path from `#filePath`).
