# T029 — SessionViewModel: session state machine

## Overview

Create the `@Observable @MainActor` view model that owns a running practice session: builds the
queue, drives prompt → reveal → grade, applies `SM2Scheduler`, persists `CardSchedule` + stats,
handles sub-day re-entry, and ends with a summary (docs/05-architecture.md §5,
docs/02-product-spec.md §1.3–§1.4).

## Do

All in `Sources/Features/Session/`. Established, already-built pieces to reuse: `Deck`/`Note`
(`Sources/Models/`), `CardSchedule` + `CardState` (`Sources/Models/CardSchedule.swift`),
`StatsRecorder` (`Sources/Models/StatsRecorder.swift`), the Scheduler module (`Grade`,
`ScheduleSnapshot`, `SchedulePreview`, `SM2Scheduler`, `SessionQueue`, `SplitMix64`,
`SRSConstants`), and the injected clock in `Sources/Support/AppClock.swift` — **read AppClock.swift
first** and inject it; any `Date()`/`.now` or system RNG in this feature is a build-failing
violation.

1. `Sources/Features/Session/PracticeMode.swift` — no mode type exists yet; create
   `enum PracticeMode: String, CaseIterable, Sendable { case trace, listen, translate, recall }`
   (docs/02 §2; later tasks build the per-mode prompts — the VM only carries the choice).
2. `Sources/Features/Session/SessionViewModel.swift` — `@Observable @MainActor final class
   SessionViewModel`:
   - `init(deck:mode:modelContext:clock:seed:newPerDay: Int = 10, maxReviewsPerDay: Int = 100)`
     (`seed: UInt64` feeds a private `var rng = SplitMix64(seed:)`). Init maps each non-deleted
     note's `CardSchedule` fields into a `ScheduleSnapshot` (keyed by `note.id`), reads today's
     already-done counts (`newIntroduced`, `reviewsDone`) from the current adjusted day's
     `DailyStats` row (0 if missing), computes `endOfToday` = next 4:00 AM local after `clock`'s
     now (reuse the AppClock helper), calls `SessionQueue.build`, sets the current card, and
     records the session start time from the clock.
   - Exposed state: `phase` (`enum Phase: Equatable { case prompt, revealed, finished }`),
     `currentNote: Note?`, `gradeCounts: [Grade: Int]`, `cardsWritten: Int`,
     `summary: SessionSummary?` (`struct SessionSummary: Equatable { let gradeCounts: [Grade: Int];
     let cardsWritten: Int; let seconds: Int }`), live chip counts `newCount`/`learnCount`/
     `dueCount` forwarded from the queue, and `gradePreviews: [Grade: SchedulePreview]` computed
     via `SM2Scheduler().preview(for:now:)` on the current snapshot (for T033's buttons).
   - `showAnswer()` — only in `.prompt`, flips to `.revealed`.
   - `grade(_ g: Grade)` — only in `.revealed`: snapshot the current card, `SM2Scheduler().apply(g,
     to:, now: clock now, rng: &rng)`, write every result field back onto the card's
     `CardSchedule`, `StatsRecorder.recordGrade(previousState: <state before apply>, ...)`, bump
     `gradeCounts`/`cardsWritten`, `queue.markGraded(...)` (sub-day results re-enter, docs/03 §5),
     advance to `queue.next(...)` clearing reveal state back to `.prompt`. When `queue.isFinished`:
     record elapsed whole seconds via `StatsRecorder.recordStudySeconds`, set `summary`, phase
     `.finished`.
   - `close()` — mid-session ✕ (docs/02 §1.3): grades already given stay persisted (each `grade()`
     wrote through immediately); just record elapsed study seconds once; ungraded cards untouched
     (still due). Persistence errors: catch and stash in a `private(set) var lastError` — no
     `try!`, no force-unwraps.

## Done when

`Tests/UnitTests/SessionViewModelTests.swift` (XCTest, `@MainActor`, in-memory ModelContainer with
the full model schema, scripted clock from AppClock.swift, `seed: 42`) passes:

- **Happy path**: deck with 3 notes whose schedules are `.review`, I 10, EF 2.5, due before now.
  Trace session; for each card `showAnswer()` then grades Good, Good, Easy, advancing the scripted
  clock 30s before each grade → ends `.finished` with `summary.cardsWritten == 3`, `gradeCounts
  [.good: 2, .easy: 1]`, `summary.seconds == 90`; persisted `CardSchedule.intervalDays` are
  25, 25, 34 (fuzz never changes stored intervals); `DailyStats` row has `cardsWritten 3`,
  `reviewsDone 3`, `secondsStudied 90`.
- **Again re-enters before finish**: 1 review card (I 10, EF 2.5) graded Again → not finished,
  same card served again (early if need be), `CardSchedule.state == .relearning`, `lapses == 1`;
  advance clock 600s, grade Good → `.finished`, `intervalDays == 5`, `cardsWritten == 2`,
  `gradeCounts [.again: 1, .good: 1]`.
- **Close preserves partial progress**: 2 review cards; grade the first Good, then `close()` →
  first card's schedule updated + `DailyStats.cardsWritten == 1` + `secondsStudied` equals the
  scripted elapsed; second card's `CardSchedule` is byte-for-byte unchanged and still due.
- `showAnswer()` before any reveal is required for grading: calling `grade(.good)` while in
  `.prompt` is a no-op (state and store unchanged).
