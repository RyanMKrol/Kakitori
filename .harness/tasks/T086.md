## Overview
On the Settings screen, the two numeric fields — 'New cards per day' and 'Max reviews per day' — cannot be cleared to type a different value. If a field shows '1' and you backspace, the '1' reappears instantly, so you can never delete it and type 5/6/7 etc. Root cause in Sources/Features/Settings/SettingsView.swift: each TextField is bound to a local @State text string (newCardsText / maxReviewsText) and carries an `.onReceive(Just(<storedValue>)) { value in if <text>.isEmpty { <text> = "\(value)" } }` modifier (new cards ~lines 51-55, max reviews ~lines 84-88). Just(value) re-emits on every body re-render, and clearing the text triggers a re-render, so the refill fires the instant the field goes empty — the field can never stay empty long enough to type a replacement digit. The fix is to allow the field to be empty WHILE it is being edited: only re-sync the stored value into the text when that field is NOT the focused field. The commit-on-submit / commit-on-focus-loss logic already exists and clamps/defaults an empty or invalid entry, so nothing else needs to change.

They already handle an empty/invalid entry by falling back to the PREVIOUS committed value — preserve that behaviour; do not change the clamp ranges or the fallback.
