# T038 — Trace session end-to-end: Home → setup → session → summary

## Overview

Wire the whole Trace loop together — deck card → DeckSetupSheet → full-screen SessionView →
SummaryView → back to Home — with mid-session close preserving grades, proven by a unit test and a
driven visual run.

## Do

- **Wiring** (docs/02-product-spec.md §1, docs/05-architecture.md §5; app-level glue may go in
  `Sources/App/` — it is in scope):
  - `Sources/Features/Home/HomeView.swift`: replace the no-op `onStudy` — tapping a deck card /
    Study presents `DeckSetupSheet` (`.sheet`; form-sheet sizing on iPad) fed with the deck's
    jpTitle, name, and live due count.
  - "Start writing" → dismiss the sheet, present `SessionView` via `.fullScreenCover`, backed by a
    `SessionViewModel` built for (deck, Trace) on the app's `ModelContainer` with the injected clock
    (read `Sources/Features/Session/SessionViewModel.swift` for the initializer).
  - `SessionView`'s `onClose` → just dismiss the cover (each grade was persisted when given —
    nothing else to save; docs/02 §1.3).
  - When the view model reports finished, show `SummaryView`
    (`Sources/Features/Summary/SummaryView.swift`) inside the cover, fed the view model's summary
    numbers + the current streak. `back-to-decks` dismisses to Home; `study-another` dismisses AND
    re-presents the deck-setup sheet for the same deck ("returns to Home with the setup sheet
    ready" — docs/02 §1.4).
- **Test** `Tests/UnitTests/SessionFlowTests.swift` (XCTest, `@MainActor`, in-memory
  `ModelContainer`, fixed injected clock — never the real store):
  1. Full session: seed one deck + ≥4 notes with `new` schedules; build the view model for
     (deck, Trace); loop `showAnswer()` → `grade(.good)` until finished. Assert: finished is true;
     every `CardSchedule` has left `new` (state/dueAt updated) and the changes are readable from a
     fresh fetch on the same container; the `DailyStats` row for the clock's 4 AM-adjusted day has
     `cardsWritten` == the number of grades given.
  2. Mid-session close preserves grades: fresh seed; grade exactly one card; discard the view model
     with NO extra teardown call. Assert that card's schedule advanced and persisted while all
     other schedules are still untouched `new`.
- **Visual check**: run `./build_run.sh Kakitori-Sim -seedDemoData YES` and drive the flow — with
  xcui if available (tap `deck-row-hiragana` → `mode-trace` → `start-writing` → `show-answer` →
  `grade-good` …), otherwise screenshot each stage — through Home → sheet → session → summary →
  back to Home.

## Done when

- Both `SessionFlowTests` cases pass against the in-memory container.
- The driven run shows: setup sheet over Home; full-screen Trace session (top bar, prompt pane,
  guide boxes, Show answer → grade buttons); summary (済, お疲れさま！); and "Back to decks"
  landing on Home with the stats row ticked up.
