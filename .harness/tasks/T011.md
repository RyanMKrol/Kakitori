# T011 — Guide box row component (genkō yōshi boxes)

## Overview

SwiftUI component that lays out the square practice boxes — hairline border, dashed centre
cross-lines — from TargetSegmenter output, wrapping long phrases into rows and typesetting
punctuation in-line without a box (docs/06-ui-design.md §2.3).

## Do

- Create `Sources/Features/Session/Canvas/GuideBox.swift`: `struct GuideBox: View` — a square
  (1:1 aspect) box drawing a hairline border (0.5pt) PLUS a dashed centre cross: one horizontal
  and one vertical line through the box centre, `StrokeStyle(lineWidth: 0.5, dash: [4, 3])`.
  Border and cross both use `boxLine` = 8% ink; ink = `#1A1A1A` (docs/06 §1 palette). Use the
  KakitoriTheme tokens in `Sources/Support/` if tokens with these meanings exist; otherwise
  define `fileprivate` Color constants with exactly those values.
- Create `Sources/Features/Session/Canvas/GuideBoxRow.swift`: `struct GuideBoxRow: View`,
  `init(units: [SegmentedUnit], maxBoxesPerRow: Int = 6)`. `SegmentedUnit` is defined in
  `Sources/Importer/TargetSegmenter.swift` (consume it — do NOT edit that file):
  `.box(String)` renders a `GuideBox`; `.inline(String)` renders as plain
  `Text` in `Font.custom("Hiragino Mincho ProN", size:)`, ink colour, vertically centred on the
  box midline, with NO box around it.
- Wrapping (docs/06 §2.3): a row holds ≤ `maxBoxesPerRow` boxes (in-line punctuation does NOT
  count toward the limit). Callers pass 6 on regular width, 4 on compact — per docs/06 §2.3
  rows are "≤6 boxes" on iPad and "≤4 (vs ≤6 on iPad), shrinking box size to fit before
  wrapping" on iPhone. Box side = `min(160, availableWidth / CGFloat(boxesInRow))` (docs/06
  §2.3 "sized min(160pt, available/unitCount)") — measure available width with GeometryReader
  so boxes SHRINK to fit the row before a new row starts. Rows are centred.
- Accessibility identifiers: each box gets `"guide-box-N"` (N = 0-based box index across the
  whole sequence); the container gets `"guide-box-row"`.
- Add a `#Preview` (previews are DEBUG-only by nature) with two variants using
  `TargetSegmenter.segment(_:)`: (1) `"おはようございます。"` at `maxBoxesPerRow: 6`, on a paper
  `#FAF8F4` background; (2) the same units at `maxBoxesPerRow: 4` in a narrow (~390pt) frame.
- Swift 6 strict concurrency; no force-unwraps in Sources.

## Done when

- The loop fires `./build_run.sh` (screenshots the launched app into `screenshots/latest.png`).
  This component is not yet wired into an app screen (the session canvas pane task does that), so
  if the launched app cannot show it, render the `#Preview` instead and judge that image;
  record in the worklog which capture you judged.
- The captured image must show: square boxes with a visible hairline border AND dashed centre
  cross-lines (both faint, 8% ink — clearly lighter than text); `"おはようございます。"` as 9 boxes
  wrapping into a row of 6 then a row of 3, with `。` typeset after the last box WITHOUT a box;
  the compact variant wrapping at 4 boxes per row with visibly smaller boxes.
