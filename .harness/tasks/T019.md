# T019 — Media copy: manifest-renamed files into Application Support layout

## Overview

Copy an `.apkg`'s numbered media payloads into the app's per-deck media directory, renamed per
the `media` JSON manifest, matching the storage layout of docs/04-content-and-data.md §5
(`Application Support/Kakitori/Media/<deckUUID>/*.mp3`; re-import overwrites by name).

## Do

Create `Sources/Importer/MediaStore.swift`:

- `struct MediaStore: Sendable { let baseURL: URL; init(baseURL: URL) }` — the base is
  **injectable**: tests pass a temp directory; production passes
  `MediaStore.defaultBaseURL`, a `static var` computed as the user's Application Support
  directory + `"Kakitori"` (per docs/04 §5:
  `Application Support/Kakitori/` contains `Kakitori.store` and `Media/`).
- `func mediaDirectory(for deckID: UUID) -> URL` — `baseURL/Media/<deckID.uuidString>/`.
- `enum MediaStoreError: Error, Equatable` with case `badManifest`.
- `@discardableResult func copyMedia(manifestURL: URL, payloadDirectory: URL, deckID: UUID) throws -> [String]`:
  1. Parse the manifest — a JSON object mapping numbered payload names to real filenames
     (docs/04 §1: `{ "0": "bc4e77e6407e004b.mp3", ... }`). Unreadable/wrong-shape JSON →
     `.badManifest`.
  2. Create `mediaDirectory(for: deckID)` with intermediate directories.
  3. For each manifest entry, copy `payloadDirectory/<key>` →
     `mediaDirectory/<value>`. **Overwrite by name**: if the destination exists, remove it
     first, then copy (docs/04 §5 — re-import overwrites by name; never error on a re-copy,
     never produce duplicates).
  4. A manifest entry whose payload file is missing on disk is skipped, not fatal.
  5. Return the real filenames actually copied.

Pure Foundation/FileManager — no SwiftData, no force-unwraps, no `try!`. NEVER touch the real
Application Support directory from tests.

## Done when

`Tests/UnitTests/MediaStoreTests.swift` (XCTest) passes. Setup: extract `Fixture.kanaDeckURL`
(helper in `Tests/UnitTests/FixtureLocator.swift`, archive opened with `ZipArchive` from T016)
into a temp directory; create `MediaStore(baseURL:)` on a second temp directory
(`FileManager.default.temporaryDirectory` + UUID); call `copyMedia` with the extracted `media`
manifest, the extraction directory as `payloadDirectory`, and a fresh `UUID`. Assert:

- The returned filename list is non-empty and at least one name ends in `".mp3"` (the fixture
  ships real MP3 media).
- For one returned name, a file exists at `mediaDirectory(for: deckID)/<name>` with size > 0.
- Running the identical `copyMedia` call a second time succeeds, and the file count inside
  `mediaDirectory(for: deckID)` is unchanged (overwrite, no duplication).
- A manifest URL pointing at a non-JSON file throws `MediaStoreError.badManifest`.
