# T006 — SwiftData model: CardSchedule + CardState

## Overview

Create the per-card SRS state model `CardSchedule` (1:1 with `Note`) and its `CardState` enum, per
docs/05-architecture.md §3, with the initial values from docs/03-srs-algorithm.md §2.

## Do

1. Create `Sources/Models/CardSchedule.swift` (`import SwiftData`, `import Foundation`):
   - `enum CardState: String, Codable, Sendable { case new, learning, review, relearning }`
     (docs/05 §3: "new | learning | review | relearning").
   - `@Model final class CardSchedule` with exactly these stored fields (docs/05-architecture.md §3):
     - `var state: CardState`
     - `var stepIndex: Int`
     - `var easeFactor: Double`
     - `var intervalDays: Double`
     - `var dueAt: Date?`
     - `var lapses: Int`
     - `@Relationship(inverse: \Note.schedule) var note: Note?`
   - `init` defaults matching the "Initial" column of docs/03-srs-algorithm.md §2:
     `state: CardState = .new`, `stepIndex: Int = 0`, `easeFactor: Double = 2.5`,
     `intervalDays: Double = 0`, `dueAt: Date? = nil`, `lapses: Int = 0`.
2. Edit `Sources/Models/Note.swift` (the existing `Note` model): add the field docs/05 §3 lists there
   — `var schedule: CardSchedule?` (plain `var`; the inverse is declared on the CardSchedule side) —
   defaulting to `nil` in Note's `init`.

No force-unwraps or `try!`; no `Date()` reads — `dueAt` is only ever set by callers who pass a time in.

## Done when

- `Tests/UnitTests/CardScheduleModelTests.swift` (XCTest) uses an in-memory container
  (`ModelConfiguration(isStoredInMemoryOnly: true)` over `Deck.self, Section.self, Note.self,
  CardSchedule.self`) — never the real store — and asserts:
  - `CardSchedule()` has the documented initial state: `.new`, `stepIndex == 0`,
    `easeFactor == 2.5`, `intervalDays == 0`, `dueAt == nil`, `lapses == 0`.
  - Insert a `Note` and a `CardSchedule`, link `note.schedule = schedule`, save, refetch: the link
    round-trips in both directions (`note.schedule` set and `schedule.note === note`).
  - A schedule with `state: .review, intervalDays: 12.0, dueAt: fixedDate, lapses: 2` fetches back
    with all values unchanged (enum + Date? round-trip through SwiftData storage).
