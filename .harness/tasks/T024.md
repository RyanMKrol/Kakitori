# T024 — SM2Scheduler: review transitions, clamps, seeded fuzz

## Overview

Extend `Sources/Scheduler/SM2Scheduler.swift` (from T023) to grade `.review` snapshots per
docs/03-srs-algorithm.md §4 "Review cards", with interval clamps to [1, 365] days and seeded ±5%
fuzz on intervals ≥ 3 days.

## Do

Same module rules as T023: Foundation only, no `Date()`/`.now`, RNG only via the injected
`rng: inout some RandomNumberGenerator` parameter (system RNG is a build-failing violation).

1. Add `let fuzzEnabled: Bool` to `SM2Scheduler` via `init(fuzzEnabled: Bool = true)` (previews in
   T025 need a fuzz-free path; production uses the default).
2. Replace the T023 `.review` placeholder in `apply` with (let `I = card.intervalDays`,
   `EF = card.easeFactor`; deltas/multipliers from `SRSConstants`):
   - **again** → `lapses += 1`; `easeFactor = max(1.3, EF − 0.20)`; `state = .relearning`;
     `stepIndex = 0`; `dueAt = now + 600` (10 min relearn step); `intervalDays` stays `I`
     unchanged — it is the pre-lapse interval T023's re-graduation multiplies by 0.5. Never fuzzed
     (sub-day).
   - **hard** → `easeFactor = max(1.3, EF − 0.15)`; `newI = (I × 1.2).rounded()`.
   - **good** → `easeFactor` unchanged; `newI = (I × EF).rounded()`.
   - **easy** → `easeFactor = EF + 0.15` first (no upper clamp), then `newI = (I × newEF × 1.3)
     .rounded()` — the interval uses the **updated** ease (docs/03 lists the EF update before the
     interval formula). Example: I 10, EF 2.5 → EF 2.65 → `(10 × 2.65 × 1.3).rounded()` = 34.
   - For hard/good/easy: clamp `newI = min(365, max(1, newI))`; `intervalDays = newI` (the clamped,
     **unfuzzed** value is what is stored); `state = .review`; `stepIndex = 0`; then
     `dueAt = now + newI × 86_400 × (1 + f)` where `f = 0` unless fuzz applies.
3. **Fuzz** (docs/03 §4: "fuzz of ±5% on intervals ≥ 3 days"): applies ONLY when grading a
   `.review` snapshot AND `fuzzEnabled` AND `newI >= 3` — never to learning/relearning steps or to
   the T023 graduation intervals (1d/4d), and it perturbs `dueAt` only, not `intervalDays`.
   `f = Double.random(in: -0.05...0.05, using: &rng)`.
