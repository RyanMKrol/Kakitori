## Overview
FEATURE (idea #8 Part B): add a Free Study session path COMPLETELY DECOUPLED from the SRS. It does NOT read due dates, does NOT write CardSchedule (bypasses SM2Scheduler entirely), does NOT write DailyStats (bypasses StatsRecorder entirely — no recordGrade, no recordCompletion, no recordStudySeconds, no ensureDailyStats side effects that consume the daily quota), and does NOT count toward or consume the deck's dailyTarget/completedToday/remainingToday. It randomly, endlessly serves cards the user has ALREADY seen — CardSchedule.state != .new (i.e. .learning, .review, or .relearning) — shuffled, repeating indefinitely so the user can keep practising handwriting by advancing. This unit builds the LOGIC + the session-driving plumbing; the UI ENTRY POINT (which button on which screen launches it) is unit sbf-c, and the grading-control choice (four grade buttons vs a single Next) is an OPEN question relayed to the owner — build the model to support advancing regardless, and default grading to a pure no-op self-grade until the answer lands. NOTE: exact grading-UI wiring (sbf-c) may adjust once the owner answers; this unit must at minimum give SessionViewModel a Free Study construction/mode and a card source.

## Do
1. Free-study card source: add a pure source (a small type/function, e.g. `FreeStudySource` under Sources/Scheduler/, mirroring SessionQueue's Foundation-only, no-SwiftData, injected-RNG style) that, given the deck's cards + their CardState, returns ONLY previously-seen cards (state == .learning || .review || .relearning; EXCLUDES .new), shuffled deterministically from an injected seed/RNG (reuse the SplitMix64 RNG SessionViewModel already holds, mirroring SessionQueue's seeding), and produces an ENDLESS sequence — when the shuffled pool is exhausted it reshuffles and continues, never returning nil while the pool is non-empty. If the pool is empty (deck has no previously-seen cards), it yields nothing (the caller shows a caught-up/empty state — do not crash).
2. Free-study mode in SessionViewModel (Sources/Features/Session/SessionViewModel.swift): add a way to construct the VM in Free Study mode (e.g. an `isFreeStudy` flag on init, or a dedicated init). In Free Study: build the current card from the FreeStudySource instead of SessionQueue; do NOT call StatsRecorder.ensureDailyStats/recordGrade/recordCompletion/recordStudySeconds; do NOT read or set dayTarget/dayCompleted (leave them 0 / inert); advancing to the next card pulls the next item from the endless source. grade(_:) in Free Study must be a pure no-op with respect to persistence: it must NOT call scheduler.apply, must NOT mutate the note's CardSchedule, and must NOT call any StatsRecorder method — it only advances to the next free-study card (optionally counting an in-memory cardsWritten for display, but persisting nothing). The endless source means Free Study never auto-finishes; ending is user-initiated (close). Keep the injected AppClock; never read Date()/.now.
3. Guard rails: make it structurally hard to leak SRS side effects from Free Study — e.g. branch early in grade() on the free-study flag and return after advancing, before any scheduler/StatsRecorder call. Do not change the normal (SRS) grade path's behaviour.
4. Tests (Tests/UnitTests/FreeStudyTests.swift, plus additions to SessionViewModelTests.swift; in-memory ModelContainer, ScriptedClock via AppClock(now:)): assert (a) the free-study source EXCLUDES .new cards and INCLUDES .learning/.review/.relearning cards; (b) it produces an endless shuffled sequence — pulling more items than the pool size keeps returning cards (reshuffle-and-continue), and the sequence is a shuffle of the seen-pool (every returned id is in the seen pool); (c) grading in Free Study writes NO CardSchedule change (capture a seen card's state/easeFactor/intervalDays/dueAt/stepIndex/lapses before and after a grade and assert all unchanged) and NO DailyStats change (assert no DailyStats row is created/mutated and the deck's completedToday/dailyTarget are untouched); (d) an empty seen-pool deck yields no free-study card (caller can detect and show caught-up). DECISION (grading UI, from owner): Free Study advances via a SINGLE 'Next / Keep going' control (there are NO grade buttons in Free Study), so expose an advance path (e.g. advanceFreeStudy(), or reuse grade() as a pure no-op that only advances) that pulls the next card from the endless source and persists nothing — the four-grade path is not used in Free Study. The persistence no-op requirement above is unchanged (no scheduler.apply, no StatsRecorder).
