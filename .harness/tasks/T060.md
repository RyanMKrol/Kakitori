## Overview
The Home TODAY'S PRACTICE banner (Sources/Features/Home/TodayBannerView.swift, calculateDueCards ~38-58) and the deck-card new/learning/due chips + isAllCaughtUp (Sources/Features/Home/DeckCardView.swift ~213-263) each walk note.schedule directly and count the FULL uncapped backlog, so the banner advertises e.g. '215 characters' (91 new + 124) and a deck never reads 'all caught up' when only the DAILY cap is hit. Meanwhile the caps (newCardsPerDay=10, maxReviewsPerDay=100 in Sources/Support/AppSettings.swift) are applied ONLY inside SessionQueue.build. This unit introduces ONE shared pure helper so every Home surface reflects the same 'today's allotment' a session would actually serve.

## Do
Add a single pure, Foundation-only helper (e.g. DailyAllowance in Sources/Scheduler/) that, given decks (or a deck) + today's DailyStats (newIntroducedToday / reviewsDoneToday) + AppSettings caps + AppClock, returns today's remaining allotment counts: remaining new-allowance = max(0, newCardsPerDay - newIntroducedToday); due reviews (state == .review, dueAt <= end-of-today) capped by max(0, maxReviewsPerDay - reviewsDoneToday); due learning/relearning (dueAt <= now, uncapped) — mirroring SessionQueue.build's cap logic so counts match what a session serves. Expose both per-deck and aggregate (banner) totals plus a scripts-with-work count. Wire TodayBannerView to show today's allotment total/script-count (and its 'All caught up' branch when the allotment is 0), and wire DeckCardView's new/learning/due chips and isAllCaughtUp to the same helper so a deck reads 'all caught up' when only the daily cap is hit. Where practical, have SessionQueue.build consume the same allowance computation to avoid two divergent cap implementations (do not change the queue's ordering/interleave). Owner-confirmed scope: the deck chips AND isAllCaughtUp move together with the banner (all three surfaces reflect the allotment); a 'study beyond today's cap' bypass affordance is OUT of scope for this unit (a separate follow-up), so do not build a bypass here.

## Done when
