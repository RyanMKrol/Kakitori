# T030 — Session screen scaffold: top bar, progress, queue chips (iPad)

## Overview

Create the full-screen practice-session shell — paper background, top bar (close, deck + mode,
progress, live queue chips) and empty hosting slots for the prompt pane, canvas pane, and action
row — bound to the existing `SessionViewModel`. iPad regular-width layout only (iPhone compact is a later task).

## Do

- Create `Sources/Features/Session/SessionView.swift`. It takes the existing `@Observable @MainActor`
  `SessionViewModel` (`Sources/Features/Session/SessionViewModel.swift` — read it for the exact API:
  queue chip counts, progress, current card, revealed flag) plus an `onClose: () -> Void` closure.
- Full-screen `KakitoriTheme.paper` background (`Sources/Support/Theme.swift`), `.ignoresSafeArea()`.
- **Top bar** (hairline bottom border, docs/02-product-spec.md §1.3):
  - Close button `✕` — circular, ≥44pt, accessibility identifier `"session-close"`. Tapping calls
    `onClose` and nothing else: `SessionViewModel` persists each grade the moment it is given, so
    dismissal does NO rollback and no extra save — "Closing mid-session (✕) preserves all grades
    already given; ungraded cards stay due" (docs/02 §1.3).
  - Deck name (bold, small) with a mode line under it, e.g. `書 · Trace` (reference prototype top bar).
  - Progress: thin capsule bar (accent fill on `boxLine` track) that animates per grade, captioned
    exactly `"\(done) done · \(left) left"` — e.g. "0 done · 10 left" (docs/02 §1.3, docs/06-ui-design.md §2.3).
  - Three live chips from the view model's queue counts, ticking down as cards grade: `"N new"` /
    `"N learn"` / `"N due"` styled with the `KakitoriTheme` chip tokens (neutral / red-tint / ink fill).
- **Below the bar**: `HStack(spacing: 0)` — left slot `promptPane` at ~40% of the width (docs/06 §2.3),
  right a `VStack` of `canvasPane` (fills remaining height) above `actionRow`. Make the three slots
  internal computed views showing faint placeholder text ("Prompt" / "Canvas" / "Actions") — later
  tasks replace them with `Sources/Features/Session/PromptPaneView.swift`,
  `Sources/Features/Session/CanvasPaneView.swift`, and `Sources/Features/Session/ActionRowView.swift`.
- SessionView is NOT yet reachable from Home (that navigation is a later task whose scope includes
  `Sources/App/`), so give the loop something to look at: add
  `Tests/UnitTests/SessionScaffoldRenderTests.swift` (XCTest, `@MainActor`) that builds a
  `SessionViewModel` against an in-memory `ModelContainer` seeded with one deck + 3 hiragana notes
  and a fixed injected clock, wraps `SessionView` in `.frame(width: 1194, height: 834)`, renders it
  with `ImageRenderer` (scale 2), and writes the PNG to `<repoRoot>/screenshots/T030-session.png`
  (derive the repo root from `#filePath`). Assert the rendered image is non-nil.
