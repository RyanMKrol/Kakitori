# T014 — Canvas feel review on real iPad + Pencil hardware (🔒 needs-human)

## Overview

The docs/07-roadmap.md M1 risk gate, done by hand on hardware: "tracing あ over a faded guide
feels good in hand — if the writing feel is wrong, nothing else matters." A simulator screenshot
cannot judge ink feel, so this is a human runbook, not a buildable task. NOTE: nothing downstream
is blocked on this task — it is a parked quality gate you can run whenever the hardware is handy.

## Do

1. Pull latest `main` (T011–T013 must be integrated). Run `xcodegen generate` and open
   `Kakitori.xcodeproj` in Xcode.
2. Select your physical iPad as the run destination. `project.yml` sets
   `CODE_SIGNING_ALLOWED: NO` for CI/simulator builds — enable automatic signing with your
   personal team in Xcode's Signing & Capabilities for the device run, and do NOT commit any
   signing changes.
3. Get the trace canvas on screen: if the session screen exists by now, run the app and start a
   Trace session; otherwise use Xcode Previews on the device — open
   `Sources/Features/Session/Canvas/TraceGuideLayer.swift` and run its `#Preview` (the ZStack of
   faded guides under a live `WritingCanvas`) on the iPad.
4. With the Apple Pencil, trace **あ**, **き**, and **ふ** over the faded guides (き/ふ are the
   docs/07 open-question glyphs with the trickiest curves). Also write a few strokes with a
   finger — `.anyInput` means both must work. Judge:
   - **Width** — does the stroke sit well over the guide, brush-adjacent rather than ballpoint?
   - **Latency** — does ink keep up with the Pencil tip?
   - **Brush character** — does the pen ink feel acceptable for kana practice (docs/05 §8 risk:
     "ink defaults may feel un-brush-like")?
   - Pencil **double-tap** triggers Undo; **Clear** empties strokes but never the faded guides.
5. If the width is off, edit the single constant `WritingCanvas.penWidth` in
   `Sources/Features/Session/Canvas/WritingCanvas.swift`, rebuild, re-trace, repeat until it
   feels right. Commit and push the tuned constant (run the local DoD first).
6. Also record a verdict on the docs/07 open question: are Hiragino Mincho ProN guide shapes
   close enough to handwritten forms for き / さ / ふ, or does M1 sign-off want a Kyōkashō font?
7. Write the verdict in `.harness/worklog/T014.md`: pass/fail on the M1 gate, final `penWidth`
   value, latency/brush notes, and the guide-font opinion.
8. Mark the gate done: `.harness/scripts/mark-done.sh T014`.

## Done when

- You have physically traced あ / き / ふ over faded guides on a real iPad with an Apple Pencil
  and judged the feel good enough to build the rest of the app on (the M1 gate).
- Any `penWidth` change is committed and pushed; the verdict (including the guide-font opinion)
  is recorded in `.harness/worklog/T014.md`.
- `.harness/scripts/mark-done.sh T014` has been run.
