## Overview
When today's daily allowance is already exhausted, starting another session on the same deck currently builds an empty SessionQueue and renders a broken card: no target, '0 done / 0 left', dead Again/Hard/Good/Easy buttons. Root cause: SessionViewModel.init (Sources/Features/Session/SessionViewModel.swift ~112-127) only sets currentEntry/currentNote inside 'if let firstEntry = queue.next(...)' with no else branch, so on an empty queue phase stays at its default .prompt (line ~22) and currentNote stays nil; showAnswer() (~130) has no nil guard and flips to .revealed, but grade() (~140-141) guards on currentNote and no-ops. Correct behaviour: an empty queue at session start must land directly in the finished/summary state (the existing finish() path already does this correctly, and HomeView.sessionCoverContent routes phase==.finished + summary to SummaryView), and the reveal/grade methods must be nil-safe so they can never present dead buttons.

## Do
In SessionViewModel.init, after building the queue, when queue.next(now:) returns nil at start, initialize the session into the finished state exactly as finish() does: currentEntry/currentNote nil, phase == .finished, and a valid SessionSummary (zero grade counts, cardsWritten 0, seconds 0) so HomeView.sessionCoverContent routes to SummaryView (an 'all done for today' summary) rather than an empty card. Prefer reusing/extracting the finish() logic (e.g. call a shared finish path with now) over duplicating it; the empty-start finish must NOT record spurious study seconds (or must record 0). Harden showAnswer() to guard that currentNote/currentEntry are present before transitioning to .revealed, so a nil-current session can never show gradeable-but-dead buttons. Leave grade()'s existing guard intact. Do NOT change the daily-cap math in SessionQueue.build. (Owner-confirmed: the empty start routes to the Summary 'done for today' screen, not a non-startable deck.)

## Done when
An XCTest (against an in-memory SwiftData ModelContainer and injected AppClock) asserts: (1) a SessionViewModel built from a deck whose daily allowance is already exhausted (empty queue at start) initializes with phase == .finished, currentNote == nil, and a non-nil SessionSummary; (2) calling showAnswer() then grade(_:) on that empty-start session is a safe no-op (no crash, phase stays .finished, no stats mutation); (3) a normal non-empty session is unaffected (still starts at .prompt with a currentNote and grades through to a summary). The empty-start path records 0 (not spurious) study seconds. Manually verified on BOTH iPad (Kakitori-Sim) and iPhone (Kakitori-Sim-Phone): after completing a deck's daily allotment, re-starting that deck shows the summary/'done for today' screen (not an empty broken card), with no regression to a fresh session.
