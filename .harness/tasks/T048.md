# T048 — iPhone compact session layout (prompt band, pinned actions)

## Overview

Add the compact (iPhone/portrait) variant of the practice session screen per docs/06-ui-design.md
§2.3 "iPhone / portrait": prompt band above the canvas, ≤4 guide boxes per row, in-place answer
reveal, full-width actions pinned at the bottom, bare-number top-bar chips.

## Do

1. In the session scaffold view under `Sources/Features/Session/` (the top-level screen that
   composes top bar + prompt pane + canvas pane + action row), read
   `@Environment(\.horizontalSizeClass)`. When it is `.compact`, switch to a vertical layout;
   the existing side-by-side regular layout must be pixel-unchanged. Branch ONLY on size class —
   never `UIDevice`/idiom checks.
2. Compact prompt band (docs/06 §2.3: "caption + headline + one sub-line"): accent small-caps
   mode caption (e.g. "TRACE MODE"), the mode headline, and exactly ONE sub-line. Reuse the
   existing prompt-pane content, restyled — do not fork the copy.
3. Answer reveal "replaces this band in place (target, reading, English, play-audio) — no bottom
   sheet" (docs/06 §2.3): the same band region swaps to the ANSWER block; no sheet, no push.
4. Guide boxes: in compact, pass the `GuideBoxRow` row-limit parameter as 4 (regular stays 6),
   "shrinking box size to fit before wrapping" (docs/06 §2.3).
5. Top bar in compact: the count chips "drop their word labels and show bare numbers"
   (docs/06 §2.3) — e.g. `3 3 4`, not `3 new · 3 learning · 4 due`.
6. Action area pinned full-width at the bottom, ≥44pt tall: "Show answer" before reveal; the four
   grade buttons (Again/Hard/Good/Easy with interval previews) after.
7. Keep every existing accessibility identifier (`show-answer`, `grade-again/hard/good/easy`,
   `session-close`, `canvas-undo`, `canvas-clear`) present in BOTH layout branches.
