# T018 — Note field mapping: name-based fields, HTML strip, sound refs

## Overview

Turn an `AnkiNote`'s positional field values into named, cleaned Kakitori values — matched **by
field name, not index**, so field reordering in anki-builder is safe (docs/04-content-and-data.md
§1.1) — with defensive HTML stripping and `[sound:...]` extraction (docs/04 §2 step 5).

## Do

Create `Sources/Importer/NoteFieldMapper.swift`:

- `struct MappedNote: Sendable, Equatable` with: `ankiGUID: String`, `ankiDeckID: Int64?`,
  `target: String`, and optionals `pronunciation`, `english`, `category`, `hint`, `image`,
  `audioFilename` (all `String?`).
- `enum NoteFieldMapperError: Error, Equatable` with case `emptyTarget(noteID: Int64)`.
- `enum NoteFieldMapper { static func map(_ note: AnkiNote, using model: AnkiModel) throws -> MappedNote }`.

Mapping rules:

1. Pair `model.fieldNames` with `note.fields` positionally (`zip` — tolerate count mismatches by
   ignoring the surplus side), producing a `[String: String]` lookup **by name**. The AnkiBuilder
   names are `Target`, `Pronunciation`, `English`, `Category`, `Hint`, `Image`, `Audio`
   (docs/04 §1.1). Unknown field names are ignored; a missing or (post-cleaning) empty optional
   field becomes `nil` — notes without `Audio` etc. must degrade gracefully.
2. **HTML stripping** (applied to every field): replace `<br>`, `<br/>`, `<br />`, `</div>`,
   `</p>` with a single space; remove every remaining tag matching the regex `<[^>]+>`; decode
   the entities `&amp;`→`&`, `&lt;`→`<`, `&gt;`→`>`, `&quot;`→`"`, `&#39;`→`'`, `&nbsp;`→space;
   finally trim whitespace/newlines. Strip defensively even though anki-builder output is clean
   today (docs/05 §8).
3. **Audio**: from the `Audio` field, extract the first match of the regex
   `\[sound:([^\]]+)\]` — the capture group (e.g. `bc4e77e6407e004b.mp3`) becomes
   `audioFilename`; all `[sound:...]` tags are removed from the text and the leftover Audio text
   is discarded. No sound tag → `audioFilename = nil`.
4. `target` = cleaned `Target` value; if it is empty after cleaning, throw
   `.emptyTarget(noteID: note.id)`.
5. Copy `note.guid` → `ankiGUID` and `note.deckID` → `ankiDeckID`.

Pure value-level logic — no SwiftData, no file I/O, no force-unwraps (build the two regexes once;
an invalid-pattern failure may `fatalError` only via a non-optional `try`-free construction such
as `NSRegularExpression` wrapped in a preconditioned static let — prefer literal
`Regex`/`firstMatch(of:)` if simpler).

## Done when

`Tests/UnitTests/NoteFieldMapperTests.swift` (XCTest) passes, using hand-built inline `AnkiModel`
/ `AnkiNote` values (no fixture needed):

- Canonical order: model fields `[Target, Pronunciation, English, Category, Hint, Image, Audio]`
  with values `["おはよう", "ohayō", "Good morning.", "Greetings", "", "", "[sound:a1.mp3]"]`
  maps to `target == "おはよう"`, `pronunciation == "ohayō"`, `english == "Good morning."`,
  `category == "Greetings"`, `hint == nil`, `image == nil`, `audioFilename == "a1.mp3"`.
- Reordered model (e.g. `[Audio, English, Target]` with matching values) produces the SAME named
  values — proving order independence.
- `"<b>ね</b>&amp;<br>next"` cleans to `"ね& next"` (tags gone, entity decoded, `<br>` → space).
- A note whose `Audio` field is empty maps to `audioFilename == nil`.
- A note whose `Target` cleans to empty throws `NoteFieldMapperError.emptyTarget`.
