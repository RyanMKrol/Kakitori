# T009 — DEBUG demo seed behind -seedDemoData launch argument

## Overview

Give DEBUG builds a deterministic, idempotent demo dataset — 3 decks, mixed-state cards, a few days
of stats — behind the `-seedDemoData YES` launch argument, so simulator screenshots show a populated
Home instead of the empty state.

## Do

1. Create `Sources/App/DemoSeed.swift`, entire contents wrapped in `#if DEBUG`:
   - `enum DemoSeed` with `@MainActor static func seedIfNeeded(context: ModelContext, clock: AppClock)`:
     - Gate: return immediately unless `UserDefaults.standard.bool(forKey: "seedDemoData")` is true
       (launching with arguments `-seedDemoData YES` sets this automatically via the argument domain).
     - **Idempotence**: fetch `Deck` where `sourceDeckName == "Demo::Hiragana"`; if found, return —
       re-launching with the flag must not duplicate anything.
     - Seed 3 decks (all `importedAt: clock.now()`), each with one `Section` (orderIndex 0) holding
       its notes, every note paired with a `CardSchedule` (`note.schedule = ...`):
       - `jpTitle: "ひらがな"`, `name: "Hiragana"`, `sourceDeckName: "Demo::Hiragana"` — 4 notes,
         e.g. `target: "あ"` (`script: .hiragana`, `units: ["あ"]`, pronunciation "a"), similarly い/う/え.
       - `jpTitle: "カタカナ"`, `name: "Katakana"`, `sourceDeckName: "Demo::Katakana"` — 3 notes
         (ア/イ/ウ, `script: .katakana`).
       - `jpTitle: "漢字"`, `name: "Kanji"`, `sourceDeckName: "Demo::Kanji"` — 3 notes (e.g.
         日 "sun", 月 "moon", 語 "language"; `script: .kanji`, set `english`).
     - **Mixed schedule states** across those 10 cards, with `now = clock.now()`: some default
       (`state: .new`, `dueAt: nil`); some `.learning` (`stepIndex: 1`, `dueAt` = now + 600 s); some
       `.review` with varied `dueAt` (now − 1 day = due, now + 3 days = not due) and
       `intervalDays` 4–12, `easeFactor` 2.5.
     - **DailyStats**: 3 consecutive adjusted days ending today — compute day strings by calling
       `clock.adjustedDay(for:)` on `now`, now − 86400, now − 172800 (never format dates by hand) —
       e.g. cardsWritten 12/8/15, secondsStudied 360/240/450, nonzero newIntroduced/reviewsDone.
     - Never call `Date()` — all times derive from the injected `clock` (AppClock in
       `Sources/Support/AppClock.swift`).
2. Edit `Sources/App/KakitoriApp.swift`: inside `#if DEBUG`, call
   `DemoSeed.seedIfNeeded(context: container.mainContext, clock: .system)` once at startup (e.g. in
   `init` after the container is built).

## Done when

- `Tests/UnitTests/DemoSeedTests.swift` (XCTest) exists. Because the UserDefaults gate is
  launch-arg driven, expose the seeding body as
  `@MainActor static func seed(context: ModelContext, clock: AppClock)` (called by `seedIfNeeded`
  after its gate) and test THAT with an in-memory container
  (`ModelConfiguration(isStoredInMemoryOnly: true)`, all five model types) — never the real store —
  asserting with a fixed clock:
  - After one `seed`: exactly 3 `Deck`s, 10 `Note`s each with a non-nil `schedule`, 3 `DailyStats`;
    at least one schedule in each of `.new`, `.learning`, `.review`.
  - After calling `seed` a second time: all counts unchanged (idempotent).
  - The 3 `DailyStats.day` values equal `clock.adjustedDay(for:)` of now / now − 1 day / now − 2 days.
- Visual verification: `./build_run.sh Kakitori-Sim -seedDemoData YES`, then check
  `screenshots/latest.png` — Home shows the deck list (ひらがな / カタカナ / 漢字), not the empty state;
  re-running the same command still shows exactly 3 decks.
