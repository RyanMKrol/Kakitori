# T053 — Empty states + import error polish

## Overview

Polish Home's empty and all-caught-up states and the import error/progress presentation per
docs/06-ui-design.md §2.1 and docs/04-content-and-data.md §2, giving every alert/empty state a
stable accessibility identifier.

## Do

Scope is `Sources/Features/` — the importer itself (`Sources/Importer/`) is OUT of scope; map its
existing error cases to user-facing copy in the Home feature layer.

1. Home empty state (no decks), `Sources/Features/Home/`: per docs/06 §2.1 — the 書 mark, the line
   "Import a deck to start writing" (character-for-character), and an Import button front and
   center that opens the Files picker; import stays additionally available behind the toolbar `+`.
   Give the empty-state container `.accessibilityIdentifier("home-empty-state")` and keep
   `import-button` on the button.
2. All-caught-up states (no copy exists in docs; the following invented strings are final):
   - A deck whose new + learning + due counts are all zero: replace its chips row with a quiet
     "All caught up" label; the Study button stays enabled (ahead-of-schedule study is allowed).
   - When the total due across all decks is zero, the today banner reads
     "All caught up. Nothing due right now." instead of "N characters to write · across M scripts".
3. Import error alert: one alert titled "Import failed", identifier "import-error-alert", with a
   message per failure reason from docs/04 §2 (steps 2 and 4 — invented strings below are final):
   - Unreadable archive / missing `collection.anki2`: "This file could not be read as an Anki deck."
   - No note model containing a `Target` field: "This deck is missing the Target field Kakitori needs."
   - Zero notes: "This deck has no cards to import."
4. Import progress: while an import runs, the import affordance shows a visible in-progress state
   (spinner/`ProgressView` and disabled button) until success or the error alert.

## Done when

- Empty-state screenshot on a clean install (uninstalling clears the app's store):
  ```
  UDID=$(./tools/loop_sim.sh)
  xcrun simctl uninstall "$UDID" com.ryankrol.kakitori 2>/dev/null || true
  ./build_run.sh Kakitori-Sim
  ```
  (no `-seedDemoData`) — `screenshots/latest.png` shows 書, "Import a deck to start writing", and
  the Import button centered; then `./build_run.sh Kakitori-Sim -seedDemoData YES` shows the
  normal populated Home again.
- A unit test in `Tests/UnitTests/` asserts the error-case → message mapping returns exactly the
  three strings above, and that the banner/deck-card presentation logic yields the "All caught up"
  variants when counts are zero.
