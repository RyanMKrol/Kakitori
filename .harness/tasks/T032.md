# T032 — Canvas pane: guide boxes + canvas + hint + undo/clear pills

## Overview

Assemble the session's right pane from the existing canvas components: hint line, Undo/Clear pills,
centred genkō yōshi guide boxes with the PencilKit canvas overlaid, trace guides in Trace mode,
auto-clear on advance, and Pencil double-tap → undo.

## Do

- Create `Sources/Features/Session/CanvasPaneView.swift` taking the `SessionViewModel`; replace the
  `canvasPane` placeholder in `Sources/Features/Session/SessionView.swift` with it.
- **Top row** (docs/06-ui-design.md §2.3: "hint line top-left, Undo/Clear pills top-right"):
  - Hint line, faint ink, Trace copy: `Trace each character in its box` (reference prototype's
    per-mode hints; the other modes bring their own copy in their own tasks).
  - Pills: `↶ Undo` (identifier `"canvas-undo"`) and `Clear` (identifier `"canvas-clear"`) — white
    fill, hairline border, rounded. Wire them to the existing `WritingCanvas` undo/clear API (read
    `Sources/Features/Session/Canvas/WritingCanvas.swift`; Undo removes the last stroke, Clear
    empties the drawing with no confirmation — docs/02-product-spec.md §3).
- **Centre**: the existing `GuideBoxRow` (`Sources/Features/Session/Canvas/`) laid out for the
  current card's `units` — box size `min(160pt, available/unitCount)`, wrapping to rows of ≤6 boxes
  (docs/06 §2.3; reuse whatever GuideBoxRow already handles, don't rebuild it). Overlay `WritingCanvas`
  across the box area; render the existing `TraceGuideLayer` UNDER the canvas, and only when the
  session mode is Trace, so Clear never removes guides (docs/02 §3).
- **Clear on advance**: `.onChange` of the view model's current-card identity → clear the drawing
  ("Grading a card immediately advances to the next card and clears the canvas" — docs/02 §1.3).
- **Pencil double-tap → Undo** (docs/06 §2.3): attach a `UIPencilInteraction` (inside the
  `WritingCanvas` representable if it lacks one) whose delegate triggers the same undo as the pill.
- Render check: `Tests/UnitTests/CanvasPaneRenderTests.swift`, same recipe as
  `Tests/UnitTests/SessionScaffoldRenderTests.swift` (in-memory container, fixed clock,
  `ImageRenderer` at 1194×834, `#filePath`-derived output). Write `screenshots/T032-canvas.png`.
  Note: the `PKCanvasView` layer renders blank/transparent in `ImageRenderer` — that's fine; the
  subject is the hint, pills, boxes, and trace glyphs.
