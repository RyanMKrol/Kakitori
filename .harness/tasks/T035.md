# T035 — Home stats row: streak, written today, minutes studied

## Overview

Add the three-stat row to Home — day streak, cards written today, minutes studied — computed live
from `DailyStats` and the existing streak function, on the 4 AM-adjusted day.

## Do

- Create `Sources/Features/Home/StatsRowView.swift` and place it in
  `Sources/Features/Home/HomeView.swift` above the deck area (replace any placeholder the shell has).
- Three stat cards (white fill, hairline border, radius 16–20pt), each a large numeral over a
  small-caps caption (docs/06-ui-design.md §2.1: "Small caps captions, large numerals";
  docs/02-product-spec.md §1.1):
  - `🔥` + streak number, caption `Day streak` — computed by the existing streak function over
    `DailyStats` (`Sources/Models/`), which already implements the "today or yesterday" grace rule
    (docs/02 §4.3). Streak numeral in `KakitoriTheme.accent`.
  - `cardsWritten` from today's `DailyStats` row, caption `Written today`.
  - `secondsStudied / 60` rendered like `12m` (trailing lowercase m), caption `Minutes studied`.
- "Today" is the 4 AM-adjusted day: use the adjusted-day helper on the injected clock in
  `Sources/Support/` (never a bare `Date()`), and fetch the `DailyStats` row whose `day` string
  matches. Missing row → show zeros, no crash.
- Read via the main-actor SwiftData context that `HomeView` already uses (`@Query` or a fetch on
  appear) so the numbers update after sessions.

## Done when

- `./build_run.sh Kakitori-Sim -seedDemoData YES` screenshot shows three stat cards across the top
  of Home with the seeded non-zero numbers and the captions "Day streak" (with 🔥),
  "Written today", and "Minutes studied" (value like "12m").
- `./build_run.sh` without the seed shows the same row with zeros — no crash, no missing row.
