# T001 — Project scaffold + CI green on an empty build (🔒 needs-human)

## Overview

The one-time human bootstrap: create the XcodeGen project definition, the formatter/linter
configs, and a minimal app entry point, then prove the whole Definition-of-Done pipeline green on
GitHub Actions. This is the ONE task that defines the project-wide conventions every later build is
gated on, and its whole-repo tooling sweep is a predictable scope-gate collision for an autonomous
builder — so the foundation is bootstrapped by hand. Once CI is green on `main`, mark this done
with `.harness/scripts/mark-done.sh T001` and the loop takes over.

## Do

1. **Install local tooling** (once per machine): `brew install xcodegen jq` (jq is likely present).
2. **Create `project.yml`** at the repo root:

   ```yaml
   name: Kakitori
   options:
     bundleIdPrefix: com.ryankrol
     deploymentTarget:
       iOS: "18.0"
     createIntermediateGroups: true
   settings:
     base:
       SWIFT_VERSION: "6.0"
       SWIFT_STRICT_CONCURRENCY: complete
       TARGETED_DEVICE_FAMILY: "1,2"          # universal: iPhone + iPad
       CODE_SIGNING_ALLOWED: "NO"
   targets:
     Kakitori:
       type: application
       platform: iOS
       sources: [Sources]
       info:
         path: Sources/Info.plist
         properties:
           UILaunchScreen: {}                  # REQUIRED — no letterboxing (legacy mode)
           CFBundleDisplayName: Kakitori
           UISupportedInterfaceOrientations:
             - UIInterfaceOrientationPortrait
             - UIInterfaceOrientationLandscapeLeft
             - UIInterfaceOrientationLandscapeRight
     KakitoriTests:
       type: bundle.unit-test
       platform: iOS
       sources: [Tests/UnitTests]
       dependencies:
         - target: Kakitori
     KakitoriUITests:
       type: bundle.ui-testing
       platform: iOS
       sources: [Tests/UITests]
       dependencies:
         - target: Kakitori
   schemes:
     Kakitori:
       build:
         targets: { Kakitori: all }
       test:
         targets: [KakitoriTests, KakitoriUITests]
   ```

3. **Create the minimal app** so the empty project builds:
   - `Sources/App/KakitoriApp.swift` — `@main struct KakitoriApp: App` showing
     `Text("Kakitori")` in a `WindowGroup`.
   - `Tests/UnitTests/SmokeTests.swift` — one XCTest asserting `true` (placeholder the first real
     task replaces).
   - `Tests/UITests/LaunchTests.swift` — one XCUITest that launches the app and asserts
     `app.state == .runningForeground` (keeps the UI-test target compiling from day one).
4. **Create `.swiftformat`** at the repo root:

   ```
   --swiftversion 6.0
   --indent 4
   --maxwidth 120
   --wraparguments before-first
   --wrapcollections before-first
   --disable wrapMultilineStatementBraces
   ```

5. **Create `.swiftlint.yml`** at the repo root (lenient start; `--strict` promotes warnings):

   ```yaml
   included:
     - Sources
     - Tests
   disabled_rules:
     - todo
     - trailing_comma        # SwiftFormat owns comma style
   line_length:
     warning: 140
     error: 200
     ignores_comments: true
     ignores_urls: true
   identifier_name:
     min_length: 1           # allow i, x in SRS interval math
   type_body_length:
     warning: 400
     error: 600
   file_length:
     warning: 600
     error: 1000
   ```

   `included: Sources, Tests` is also what keeps the linter OFF the vendored `.harness/**` tree and
   the harness-authored root prose — keep that exclusion intact forever.
6. **Create the dedicated simulator**: `./tools/loop_sim.sh` (creates the `Kakitori-Sim` iPad and
   prints its UDID).
7. **Run the full local DoD** from the repo root and get it green:

   ```sh
   ./tools/ensure-dod-tools.sh
   xcodegen generate
   ./tools/bin/swiftformat Sources Tests --lint
   ./tools/bin/swiftlint --strict
   xcodebuild test -project Kakitori.xcodeproj -scheme Kakitori \
     -destination 'platform=iOS Simulator,name=Kakitori-Sim' CODE_SIGNING_ALLOWED=NO
   ```

8. **Sanity-check `build_run.sh`** end-to-end once: `./build_run.sh` should build, install, launch,
   and write `screenshots/latest.png` showing the placeholder screen edge-to-edge (NO black
   letterbox bars — if bars appear, `UILaunchScreen` is missing from the generated Info.plist).
9. **Commit + push to `main`** and confirm the `CI` workflow goes green on GitHub Actions.
10. **Mark this task done**: `.harness/scripts/mark-done.sh T001`.

## Done when

- `CI` workflow is green on `main` with the real DoD steps (xcodegen → swiftformat --lint →
  swiftlint --strict → xcodebuild test), matching `.harness/config/harness.env`'s `LOCAL_DOD`.
- `Kakitori.xcodeproj` is generated (git-ignored), never committed.
- `./build_run.sh` produces a full-screen screenshot of the placeholder app on `Kakitori-Sim`.
- SwiftFormat/SwiftLint configs exclude everything outside `Sources`/`Tests` (`.harness/**` and the
  root prose are never swept).
- `.harness/scripts/mark-done.sh T001` has been run so dependent tasks unblock.
