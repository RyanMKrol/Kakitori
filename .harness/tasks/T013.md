# T013 — Trace guide layer: faded glyphs under the canvas

## Overview

Renders each writeable unit's glyph as a faded serif guide centred in its guide box, in a SwiftUI
layer that sits UNDER the transparent `WritingCanvas` — so Clear (which only empties the
`PKDrawing`) can never erase a guide (docs/05-architecture.md §5, docs/02-product-spec.md §3).

## Do

- Create `Sources/Features/Session/Canvas/TraceGuideLayer.swift`:
  `struct TraceGuideLayer: View`, `init(units: [SegmentedUnit], maxBoxesPerRow: Int = 6,
  isVisible: Bool)`. `SegmentedUnit` comes from `Sources/Importer/TargetSegmenter.swift`
  (do not edit that file). When `isVisible == false` no glyphs render (empty boxes only) —
  only Trace mode shows guides; every other mode passes `false` (docs/02 §2.1 vs §2.2–2.4).
- Each `.box(glyph)` unit renders its glyph centred in its guide box in
  `Font.custom("Hiragino Mincho ProN", size: boxSide * 0.72)` at `inkFaint` = 12% ink, i.e.
  `#1A1A1A` at 0.12 opacity (docs/06-ui-design.md §1: "inkFaint | 12% ink | guide glyphs in
  Trace mode"; docs/04-content-and-data.md §3: faded glyph in a textbook font). `.inline`
  punctuation renders exactly as `GuideBoxRow` already does (always visible, never faded-off).
- Glyph positions MUST match `GuideBoxRow`'s box positions exactly (same ≤`maxBoxesPerRow`
  chunking, same `min(160, availableWidth / boxesInRow)` sizing). Do this by REUSING the layout:
  extend `Sources/Features/Session/Canvas/GuideBox.swift` with an optional
  `traceGlyph: String? = nil` drawn centred inside the box, and
  `Sources/Features/Session/Canvas/GuideBoxRow.swift` with a `traceGlyphsVisible: Bool = false`
  parameter that feeds each box its glyph — then `TraceGuideLayer` composes `GuideBoxRow`
  (both files are in this task's scope). Do NOT duplicate the layout math.
- Intended composition (document it in a doc comment): `ZStack { TraceGuideLayer(...);
  WritingCanvas(...) }` — guides behind, strokes in front, Clear only affects the canvas.
- Container gets `accessibilityIdentifier = "trace-guide-layer"`.
- Add a `#Preview` with two variants on paper `#FAF8F4`: (1)
  `ZStack { TraceGuideLayer(units: TargetSegmenter.segment("ありがとう"), isVisible: true);
  WritingCanvas(...) }`; (2) the same with `isVisible: false`.
- Swift 6 strict concurrency; no force-unwraps in Sources.

## Done when

- The loop fires `./build_run.sh`; this component has no app screen yet, so if the launched app
  cannot show it, render the `#Preview` and judge that image; record in the worklog which
  capture you judged.
- The image must show: faded serif (mincho, NOT sans) あ り が と う glyphs centred one-per-box,
  clearly lighter than full ink but readable at 12%; the hairline borders and dashed centre
  cross-lines still visible; the `isVisible: false` variant showing the same boxes empty.
- Code inspection holds: glyphs are drawn by the SwiftUI layer (never added to the
  `PKDrawing`), so `WritingCanvasController.clear()` cannot remove them; layout is reused from
  `GuideBoxRow`, not duplicated.
