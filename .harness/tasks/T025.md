# T025 — SM2Scheduler: grade previews match applied results

## Overview

Implement `preview(for:now:)` on `SM2Scheduler` and declare `Scheduler` conformance, so the grade
buttons can show exactly the interval each grade would produce ("Again <1m", "Good 10m", "Easy 4d"
— docs/02-product-spec.md §1.3), with fuzz disabled so previews are stable.

## Do

In `Sources/Scheduler/SM2Scheduler.swift` (module rules unchanged: Foundation only, no
`Date()`/`.now`, no system RNG):

1. Add `extension SM2Scheduler: Scheduler` and implement
   `func preview(for card: ScheduleSnapshot, now: Date) -> [Grade: SchedulePreview]`:
   - For each of the four grades, compute the result via the **fuzz-free** path — e.g. run
     `SM2Scheduler(fuzzEnabled: false).apply(grade, to: card, now: now, rng: &dummy)` with a local
     `var dummy = SplitMix64(seed: 0)` (unused when fuzz is off). Previews NEVER fuzz, even on a
     fuzz-enabled scheduler — docs/03 §4: the per-grade interval is computed before grading for the
     button labels, so the label must equal what apply then persists (fuzz only nudges `dueAt`).
   - Wrap each result as `SchedulePreview(dueAt:intervalDays:label:)` where `label` is:
     - sub-day results (result state `.learning`/`.relearning`): with
       `seconds = dueAt − now`, `minutes = Int(seconds / 60)`; label is `"<1m"` when
       `minutes <= 1`, else `"\(minutes)m"` (so the 60s step → `"<1m"`, the 600s step → `"10m"`).
     - day results (result state `.review`): `"\(Int(intervalDays))d"` (e.g. `"1d"`, `"4d"`,
       `"12d"`, `"365d"`).
2. No behavior change to `apply`.

## Done when

`Tests/UnitTests/SM2PreviewTests.swift` (XCTest) passes with
`now = Date(timeIntervalSince1970: 1_000_000)`:

- **Preview == apply matrix.** For each snapshot in: new (step 0, EF 2.5), learning step 0
  (EF 2.5), learning step 1 (EF 2.5), review (I 10, EF 2.5), relearning (I 10, EF 2.3, lapses 1) —
  and for **all four grades**: `SM2Scheduler().preview(for:now:)[grade]` has `dueAt` and
  `intervalDays` exactly equal to `SM2Scheduler(fuzzEnabled: false).apply(grade, to:, now:,
  rng: &rng)` (seeded `SplitMix64`). Note the preview comes from a default **fuzz-enabled**
  scheduler — proving previews bypass fuzz.
- **Pinned labels**, learning step 0: again `"<1m"`, hard `"<1m"`, good `"10m"`, easy `"4d"`.
- **Pinned labels**, review I 10 / EF 2.5: again `"10m"`, hard `"12d"`, good `"25d"`, easy `"34d"`.
- **Pinned labels**, learning step 1: good `"1d"`; relearning I 10: good `"5d"`.
- Every preview dictionary contains exactly 4 entries (one per `Grade` case).
