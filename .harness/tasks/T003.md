# T003 — AppClock injected time source + 4 AM adjusted-day helper

## Overview

Create the app-wide injected time source. Every "current time" read in domain logic goes through
this type — a bare `Date()` / `.now` in scheduling/stats code is a build-failing violation — and it
owns the one implementation of the 4 AM local day-rollover (Anki convention, docs/03-srs-algorithm.md §7).

## Do

Create `Sources/Support/AppClock.swift` with a single value type:

```swift
struct AppClock: Sendable {
    var now: @Sendable () -> Date
    var timeZone: TimeZone
    var calendar: Calendar   // gregorian, with timeZone applied
}
```

- `static let system: AppClock` — `now: { Date() }`, `timeZone: .current`. (This is the ONLY place
  in the app allowed to write `Date()`.)
- `static func fixed(_ date: Date, timeZone: TimeZone = .current) -> AppClock` — test/preview clock
  that always returns `date`.
- `func adjustedDay(for date: Date) -> String` — the 4 AM rollover (docs/03-srs-algorithm.md §7:
  "The 'day' rolls over at 4:00 AM local time"): subtract 4 hours (`date.addingTimeInterval(-4 * 3600)`),
  then format that instant as `"YYYY-MM-DD"` in `timeZone` using a `DateFormatter` with
  `dateFormat = "yyyy-MM-dd"`, `locale = Locale(identifier: "en_US_POSIX")`, and the clock's time zone.
- `var today: String { adjustedDay(for: now()) }`.

Swift 6 strict concurrency: the struct and its statics must be `Sendable`-clean (build the formatter
inside `adjustedDay`, not as a shared stored property). No force-unwraps, no `try!`.

## Done when

- `Tests/UnitTests/AppClockTests.swift` (XCTest, not Swift Testing) exists and asserts, using a fixed
  `TimeZone(identifier: "Asia/Tokyo")` clock and dates built from `DateComponents`:
  - 2026-07-17 03:59 local → `adjustedDay` returns `"2026-07-16"` (before 4 AM = previous day)
  - 2026-07-17 04:00 local → `"2026-07-17"` (at 4 AM the new day starts)
  - 2026-07-17 00:30 local → `"2026-07-16"` (late-night session stays on the prior day)
  - 2026-07-17 23:30 local → `"2026-07-17"` (ordinary evening)
  - `AppClock.fixed(date).today` equals `adjustedDay(for: date)`.
- `grep -rn "Date()" Sources/Support/AppClock.swift` shows it only inside `AppClock.system`.
