# T017 — AnkiCollection: read models/decks/notes from collection.anki2

## Overview

Read an extracted `collection.anki2` (Anki schema v11 SQLite database) read-only via the system
`SQLite3` C library — no GRDB — and surface plain Sendable value types for the rest of the import
pipeline (docs/04-content-and-data.md §1, docs/05-architecture.md §2).

## Do

Create `Sources/Importer/AnkiCollection.swift` (`import SQLite3`, `import Foundation`):

- Value types, all `Sendable` structs:
  - `AnkiModel { let id: Int64; let name: String; let fieldNames: [String] }` — field names in
    `ord` order.
  - `AnkiDeckInfo { let id: Int64; let name: String }` — names use Anki's `::` hierarchy
    (docs/04 §1.2, e.g. `Japanese for Busy People Book 1: Kana::Lesson 1: …`).
  - `AnkiNote { let id: Int64; let guid: String; let modelID: Int64; let fields: [String];
    let tags: String; let deckID: Int64? }`.
- `enum AnkiCollectionError: Error, Equatable`: `cannotOpen`, `malformedCollection`,
  `noAnkiBuilderModel`, `zeroNotes`.
- `struct AnkiCollection { let models: [AnkiModel]; let decks: [AnkiDeckInfo];
  let notes: [AnkiNote]; init(databaseURL: URL) throws }`.

`init` behaviour:

1. `sqlite3_open_v2(path, &db, SQLITE_OPEN_READONLY, nil)`; non-OK → `.cannotOpen`. Always
   `sqlite3_close` (use `defer`); finalize every statement. No force-unwraps — a NULL
   `sqlite3_column_text` pointer paths to `.malformedCollection`.
2. `SELECT models, decks FROM col` (single row). Both columns are JSON blobs
   (docs/04 §1): `models` is an object keyed by model-id string, each value has `"name"` and
   `"flds"` (array of objects with `"name"` and `"ord"`) — sort by `ord` to build `fieldNames`.
   `decks` is an object keyed by deck-id string, each value has `"name"`. Parse with
   `JSONSerialization`; malformed JSON → `.malformedCollection`.
3. `SELECT id, guid, mid, flds, tags FROM notes`. Split `flds` on the `0x1F` unit separator with
   `components(separatedBy: "\u{1F}")` — NEVER `split(separator:)`, which would drop empty
   fields and break positional mapping.
4. Note→deck mapping: `SELECT nid, did FROM cards` into a `[Int64: Int64]` (first card per note
   wins) and set each `AnkiNote.deckID`. Only deck membership is read from `cards`; all
   scheduling columns stay ignored per docs/04 §1 (Kakitori schedules itself).
5. Validate (docs/04 §2 step 4, docs/05 §6): no model whose `fieldNames` contains `"Target"` →
   throw `.noAnkiBuilderModel`; `notes` empty → throw `.zeroNotes`.

extract `Fixture.kanaDeckURL` (helper from `Tests/UnitTests/FixtureLocator.swift`) into a temp
directory, then `AnkiCollection(databaseURL:)` on the extracted `collection.anki2`. Assert:

- `notes.count == 114` (the fixture is a real 114-note kana deck).
- At least one model's `fieldNames` contains `"Target"`, and that model's `fieldNames` contains,
  in this relative order, all seven AnkiBuilder field names from docs/04 §1.1:
  `Target, Pronunciation, English, Category, Hint, Image, Audio`.
- `decks` is non-empty and at least one deck name contains `"::"` (the fixture has subdecks).
- Every note's `fields.count` equals its model's `fieldNames.count`, and every note has a
  non-nil `deckID`.
- Opening a path with no database file throws `AnkiCollectionError.cannotOpen` (or a thrown
  error — assert `XCTAssertThrowsError`).
