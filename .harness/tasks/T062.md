## Overview
Bundled decks currently store their raw Anki names ('Tofugu Hiragana Anki Deck' / 'Tofugu Katakana Anki Deck') as both `name` and `sourceDeckName`, and leave `jpTitle` nil. Apply a DISPLAY-only friendly-name + Japanese-title override at import time so these decks read as 'Hiragana'/'Katakana' with JP titles ひらがな/カタカナ, WITHOUT changing the stored `sourceDeckName` (the re-import idempotence key and `deck-row-` accessibilityIdentifier source).

## Do
Add a small static mapping keyed on `sourceDeckName` (the raw Anki top-deck name) → (friendly `name`, `jpTitle`): 'Tofugu Hiragana Anki Deck' → ('Hiragana','ひらがな'), 'Tofugu Katakana Anki Deck' → ('Katakana','カタカナ'). In ApkgImporter.firstImport (~L201) set the new deck's `name` and `jpTitle` from the mapping when it matches (fall back to `parsed.deckName` / nil otherwise); leave `sourceDeckName = parsed.deckName` UNCHANGED. In ApkgImporter.reimport (~L246) ALSO apply the mapping so existing installs get renamed on their next re-import (set `deck.name` and `deck.jpTitle`). Bump `BundledDeckLoader.bundleVersion` (currently 1) by one so existing installs re-import and pick up the new name/title on next launch. Do NOT change `deckResourceNames` or the .apkg files.

## Done when
After importing (first import AND re-import) each bundled deck: `deck.name` == 'Hiragana'/'Katakana', `deck.jpTitle` == 'ひらがな'/'カタカナ', and `deck.sourceDeckName` is still the original 'Tofugu … Anki Deck' string (unchanged). A re-import (apply called twice on the same parsed deck) is still idempotent — matched by unchanged sourceDeckName, no duplicate deck, note count stable — and the second apply leaves name/jpTitle at the friendly values. Add/extend an XCTest against an in-memory ModelContainer covering both the firstImport values and the reimport idempotence+rename. Non-bundled decks (sourceDeckName not in the map) keep `name = parsed.deckName` and `jpTitle == nil`.
