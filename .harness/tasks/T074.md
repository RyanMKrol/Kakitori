## Overview
On the Session writing canvas the guide box occupies only a corner of a much larger full-pane surface, which produces two linked defects: a single-character target's box sits jammed top-left instead of centred, and PencilKit accepts ink anywhere on the pane (well outside the visible box). Root cause #1 (centring): a trailing Spacer() in GuideBoxRow (Sources/Features/Session/Canvas/GuideBoxRow.swift:52) eats leftover width and defeats the row's own .frame(alignment:.center), so boxes hug the leading edge. Root cause #2 (unbounded ink): the PKCanvasView (WritingCanvas.swift, drawingPolicy .anyInput) is stretched to fill the whole pane in CanvasPaneView's ZStack (CanvasPaneView.swift:15-23), decoupled from the guide-box geometry that is computed inside GuideBoxRow's GeometryReader. Fix them as ONE unit by making the drawable rect coincide with the visible (centred) guide-box region, so 'centre the box' and 'draw only in the box' become one consistent surface. Note: TraceGuideLayer/GuideBoxRow render in every mode (only trace GLYPHS are gated by isVisible), so the boxes — and this fix — apply across trace/recall/listen/translate. This is ordinary, testable, visual layout work and is entirely separate from T014 (needs-human Apple-Pencil ink-FEEL tuning); do not touch ink feel here.
