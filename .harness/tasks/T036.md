# T036 — Home deck list: deck cards, mastery %, status chips, today banner

## Overview

Add the dark today-practice banner and the 2-up deck-card grid to Home (mastery %, count chips,
Study button), plus the no-decks empty state with an import button.

## Do

- Create `Sources/Features/Home/TodayBannerView.swift` and `Sources/Features/Home/DeckCardView.swift`;
  compose both in `Sources/Features/Home/HomeView.swift` below the stats row.
- **Today banner** (docs/02-product-spec.md §1.1, docs/06-ui-design.md §2.1): full-width dark card
  (`KakitoriTheme.ink` fill, ~20pt radius, paper text): small-caps kicker `TODAY'S PRACTICE`
  (reference prototype), then the copy `N characters to write · across M scripts` — splitting into a
  bold "N characters to write" headline with "across M scripts" beneath is fine, characters must
  match. N = live total due across all decks (the sum of the new/learning/due counts below);
  M = number of decks with ≥1 such card. Hide the banner when N == 0.
- **Deck cards** in a 2-column `LazyVGrid` (regular width; the iPhone list is a later task). Each
  card (white fill, hairline border, ~20pt radius; docs/02 §1.1, docs/06 §2.1):
  - JP title (`deck.jpTitle`, `KakitoriTheme.japaneseDisplayFont` ~32pt), EN title (`deck.name`,
    bold 15pt), subtitle `N cards` (count of non-deleted notes — the Deck model has no stored
    subtitle field, so derive the count).
  - Mastery badge right-aligned, `P%` where P = mature/total cards, mature = `intervalDays >= 21`
    (docs/02 §4.4). Accent text on accent-tint fill.
  - Chips row `3 new · 3 learning · 4 due` as three chips using the `KakitoriTheme` chip tokens
    (Home wording is "learning"; docs/06 §2.1). Counts from each note's `CardSchedule`:
    new = state new; learning = learning + relearning; due = review with `dueAt` ≤ now (injected
    clock, never bare `Date()`).
  - Accent `Study →` label bottom-right.
  - Whole card tappable, identifier `"deck-row-" + sourceDeckName.lowercased()` (the demo seed's
    Hiragana deck → `deck-row-hiragana`). Tap/Study calls an injected `onStudy(Deck)` closure —
    HomeView passes a no-op for now; the deck-setup sheet presentation is wired by the later
    end-to-end task.
- **Empty state** when no non-deleted decks exist (docs/06 §2.1): centred 書 mark,
  `Import a deck to start writing`, and an `Import deck` button (identifier
  `"import-button-empty"`, no action yet — the import task wires it).

## Done when

- `./build_run.sh Kakitori-Sim -seedDemoData YES` screenshot shows: the dark banner with
  "… characters to write · across … scripts", and 2-up deck cards each with JP + EN titles,
  a mastery % badge, three chips ("N new", "N learning", "N due"), and "Study →".
- `./build_run.sh` without the seed shows the 書 empty state with "Import a deck to start writing"
  and the Import deck button.
