# T045 — Mode availability rules per deck and card

## Overview

Pure rules deciding which practice modes a deck offers (docs/02-product-spec.md §2.5 table) and
which cards qualify per mode, plus hiding unavailable modes in the deck setup sheet
(docs/02 §1.2: "Modes that don't apply to the deck are hidden").

## Do

1. Create `Sources/Scheduler/ModeAvailability.swift` — a pure `enum ModeAvailability` (no SwiftUI,
   no SwiftData queries; it references `PracticeMode` from
   `Sources/Features/Session/PracticeMode.swift` and `Script` from `Sources/Models/Note.swift`,
   same target):
   - `static func deckModes(scripts: Set<Script>) -> [PracticeMode]` — returns, in this order:
     `.trace`, `.listen`, `.translate` (ONLY if `scripts` contains `.kanji` or `.mixed`), `.recall`,
     `.mixed`. Per the docs/02 §2.5 table: kana decks get Trace / Listen & Write / Recall and NOT
     Translate & Write ("✗ (no gloss)" — a bare kana has a sound, not a translation); the kanji
     deck gets all four. Mixed is always offered (it rotates through whatever is available).
   - `static func cardQualifies(_ mode: PracticeMode, hasAudio: Bool, ttsAvailable: Bool, english: String?) -> Bool`:
     - `.listen` → `hasAudio || ttsAvailable`. docs/04-content-and-data.md §1.1 excludes notes with
       no `Audio` from Listen & Write sessions; docs/04 §4 provides the ja-JP speech fallback, so
       the exclusion bites only when TTS is also unavailable.
     - `.translate` → `english` non-nil and non-empty after trimming whitespace and newlines.
     - `.trace`, `.recall`, `.mixed` → always `true`.
2. `Sources/Features/DeckSetup/DeckSetupSheet.swift`: build the mode rows from
   `ModeAvailability.deckModes(scripts: Set(deck's non-deleted notes' script))` instead of listing
   every mode — unavailable modes are hidden entirely, not disabled. Row content (glyph, label,
   blurb, selection) is unchanged.
3. Where the session queue is filled for a chosen mode (session start in
   `Sources/Features/Session/SessionViewModel.swift` / `Sources/Scheduler/SessionQueue.swift`),
   filter cards with `cardQualifies` for the session's mode (pass
   `hasAudio: note.audioFilename != nil`, `ttsAvailable:` from the injected `AudioPlaying.isAvailable`,
   `english: note.english`). Trace and Recall sessions are unaffected.

## Done when

`Tests/UnitTests/ModeAvailabilityTests.swift` (XCTest) passes, asserting:

- `deckModes(scripts: [.hiragana])` == `[.trace, .listen, .recall, .mixed]` (no translate); same for
  `[.katakana]` and `[.hiragana, .katakana]`.
- `deckModes(scripts: [.kanji])` == `[.trace, .listen, .translate, .recall, .mixed]`; a set
  containing `.mixed` script also includes `.translate`.
- Card level: `cardQualifies(.listen, hasAudio: false, ttsAvailable: false, english: nil)` is false;
  `(.listen, false, true, nil)` true; `(.listen, true, false, nil)` true.
- `cardQualifies(.translate, ...)` with english `nil`, `""`, and `"  "` is false; with
  `"Good morning."` is true. `.trace` and `.recall` are true even with `(false, false, nil)`.
