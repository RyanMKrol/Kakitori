# T050 — Dark mode palette

## Overview

Make the KakitoriTheme palette adaptive so the app inverts correctly in dark mode per
docs/06-ui-design.md §1 "Dark mode: paper → #17150F-ish warm near-black, ink inverts, accent
stays", and purge hardcoded light-only colors from every Feature view.

## Do

1. `Sources/Support/Theme.swift`: convert each palette token to an adaptive `Color` (e.g.
   `Color(uiColor: UIColor { trait in trait.userInterfaceStyle == .dark ? darkValue : lightValue })`):
   - `paper`: light `#FAF8F4` → dark `#17150F` (warm near-black, docs/06 §1).
   - `ink`: light `#1A1A1A` → dark warm off-white `#F0EDE6` (docs say only "ink inverts"; this
     exact value is invented — keep it warm, not pure white).
   - `accent`: `#C43B2E` in BOTH modes ("accent stays").
   - `inkFaint` (12% ink) and `boxLine` (8% ink): keep them derived as opacities OF the adaptive
     `ink` token so they invert automatically.
   - Chip fills (`chipNew` neutral / `chipLearn` red-tint / `chipDue` ink fill): derive from the
     adaptive tokens; verify chip label text keeps contrast in dark (chipDue text sits on an
     ink-filled chip, so it must use `paper`).
2. Audit EVERY view under `Sources/Features/` (Home, DeckSetup, Session, Summary) plus
   `Sources/Support/` for hardcoded light values — `Color.white`, `Color.black`, `.foregroundColor(.black)`,
   raw hex literals outside Theme.swift — and replace each with the matching KakitoriTheme token.
3. Canvas surfaces: the canvas background uses `paper`, guide-box borders/dashed crosses use
   `boxLine`, trace guides use `inkFaint`, and the user's stroke color uses the `ink` token so
   strokes stay visible on dark paper.

## Done when

- Dark screenshot (appearance is NOT reset by build_run.sh, so set it first):
  ```
  UDID=$(./tools/loop_sim.sh)
  xcrun simctl ui "$UDID" appearance dark
  ./build_run.sh Kakitori-Sim -seedDemoData YES
  ```
  `screenshots/latest.png` shows: warm near-black background (no pure-black, no leftover white
  panels), warm off-white text, vermillion accent on mode labels / primary buttons / streak.
- Then RESET in this same task — appearance persists on the simulator and would corrupt later
  tasks' screenshots:
  ```
  xcrun simctl ui "$UDID" appearance light
  ./build_run.sh Kakitori-Sim -seedDemoData YES
  ```
  and confirm the light screenshot is unchanged from before this task (paper `#FAF8F4`, ink text).
