# T010 — TargetSegmenter: writeable units + script classification

## Overview

Pure-Foundation logic that splits a note's `Target` string into guide-box units plus in-line
punctuation, and classifies its script by Unicode range scan (docs/04-content-and-data.md §3).
No SwiftUI, no SwiftData — the guide-box UI consumes the output in a later task.

## Do

Create `Sources/Importer/TargetSegmenter.swift` with exactly these types (all `Sendable`):

```swift
enum SegmentedUnit: Equatable, Sendable {
    case box(String)      // one writeable character → one guide box
    case inline(String)   // punctuation typeset between boxes — gets NO box
}
enum TargetScript: String, Equatable, Sendable {
    case hiragana, katakana, kanji, mixed
}
enum TargetSegmenter {
    static func segment(_ target: String) -> [SegmentedUnit]
    static func classify(_ target: String) -> TargetScript
}
```

- `segment` walks the string's `Character`s in order. The punctuation set is checked FIRST and is
  exactly `。、！？・ー` (docs/04 §3 — note `ー` U+30FC sits in the katakana Unicode block but is
  punctuation here, so the set check must precede any range check) → `.inline`. Whitespace and
  newlines are dropped (no unit). Every other character → `.box`. Order is preserved.
- `classify` scans the Unicode scalars of `.box` units only (`.inline` ignored): all scalars in
  U+3040–U+309F → `.hiragana`; all in U+30A0–U+30FF → `.katakana`; all in U+4E00–U+9FFF (CJK
  Unified Ideographs) → `.kanji`. Any combination of ranges, any scalar outside all three (Latin,
  digits), or an empty box list → `.mixed`.
- Foundation only. No force-unwraps, no `try!`.

## Done when

- `Tests/UnitTests/TargetSegmenterTests.swift` exists (XCTest, NOT Swift Testing) with table cases:
  - pure kana `"ありがとう"` → 5 `.box` units; `classify` == `.hiragana`
  - katakana `"カメラ"` → 3 `.box`; `.katakana`
  - kanji `"日本語"` → 3 `.box`; `.kanji`
  - mixed `"お名前"` → 3 `.box`; `.mixed`
  - punctuation `"おはようございます。"` → 9 `.box` then `.inline("。")` last, in order; `.hiragana`
  - `"コーヒー"` → `.box` only for `コ` and `ヒ`, each `ー` as `.inline` in its original position;
    `.katakana`
  - long phrase `"はじめまして、どうぞよろしく。"` → exactly 13 `.box` units, with `.inline("、")`
    after the 6th box and `.inline("。")` last (15 elements total); `.hiragana`
- Punctuation cases assert the FULL `[SegmentedUnit]` array via `XCTAssertEqual` (order matters).
