# T054 — App icon (🔒 needs-human RUNBOOK)

## Overview

Design and wire up the app icon. This is a human design decision — the loop must not attempt it.
docs/06-ui-design.md §5 gives the direction: "Vermillion hanko-stamp square, off-white 書 (or the
Kakitori mark) inside, on paper background. Matches the in-app identity; reads at small sizes."

## Do (human steps)

1. Produce a 1024×1024 master PNG (no alpha — App Store icons reject transparency). The obvious
   direction per the app's mark: hanko-red (`#C43B2E`) stamp bearing 書 in a Mincho face on the
   paper background (`#FAF8F4`). docs/06 §5 says a stamp SQUARE; the in-app header mark is a red
   circle — your call. Any tool is fine (design app, or render the mark in the HTML prototype and
   crop).
2. Add the asset catalog: `Sources/Assets.xcassets/AppIcon.appiconset/` with the 1024 PNG and a
   `Contents.json` using the iOS `single-size` format (one `universal` 1024 entry).
3. Wiring: if the Kakitori target in `project.yml` does not already pick up
   `Sources/Assets.xcassets` as a resource, add it, plus the build setting
   `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` if the generated project doesn't set it by
   default. Then `xcodegen generate` (never hand-edit the `.xcodeproj`).
4. Verify: `./build_run.sh Kakitori-Sim`, then in the Simulator press Home (Device → Home) and
   check the Kakitori icon on the home screen — not the blank/grid placeholder, legible at that
   size.
5. Commit the asset + any `project.yml` change (and push).

## Done when

- The simulator home screen shows the new icon.
- The full local DoD suite still passes.
- You mark it complete yourself: `.harness/scripts/mark-done.sh T054`
