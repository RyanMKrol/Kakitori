# T042 — Listen & Write mode (聞き取り)

## Overview

Add the Listen & Write session prompt — a large play button, autoplay on card entry, empty guide
boxes (docs/02-product-spec.md §2.2) — and wire the real `AudioService` into the answer block's
play button for every mode (docs/04-content-and-data.md §4: replay on reveal in every mode).

## Do

1. If a `PracticeMode` enum does not already exist, create `Sources/Features/Session/PracticeMode.swift`:
   `enum PracticeMode: String, CaseIterable, Codable, Sendable { case trace, listen, translate, recall, mixed }`
   with three computed vars:
   - `glyph`: 書 / 聞 / 訳 / 思 / 混 (docs/06-ui-design.md §2.2 gives 書 聞 訳 思; 混 for Mixed comes
     from the prototype's mode list — the docs give Mixed no glyph).
   - `label`: "Trace" / "Listen & write" / "Translate & write" / "Recall" / "Mixed" (prototype MODES).
   - `blurb`: "Write over a faded guide" / "Hear it, then write what you heard" /
     "See the English, write the Japanese" / "From the reading, write from memory" /
     "Rotate through all four modes" (prototype MODES one-liners, per docs/02 §1.2).
2. `Sources/Features/Session/SessionViewModel.swift` (`@Observable @MainActor`):
   - Inject audio via the protocol from `Sources/Audio/AudioService.swift`: an init parameter
     `audio: any AudioPlaying = AudioService()` stored on the VM (default keeps DI inside this
     task's scope; tests can pass `FakeAudioPlayer`).
   - `var autoplayEnabled: Bool` = `(UserDefaults.standard.object(forKey: "audioAutoplay") as? Bool) ?? true`
     (setting default on, docs/02 §5).
   - `func replayAudio()` — calls `audio.play(target:audioFilename:deckID:)` for the current note.
   - Whenever a new card becomes current AND its presented mode is `.listen` AND `autoplayEnabled`,
     call `replayAudio()` once (autoplay on card entry; when the setting is off, no autoplay — the
     button still works).
3. Prompt pane (the session prompt view in `Sources/Features/Session/`), when the presented mode is
   `.listen`, before reveal: accent small-caps caption "Listen & write" (same style as the existing
   Trace caption), a large play button — SF Symbol `play.circle.fill`, at least 64pt, in
   `KakitoriTheme.accent`, `.accessibilityIdentifier("play-audio")`, tap calls `replayAudio()` —
   and the sub-line "Tap to hear it again, then write what you hear." No target, reading, or English
   text before reveal (docs/02 §2.2: "No text shown before reveal").
4. Canvas: in `.listen` the guide boxes are EMPTY — the faded trace glyph layer renders only when the
   presented mode is `.trace`. Canvas hint line for listen: "Write the word you heard" (prototype).
5. Answer reveal, ALL modes: replace the disabled play-button placeholder in the answer block with an
   enabled button that calls `replayAudio()` (deck MP3, or ja-JP speech when `audioFilename` is nil).
