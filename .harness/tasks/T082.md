## Overview
BUG (idea #8 Part A): if a deck's day is already done (its DailyStats dayTarget is met, or the built SessionQueue has nothing to serve) and the user taps Start Writing (via DeckSetupSheet) or 'Study another', SessionViewModel.init() calls finish(now:) immediately (Sources/Features/Session/SessionViewModel.swift lines 143-150) and HomeView's sessionCoverContent renders a hollow SummaryView reading 'Session complete — 0 cards written in 0 min'. Pushing the user through an empty session is wrong. Fix: distinguish 'finished a real session' from 'there was nothing to study', and when there was nothing to study surface a caught-up state instead of a hollow 0/0 summary. This unit does the plumbing + the caught-up screen; it does NOT add Free Study (that is unit sbf-b/sbf-c) and it does NOT change the 'Study another' routing (that is unit sbf-d).
