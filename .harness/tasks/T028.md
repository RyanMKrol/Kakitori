# T028 — StatsRecorder: grades + study time into DailyStats

## Overview

Create the small persistence helper that turns each grade and each chunk of study time into
counters on the `DailyStats` row for the current 4 AM-adjusted day (create-if-missing), per
docs/03-srs-algorithm.md §7.

## Do

Create `Sources/Models/StatsRecorder.swift`. Established facts (already built — reuse, don't
recreate): `DailyStats` is a SwiftData `@Model` at `Sources/Models/DailyStats.swift` with
`day: String` ("YYYY-MM-DD" of the 4 AM-adjusted day), `cardsWritten: Int`, `newIntroduced: Int`,
`reviewsDone: Int`, `secondsStudied: Int` — streak logic already lives with it (leave it alone).
`CardState` (`new/learning/review/relearning`) is in `Sources/Models/CardSchedule.swift`.
`Sources/Support/AppClock.swift` (T003) holds the injected time source and the 4 AM adjusted-day
helper — **read that file first and reuse its helper** for the day key; do not reimplement. The
rule it encodes (docs/03 §7: day rolls over at 4:00 AM local): adjusted day = the local calendar
date of `now - 4 hours`, formatted `"YYYY-MM-DD"`.

Define (SwiftData is allowed in this file; all access on the main actor):

```swift
@MainActor
enum StatsRecorder {
    /// previousState = the card's CardState BEFORE the grade was applied.
    static func recordGrade(previousState: CardState, now: Date, in context: ModelContext) throws
    static func recordStudySeconds(_ seconds: Int, now: Date, in context: ModelContext) throws
}
```

- Both fetch the `DailyStats` row whose `day` equals the adjusted-day key for `now`
  (`FetchDescriptor` + `#Predicate`), inserting a fresh all-zero row if none exists.
- `recordGrade`: `cardsWritten += 1` always; `newIntroduced += 1` only when
  `previousState == .new` (card seen for the first time); `reviewsDone += 1` only when
  `previousState == .review`. Learning/relearning grades bump `cardsWritten` only.
- `recordStudySeconds`: `secondsStudied += seconds`.
- Time is injected: `now` is always a parameter — `Date()`/`.now` inside this file is a
  build-failing violation. Use `throws`, never `try!`; no force-unwraps.

## Done when

`Tests/UnitTests/StatsRecorderTests.swift` (XCTest, `@MainActor`) passes using an in-memory store
(`ModelConfiguration(isStoredInMemoryOnly: true)`) and fixed dates built via
`Calendar.current.date(from: DateComponents(...))` in **local** time:

- A grade at 2026-07-17 15:00 with `previousState: .new` creates row `day == "2026-07-17"` with
  `cardsWritten 1, newIntroduced 1, reviewsDone 0`.
- **3:59 AM lands on the previous day**: a grade at 2026-07-18 03:59 with `previousState:
  .learning` lands on the SAME `"2026-07-17"` row (`cardsWritten 2`, `newIntroduced` still 1);
  a grade at 2026-07-18 04:00 creates a separate `"2026-07-18"` row.
- A grade with `previousState: .review` bumps `reviewsDone` to 1 and leaves `newIntroduced`
  untouched.
- Seconds accumulate: `recordStudySeconds(30, ...)` then `recordStudySeconds(45, ...)` at the same
  adjusted day → that row's `secondsStudied == 75`.
- Exactly one `DailyStats` row exists per adjusted day after all of the above (no duplicates).
