# T008 — App entry: ModelContainer wiring + Home shell

## Overview

Replace the placeholder app entry with real SwiftData wiring over every model type, and give the app
a minimal Home screen: the Kakitori header plus an empty-state message when no decks exist. Later
tasks build the real stats row and deck cards on top of this shell.

## Do

1. Rewrite `Sources/App/KakitoriApp.swift`:
   - `@main struct KakitoriApp: App` builds one `ModelContainer` over ALL model types:
     `Deck.self, Section.self, Note.self, CardSchedule.self, DailyStats.self` (all in `Sources/Models/`).
   - Build it in a throwing static helper wrapped in `do/catch`; on failure call
     `fatalError("Failed to create ModelContainer: \(error)")` — never `try!` or force-unwrap.
   - `WindowGroup { NavigationStack { HomeView() } }` with `.modelContainer(container)`.
2. Create `Sources/Features/Home/HomeView.swift` — SwiftUI, `@Query private var decks: [Deck]`,
   background `KakitoriTheme.paper` (the token enum in `Sources/Support/Theme.swift`):
   - **Header** (docs/02-product-spec.md §1.1 / docs/06-ui-design.md §2.1): the 書 mark — `Text("書")`
     in `KakitoriTheme.japaneseDisplayFont(size: 28)`, colored `KakitoriTheme.paper`, inside a
     `Circle().fill(KakitoriTheme.accent)` ~44pt — beside the app name `Text("Kakitori")` and the
     tagline `Text("Write your Japanese, card by card")` (docs/02 §1.1). Give the header container
     `.accessibilityIdentifier("home-header")`.
   - **Empty state** when `decks.isEmpty` (docs/06-ui-design.md §2.1): a centered 書 mark in
     `KakitoriTheme.inkFaint` plus `Text("Import a deck to start writing")` in `KakitoriTheme.ink`.
     Container gets `.accessibilityIdentifier("home-empty-state")`. No Import button yet — import UI
     is a later task.
   - **When decks exist**: a plain `List`/`ForEach` of each deck showing `jpTitle ?? name` in
     `KakitoriTheme.japaneseDisplayFont(size: 24)` with `name` as a caption — deliberately minimal,
     `.accessibilityIdentifier("home-deck-list")`. No navigation, no Study button yet.

House rules: `@Observable` if any view model is introduced (never `ObservableObject` — though none
is needed here), `NavigationStack` (never `NavigationView`), Swift 6 strict concurrency clean, no
force-unwraps or `try!` in Sources.

## Done when

- The app builds and launches to Home: header (accent circle 書 + "Kakitori" + tagline) with the
  empty-state message visible on a fresh install (store starts empty).
- The `ModelContainer` includes all five `@Model` types — inserting any of them at runtime cannot
  throw "model not registered".
- Identifiers `home-header`, `home-empty-state` (empty store) and `home-deck-list` (populated store)
  are set. The old placeholder `Text("Kakitori")` body from the scaffold task is gone.
