## Overview
On the post-session SummaryView, the iPhone (compact) button LABELED 'Back home' does not go home — it immediately re-opens the DeckSetupSheet for the deck just finished. Root cause: SummaryView's two buttons have their COMPACT labels swapped relative to their actions. The first button (accent) fires `onBackToDecks`, which in HomeView clears the session and returns to Home opening nothing (the true go-home path), but its compact label reads 'Study another script'. The second button (outlined) fires `onStudyAnother`, which sets `deckPendingRestudy` so the fullScreenCover's `onDismiss` re-opens the setup sheet, but its compact label reads 'Back home'. So on iPhone, tapping 'Back home' lands the user in the setup sheet instead of Home. The regular (iPad) labels — 'Back to decks' (go home) and 'Study another deck' (re-open setup) — are correctly paired and must NOT change.

## Do
In Sources/Features/Summary/SummaryView.swift, `buttonsView`: swap the two COMPACT-branch labels so each matches its button's action.

- The button with `action: onBackToDecks` (accent button, `.accessibilityIdentifier("back-to-decks")`) currently shows `Text(isCompact ? "Study another script" : "Back to decks")`. Change the compact label to "Back home", so it reads `Text(isCompact ? "Back home" : "Back to decks")`. This button goes home (clears the session, opens nothing) — its label now says so on both size classes.
- The button with `action: onStudyAnother` (outlined button, `.accessibilityIdentifier("study-another")`) currently shows `Text(isCompact ? "Back home" : "Study another deck")`. Change the compact label to "Study another script", so it reads `Text(isCompact ? "Study another script" : "Study another deck")`. This button re-opens the just-finished deck's setup sheet — its label now describes re-studying, not going home.

Do NOT change the regular (non-compact) labels, the button actions, the accessibility identifiers, the ordering, or the styling. Do NOT change any wiring in HomeView.swift — the `onBackToDecks` / `onStudyAnother` callbacks and the fullScreenCover `onDismiss` are already correct; Home is in scope only so you may read it to confirm the action semantics before editing the labels (no edit to HomeView.swift is expected).

Separate, explicitly OUT OF SCOPE: the re-opened setup sheet shows '0 cards to study' for a just-finished deck — do not attempt to fix that here.
