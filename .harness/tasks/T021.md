# T021 — ApkgImporter: re-import idempotence + soft delete

## Overview

Extend `Sources/Importer/ApkgImporter.swift` (T020) so importing the SAME deck again — matched by
`Deck.sourceDeckName` — updates content and media in place, preserves all scheduling state, adds
genuinely new notes as `.new`, and soft-deletes notes that disappeared (docs/04-content-and-data.md
§2 step 8: "Field edits never reset progress"). Also wire `MediaStore` (T019) into the pipeline.

## Do

Only modify `Sources/Importer/ApkgImporter.swift` and add tests — `Sources/Models/` stays
untouched (notes are matched by the deterministic `Note.id` that T020 derives from the Anki guid
via `NoteIdentity.uuid(forAnkiGUID:)`; no new stored properties are needed).

1. Rework `apply(_ parsed: ParsedDeck) throws -> UUID`: fetch an existing `Deck` with
   `sourceDeckName == parsed.deckName`.
   - **None found** → the T020 first-import path, unchanged.
   - **Found** → re-import: update `importedAt`; build a dictionary of the deck's existing
     `Note`s keyed by `id`; compute each incoming note's id with
     `NoteIdentity.uuid(forAnkiGUID:)`; then:
     - **Matched** id → update content in place (`target`, `pronunciation`, `english`,
       `category`, `hint`, `audioFilename`, `units`, `script`, section assignment) and set
       `isDeleted = false` (a note that returns is revived). Do NOT touch `note.schedule` in
       any way — state, `stepIndex`, `easeFactor`, `intervalDays`, `dueAt`, `lapses` all
       survive verbatim.
     - **New** id (not in the dictionary) → insert a new `Note` + a new `CardSchedule` in state
       `.new`, exactly like first import.
     - **Disappeared** (existing id absent from the incoming set) → set `isDeleted = true`.
       NEVER hard-delete the `Note` and never remove or reset its `CardSchedule` (soft delete
       only, per docs/04 §2 step 8).
     - Sections: create any `Section` whose `sectionName` doesn't exist yet; reuse existing
       ones by name.
   - Save once at the end; still return the deck's `UUID` (existing deck keeps its original id,
     so its media directory is stable).
2. Wire media (T019, docs/04 §2 step 6): in `importDeck(from:)`, after `apply` returns the deck
   UUID, call `MediaStore(baseURL: mediaBaseURL).copyMedia(manifestURL:payloadDirectory:deckID:)`
   with the extracted `media` manifest and extraction temp directory (keep the temp directory
   alive until after the copy). `MediaStore` overwrites by name, so re-import refreshes media
   in place. `parse` must therefore also expose the extraction directory — e.g. add
   `let extractionDirectory: URL` to `ParsedDeck` (it is `Sendable`; `URL` is fine).

## Done when

`Tests/UnitTests/ApkgReimportTests.swift` (XCTest) passes, using the T020 test setup (in-memory
`ModelContainer` with `isStoredInMemoryOnly: true`, temp media base, `Fixture.kanaDeckURL`,
`@testable import`):

- **Idempotence**: `importDeck` run twice → still exactly 1 `Deck`, exactly 114 `Note` rows
  (no duplicates: 114 distinct `Note.id` values), all `isDeleted == false`.
- **Scheduling preserved**: after the first import, mutate one note's schedule to a
  distinctive state (`state = .review`, `intervalDays = 5`, `easeFactor = 2.6`,
  `dueAt = ` a fixed future `Date`), save, re-import, and assert every one of those values is
  unchanged and untouched notes still have `state == .new` with `dueAt == nil`.
- **Soft delete**: build a doctored `ParsedDeck` from `parse(url: Fixture.kanaDeckURL)` with one
  note removed (`notes.dropLast()`), call `apply` directly on the actor → the dropped note's
  `Note` row still exists with `isDeleted == true` AND a surviving non-nil `schedule`; exactly
  113 notes have `isDeleted == false`.
- **Revival**: re-applying the FULL `ParsedDeck` afterwards flips that note back to
  `isDeleted == false`, still 114 total rows.
- **Media**: after `importDeck`, at least one `.mp3` file exists under
  `<mediaBase>/Media/<deckUUID>/`, and a second `importDeck` leaves the file count there
  unchanged.
