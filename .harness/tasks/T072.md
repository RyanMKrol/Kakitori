## Overview
Home shows FIVE deck cards but should show THREE. The Foundations swap bumped BundledDeckLoader.bundleVersion 1->2 and now imports the first-party "Kakitori Foundations" source split into three decks (Hiragana/Katakana/Kanji, sourceDeckName "Kakitori Foundations::<section>"). But BundledDeckLoader.load() only ADDS/updates the currently-bundled decks; it never removes decks no longer produced by the bundle. So any install that was on version 1 keeps its two retired Tofugu decks (sourceDeckName "Tofugu Hiragana Anki Deck"/"Tofugu Katakana Anki Deck") on-disk in SwiftData alongside the three Foundations decks = 5 cards. A fresh install shows 3 correctly (nothing to prune), which is why earlier post-uninstall screenshots missed it. The version gate only decides whether import RUNS, not whether stale decks get pruned. This is a destructive data migration and must be robust: delete ONLY decks whose sourceDeckName is not in the expected set the current bundle produces.
