# T004 — SwiftData models: Deck + Section

## Overview

Create the first two SwiftData `@Model` types — `Deck` and its child `Section` — matching the data
model in docs/05-architecture.md §3 exactly, so importer and UI tasks share one schema.

## Do

Create `Sources/Models/Deck.swift` containing BOTH models (`import SwiftData`, `import Foundation`):

- `@Model final class Deck` with exactly these stored fields (docs/05-architecture.md §3):
  - `var id: UUID`
  - `var name: String` — top-level Anki deck name
  - `var jpTitle: String?` — display override (e.g. ひらがな)
  - `var sourceDeckName: String` — re-import matching key
  - `var importedAt: Date`
  - `@Relationship(deleteRule: .cascade, inverse: \Section.deck) var sections: [Section]`
  - A memberwise `init` with defaults `id: UUID = UUID()`, `sections: [Section] = []`.
- `@Model final class Section`. docs/05 references Section (Deck 1—* Section 1—* Note) but gives no
  field list, so define this minimal shape:
  - `var id: UUID`
  - `var name: String` — Anki subdeck / section name
  - `var orderIndex: Int` — stable display order within the deck
  - `var deck: Deck?` — the inverse of `Deck.sections` (plain `var`, no @Relationship attribute here;
    the inverse is declared on the Deck side above)
  - `init(id: UUID = UUID(), name: String, orderIndex: Int)`.

Do NOT reference a `Note` type — it does not exist yet (a later task adds `Note` and gives Section
its `notes` array). No force-unwraps or `try!`. The `init` must not read `Date()` — `importedAt` is
always passed in by the caller.

## Done when

- `Tests/UnitTests/DeckModelTests.swift` (XCTest) exists, builds an in-memory container —
  `ModelContainer(for: Deck.self, Section.self, configurations: ModelConfiguration(isStoredInMemoryOnly: true))`
  — never the app's real store, and asserts:
  - Insert a `Deck(name: "Hiragana", jpTitle: "ひらがな", sourceDeckName: "Kaki::Hiragana", importedAt: fixedDate)`
    then fetch it back: all five scalar fields round-trip unchanged.
  - Append two `Section`s to `deck.sections`, save, refetch: `sections.count == 2` and each
    section's `deck` back-reference points at the deck (relationship integrity both directions).
- `Sources/Models/Deck.swift` is the only new source file; field names match docs/05 §3 exactly.
