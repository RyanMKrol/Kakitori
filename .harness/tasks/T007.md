# T007 — SwiftData model: DailyStats + streak computation

## Overview

Create the per-day counters model `DailyStats` (docs/05-architecture.md §3) and a pure streak
function implementing the display rule from docs/02-product-spec.md §4.3 / docs/03-srs-algorithm.md §7:
consecutive adjusted-days with ≥1 card graded, not shown as broken until a full day passes.

## Do

Create `Sources/Models/DailyStats.swift` (`import SwiftData`, `import Foundation`):

1. `@Model final class DailyStats` with exactly these stored fields (docs/05-architecture.md §3):
   - `var day: String` — "YYYY-MM-DD" of the 4am-adjusted day
   - `var cardsWritten: Int` — count of grades given (docs/02 §4.2)
   - `var newIntroduced: Int`
   - `var reviewsDone: Int`
   - `var secondsStudied: Int`
   - `init(day: String, cardsWritten: Int = 0, newIntroduced: Int = 0, reviewsDone: Int = 0, secondsStudied: Int = 0)`.
2. A pure static function on `DailyStats` (no SwiftData access inside it):
   `static func currentStreak(activeDays: Set<String>, now: Date, clock: AppClock) -> Int`
   - `activeDays` = the set of `day` strings whose `cardsWritten >= 1` ("a day counts if ≥1 card was
     graded that day", docs/02-product-spec.md §4.3). Callers build this set from fetched DailyStats.
   - `AppClock` is the existing injected time source in `Sources/Support/AppClock.swift`; use
     `clock.adjustedDay(for: now)` for "today" and step backwards one calendar day at a time using
     `clock.calendar` (never `Calendar.current`, never `Date()` — time is always the injected `now`).
   - Rule (docs/02 §4.3): the streak is the run of consecutive active days ending at today, **or
     ending at yesterday when today is not active yet** ("the streak isn't shown as broken until the
     day actually passes"). If neither today nor yesterday is active, the streak is 0.

Read-only dependency: `AppClock` from `Sources/Support/AppClock.swift` — use it, do not modify it.
No force-unwraps or `try!`.

    and fetch all five fields back unchanged.
  - **Streak cases**, using a fixed clock (`AppClock.fixed`, `TimeZone(identifier: "Asia/Tokyo")`)
    where `now` maps to adjusted day "2026-07-17":
    - `{"2026-07-17","2026-07-16","2026-07-15"}` → 3 (today active)
    - `{"2026-07-16","2026-07-15"}` → 2 (today inactive, yesterday active — streak still shown)
    - `{"2026-07-17","2026-07-15"}` → 1 (gap breaks the run)
    - `{"2026-07-15"}` → 0 (neither today nor yesterday active)
    - `{}` → 0
  - A case proving the 4 AM rollover flows through: with `now` = 2026-07-18 03:00 local (adjusted
    day is still "2026-07-17"), `{"2026-07-17"}` → 1.
