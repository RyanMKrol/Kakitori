# T051 — Dynamic Type + VoiceOver pass

## Overview

Cross-cutting accessibility pass per docs/06-ui-design.md §4: chrome text scales with Dynamic Type
while answer glyphs and guide boxes stay fixed, all targets ≥44pt, and VoiceOver gets a canvas
custom action, spoken grade intervals, and hidden decorative glyphs.

## Do

1. Dynamic Type (docs/06 §4: "everything scales except the answer glyphs and guide boxes (they're
   the content, not chrome)"): all chrome text across `Sources/Features/` and
   `Sources/Support/Theme.swift` uses Dynamic-Type-scaling fonts (text styles / relative variants).
   The big answer glyphs and trace guide glyphs use FIXED point sizes (`Font.custom(_:fixedSize:)`
   or fixed `.system(size:)`), and guide-box dimensions must NOT derive from `@ScaledMetric`.
   Prompt/answer metadata text (reading, English, hint) DOES scale.
2. Hit targets: every interactive element ≥44pt; "grade buttons full-height rows on small screens"
   (docs/06 §4).
3. VoiceOver:
   - The canvas exposes a custom action: `.accessibilityAction(named: "Show answer") { ... }`
     invoking the same reveal as the show-answer button (docs/06 §4).
   - Grade buttons get an `accessibilityLabel` in the docs' pattern "Again — due in under
     1 minute" (docs/06 §4): grade name + "due in" + the button's existing interval-preview label,
     for all four of Again/Hard/Good/Easy.
   - Decorative glyphs are `.accessibilityHidden(true)`: the 書 header mark, deck-row
     representative glyphs (あ/ア/語), mode-picker glyphs (書/聞/訳/思, already labelled by their
     row text), and the summary 済 disc.
   - Stats and count chips get meaningful labels (e.g. a bare "3" due chip reads "3 due").
   - Do not change any accessibility IDENTIFIERS — labels and identifiers are separate properties.

## Done when

- Baseline: `./build_run.sh Kakitori-Sim -seedDemoData YES` renders unchanged at default size.
- Accessibility-XL screenshot — build_run.sh RESETS content size to `large` at its start, so set
  the size AFTER it and relaunch manually:
  ```
  UDID=$(./tools/loop_sim.sh)
  xcrun simctl ui "$UDID" content_size accessibility-extra-large
  xcrun simctl terminate "$UDID" com.ryankrol.kakitori 2>/dev/null || true
  xcrun simctl launch "$UDID" com.ryankrol.kakitori -seedDemoData YES && sleep 3
  xcrun simctl io "$UDID" screenshot screenshots/latest.png
  ```
  The shot must show chrome text (stats captions, deck titles, buttons) clearly enlarged, nothing
  clipped; if xcui can drive into a session (`.harness/custom/visual-verify-build.md`), also
  confirm guide boxes and the answer glyph are NOT enlarged.
- Then RESET in this same task: `xcrun simctl ui "$UDID" content_size large` — the setting
  persists and corrupts later tasks' screenshots.
