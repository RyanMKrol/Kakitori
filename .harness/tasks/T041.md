# T041 — AudioService: deck MP3 playback + ja-JP speech fallback

## Overview

Create the protocol-backed audio service: plays a deck's imported MP3s from the media directory, and
falls back to Japanese text-to-speech when a note has no audio file (docs/04-content-and-data.md §4).
No UI in this task; tests never do real playback.

## Do

1. Create `Sources/Audio/AudioService.swift` (`import Foundation`, `import AVFoundation`):
   - `@MainActor protocol AudioPlaying: AnyObject` with exactly:
     - `var isAvailable: Bool { get }` — whether the ja-JP TTS fallback exists.
     - `func playDeckAudio(filename: String, deckID: UUID)`
     - `func speakTarget(_ target: String)`
   - Extension on `AudioPlaying` holding the fallback-selection logic (pure, shared by live + fake):
     `func play(target: String, audioFilename: String?, deckID: UUID)` — if `audioFilename` is
     non-nil and non-empty call `playDeckAudio(filename:deckID:)`, otherwise `speakTarget(target)`
     (docs/04 §4: "if a note lacks audio, AVSpeechSynthesizer (ja-JP) reads Target").
   - `static func mediaFileURL(filename: String, deckID: UUID, mediaBaseURL: URL) -> URL` on the
     class (pure): `mediaBaseURL/deckID.uuidString/filename` — matches the import layout
     `Application Support/Kakitori/Media/<deckUUID>/<filename>` (docs/04 §2 step 6, §5).
   - `@MainActor final class AudioService: AudioPlaying`: `init(mediaBaseURL: URL? = nil)`; when nil,
     resolve `FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first`
     + `Kakitori/Media` via `guard let` (no force-unwrap; fall back to `FileManager.default.temporaryDirectory`).
     `playDeckAudio` resolves the URL, `try? AVAudioPlayer(contentsOf:)` kept in a stored property so
     playback is not deallocated, then `play()`; a missing file is a silent no-op, never a crash.
     `speakTarget` speaks an `AVSpeechUtterance` with `AVSpeechSynthesisVoice(language: "ja-JP")`
     through one stored `AVSpeechSynthesizer`. `isAvailable` = `AVSpeechSynthesisVoice(language: "ja-JP") != nil`.
2. Create `Tests/UnitTests/FakeAudioPlayer.swift` (test target — later tasks reuse it):
   `@MainActor final class FakeAudioPlayer: AudioPlaying` with
   `enum Call: Equatable { case deck(filename: String, deckID: UUID); case speak(String) }`,
   `private(set) var calls: [Call] = []`, and `var isAvailable = true` (settable for tests). The two
   protocol methods only append to `calls` — no AVFoundation types touched.

## Done when

`Tests/UnitTests/AudioServiceTests.swift` (XCTest, `@MainActor` test methods) passes, asserting:

- URL resolution: for a fixed `UUID(uuidString: "11111111-2222-3333-4444-555555555555")` and
  filename `"bc4e77e6407e004b.mp3"`, `AudioService.mediaFileURL` returns exactly
  `<base>/11111111-2222-3333-4444-555555555555/bc4e77e6407e004b.mp3`.
- Fallback selection: on a `FakeAudioPlayer`, `play(target: "おはよう", audioFilename: nil, deckID:)`
  records `[.speak("おはよう")]`; the same with `audioFilename: ""` also records `.speak`; with
  `audioFilename: "a.mp3"` records `[.deck(filename: "a.mp3", deckID:)]` and nothing else.
- Call order: two `play` calls record two entries in call order.
- No test constructs `AVAudioPlayer` or a real `AudioService` playback path (fake + pure helpers only).
