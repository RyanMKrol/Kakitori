# T012 — WritingCanvas: PKCanvasView wrapper with undo/clear

## Overview

The PencilKit writing surface: a `UIViewRepresentable` over `PKCanvasView` accepting finger and
Pencil, one fixed black pen, undo via the view's undoManager, clear by assigning an empty
`PKDrawing` (docs/05-architecture.md §1 + §5, docs/02-product-spec.md §3).

## Do

Create `Sources/Features/Session/Canvas/WritingCanvas.swift` containing:

- `@MainActor @Observable final class WritingCanvasController`:
  - `private(set) var isDrawingEmpty = true` — exposed so callers can tell whether anything
    has been written.
  - `func undo()` — calls the canvas view's `undoManager?.undo()` (docs/05 §5: "Undo uses
    `PKCanvasView.undoManager`").
  - `func clear()` — sets the canvas view's `drawing = PKDrawing()` (docs/05 §5: "Clear sets an
    empty `PKDrawing`"). No confirmation logic here.
  - a `weak` reference to the `PKCanvasView`, set by the representable.
- `struct WritingCanvas: UIViewRepresentable` taking a `WritingCanvasController`. In
  `makeUIView`: `drawingPolicy = .anyInput` (finger + Pencil, docs/05 §1); `backgroundColor =
  .clear` and `isOpaque = false` (the trace-guide layer sits BEHIND this view and must show
  through); `tool = PKInkingTool(.pen, color: .black, width: Self.penWidth)`. Never attach a
  `PKToolPicker` (docs/02 §3: "no tool picker").
- `static let penWidth: CGFloat = 6` — the ONE named tuning constant. The hardware feel-review
  task adjusts only this value, so no other width literal may appear.
- Coordinator conforms to `PKCanvasViewDelegate`; `canvasViewDrawingDidChange` sets
  `controller.isDrawingEmpty = canvasView.drawing.strokes.isEmpty`.
- Add a `UIPencilInteraction` to the canvas view whose delegate maps Pencil double-tap to
  `controller.undo()` (docs/02 §2.3 interaction details: "Pencil double-tap → Undo").
- The canvas view gets `accessibilityIdentifier = "writing-canvas"`.
- Add a `#Preview`: `WritingCanvas` in a ~500×220 frame on paper `#FAF8F4` with a thin outline
  so the drawable region is visible, plus two `Button`s "Undo" and "Clear" (accessibility
  identifiers `"canvas-undo"` and `"canvas-clear"`) wired to the controller — this proves the
  controller API shape the session screen will consume.
- Swift 6 strict concurrency; no force-unwraps or `try!` in Sources.

## Done when

- The loop fires `./build_run.sh`; this component has no app screen yet, so if the launched app
  cannot show it, render the `#Preview` and judge that image; record in the worklog which
  capture you judged. The image must show the outlined canvas region on paper background with
  the Undo and Clear buttons visible.
- Code inspection holds: `.anyInput` policy; single `PKInkingTool(.pen, color: .black)` whose
  width is the sole `penWidth` constant; no `PKToolPicker`; `undo()` routes through
  `undoManager`; `clear()` assigns `PKDrawing()` and touches nothing else (guides live in a
  separate layer the canvas never owns); `isDrawingEmpty` flips via the delegate callback.
