## Overview
The handwriting area on the session card (Listen & Write, plus Trace/Translate) is too small: the guide-box grid is hard-capped at a fixed ~160pt per box, so on the large iPad pane (and on iPhone) it floats in a big empty region with excessive margin. This one change grows the grid + canvas to fill much more of the available space while staying square and leaving breathing room, and confirms the multi-character case as ONE continuous drawing surface with N side-by-side guide cells and no obvious dead-ink gap between cells. The app already uses a SINGLE PKCanvasView (CanvasPaneView layers one WritingCanvas over the guide grid) so this is NOT a rewrite from N canvases to one; it is a sizing + gap change to the existing single-canvas layout. Purely layout/sizing: no change to ink, grading, audio, or segmentation.

## Do
In Sources/Features/Session/Canvas/GuideBoxGridGeometry.swift, relax the sizing math so the grid fills the available width instead of stopping at the current fixed cap: raise maxBoxSize substantially (target a per-box cap in the ~240-260pt ballpark, or make the box fill available width with that as the ceiling) and shrink horizontalInset to a small breathing-room margin (~16-24pt) so the grid grows close to but not overlapping the surrounding UI. Tighten interItemSpacing so the gap between adjacent boxes is minimal (the single continuous canvas already spans the gap; a smaller gap removes the obvious dead-ink gutter a cross-boundary stroke otherwise leaves). Keep every box square and keep gridSize/boxSize/rows returning consistent values so the WritingCanvas frame in CanvasPaneView.swift (which is sized from GuideBoxGridGeometry.gridSize) still coincides exactly with the visible boxes. Confirm CanvasPaneView.swift still renders exactly ONE WritingCanvas over the guide grid in a ZStack (do NOT reintroduce per-character canvases). Adjust GuideBoxRow.swift / GuideBox.swift only as needed so the rendered boxes match the new sizing (e.g. spacing, box frame). Update Tests/UnitTests/GuideBoxGridGeometryTests.swift and Tests/UnitTests/CanvasPaneRenderTests.swift to reflect the new max box size / inset / spacing constants so they assert the grown sizing rather than the old 160pt cap. Do not touch ink colour, grading, audio, or TargetSegmenter.
