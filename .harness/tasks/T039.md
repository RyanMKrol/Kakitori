# T039 — Import UI: Files picker, progress, error alerts

## Overview

Put deck import behind the UI: a toolbar import button on Home opening the Files picker for .apkg,
the existing ApkgImporter actor running off-main with determinate progress, and failure alerts that
state the specific reason.

## Do

- `Sources/Features/Home/HomeView.swift`: add a toolbar `+` import button, identifier
  `"import-button"` (docs/06-ui-design.md §2.1: "Import also lives behind a toolbar `+`"). The empty
  state's "Import deck" button triggers the same action.
- Create `Sources/Features/Home/ImportCoordinator.swift`: `@MainActor @Observable final class
  ImportCoordinator` with `static let shared = ImportCoordinator()`, a state enum
  (idle / running(progress: Double) / failed(message: String)), and `func begin(url: URL)`.
  `begin` calls `url.startAccessingSecurityScopedResource()` (deferred stop), then awaits the
  existing `ApkgImporter` actor (`Sources/Importer/ApkgImporter.swift` — an actor, so the work runs
  off-main; read it for its progress-reporting API and typed `ImporterError` cases), forwarding
  progress into the state. The `static shared` is the seam the app entry will use to route
  "Open in Kakitori" URLs into this same path in the follow-on document-type task.
- HomeView, observing the coordinator:
  - `.fileImporter(isPresented:allowedContentTypes:)` from the button. Content types:
    `UTType(filenameExtension: "apkg")` plus `.zip` and `.data` fallbacks — build the array by
    conditionally appending the optional, no force-unwraps. Picked URL → `ImportCoordinator.shared.begin(url:)`.
  - While running: a determinate `ProgressView(value:)` (percent from the coordinator) in a small
    overlay/sheet.
  - On failure: `.alert("Import failed", ...)` whose message is the SPECIFIC reason, mapped from the
    `ImporterError` case. docs/04-content-and-data.md §2 step 4 and docs/05-architecture.md §6 fix
    the three cases (bad zip / no AnkiBuilder model / zero notes); the exact user copy is fixed here:
    - bad zip → `This file is not a valid .apkg archive.`
    - no AnkiBuilder model (no note type with a Target field) → `No AnkiBuilder note type with a Target field was found in this deck.`
    - zero notes → `The deck contains no notes.`
    - any other error → its `localizedDescription`.
- Success needs no manual refresh: the deck list is SwiftData-query-driven and updates when the
  importer saves (docs/05 §6).

## Done when

- `./build_run.sh` (unseeded) screenshot shows the toolbar import button alongside the empty state;
  tapping `import-button` (via xcui if available) presents the Files picker sheet.
- The three alert strings above appear verbatim in source, each mapped to its `ImporterError` case,
  and the importer is invoked with `await` on the actor (no main-thread file IO).
