# T015 — Deck fixtures: kana deck into Tests/Fixtures (🔒 needs-human)

## Overview

Copy the real JBP kana deck out of the local anki-builder checkout into the repo so the importer
tasks (T016–T021) have a genuine `.apkg` fixture to test against (docs/04-content-and-data.md:
`anki-builder/output/epubs/japanese-for-busy-people-book-1-kana/deck-v5.apkg`, docs/05 §7 fixture
tests). Human-gated because the source file lives outside the repo on your machine.

## Do

1. From the repo root:

   ```sh
   mkdir -p Tests/Fixtures
   cp "$HOME/Development/anki-builder/output/epubs/japanese-for-busy-people-book-1-kana/deck-v5.apkg" \
      Tests/Fixtures/kana-deck-v5.apkg
   ```

   The destination path is load-bearing: importer unit tests bind to exactly
   `Tests/Fixtures/kana-deck-v5.apkg`.
2. Sanity-check the size is reasonable to commit — it contains recorded MP3s, so it is not tiny,
   but it must be roughly ≤ 20 MB (`du -h Tests/Fixtures/kana-deck-v5.apkg`; expected ~2 MB).
   If it is wildly larger, stop and investigate rather than committing.
3. Sanity-check it is a real `.apkg` (a zip): `unzip -l Tests/Fixtures/kana-deck-v5.apkg | head`
   should list `collection.anki2` and a `media` manifest (docs/04 §1).
4. Confirm git will take it: `git check-ignore -v Tests/Fixtures/kana-deck-v5.apkg` should print
   nothing; if an ignore rule matches, add a negation for this path rather than removing the rule.
5. Commit the fixture and push to `main`. (No `project.yml` change needed — importer tests will
   resolve the fixture via a `#filePath`-relative path, not a bundle resource.)
6. Mark the gate done: `.harness/scripts/mark-done.sh T015`.

## Done when

- `Tests/Fixtures/kana-deck-v5.apkg` exists at exactly that path, is a valid zip containing
  `collection.anki2`, and is roughly ≤ 20 MB.
- The fixture is committed and pushed on `main` (not ignored, not LFS-mangled), so the importer
  tasks can build cold against it.
- `.harness/scripts/mark-done.sh T015` has been run so T016+ unblock.
