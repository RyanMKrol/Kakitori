## Overview
Re-attempt of T021, which was blocked because Note's soft-delete flag was named `isDeleted` — a read-only built-in on NSManagedObject (SwiftData is Core Data-backed), so writes to it never persisted and the soft-delete/revival assertions could never pass. The original spec pinned scope to Sources/Importer/ only and forbade touching Sources/Models/, so the builder could not fix the collision and correctly reported failed:blocked. This follow-up adds Sources/Models/Note.swift to scope so the property can be renamed, then implements the re-import logic against the renamed field.

## Do
1. In Sources/Models/Note.swift, RENAME the soft-delete stored property `isDeleted` (and its `init` parameter + assignment) to `isSoftDeleted` (Bool, default false). This avoids the NSManagedObject.isDeleted collision that blocked T021 — a plain rename, no migration plan needed (the app has no VersionedSchema and no shipped store yet). Update the one existing reference in Sources/Importer/ApkgImporter.swift (currently `isDeleted: false` at the first-import Note construction) to `isSoftDeleted: false`.
2. Then implement the re-import behaviour from the original T021 in Sources/Importer/ApkgImporter.swift, using `isSoftDeleted` throughout:
   - Rework `apply(_ parsed: ParsedDeck) throws -> UUID`: fetch an existing `Deck` with `sourceDeckName == parsed.deckName`. None found -> the T020 first-import path, unchanged. Found -> re-import: update `importedAt`; build a dictionary of the deck's existing `Note`s keyed by `id`; compute each incoming note's id with `NoteIdentity.uuid(forAnkiGUID:)`; then:
     - Matched id -> update content in place (`target`, `pronunciation`, `english`, `category`, `hint`, `audioFilename`, `units`, `script`, section assignment) and set `isSoftDeleted = false` (a returning note is revived). Do NOT touch `note.schedule` in any way — `state`, `stepIndex`, `easeFactor`, `intervalDays`, `dueAt`, `lapses` all survive verbatim.
     - New id (absent from the dictionary) -> insert a new `Note` + a new `CardSchedule` in state `.new`, exactly like first import.
     - Disappeared (existing id absent from the incoming set) -> set `isSoftDeleted = true`. NEVER hard-delete the `Note`, never remove or reset its `CardSchedule` (soft delete only, per docs/04-content-and-data.md §2 step 8).
     - Sections: create any `Section` whose `sectionName` doesn't exist yet; reuse existing ones by name.
   - Save once at the end; still return the deck's `UUID` (existing deck keeps its original id, so its media directory is stable).
