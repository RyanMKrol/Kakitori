## Overview
Autoplay currently fires too early because SessionViewModel triggers it synchronously: init() calls autoplayOnCardEntry() during view-model construction (before SessionView's body has rendered), and grade() calls it while SwiftUI is still animating the card change. Symptoms: in Listen/Write mode the first card's audio plays the instant Start Writing is tapped, before the card screen is up; and on advancing, the next card's audio starts during the fade-out, before the next card is drawn. Fix: move the autoplay trigger so it fires when the card view actually appears — for BOTH the first card and each subsequent card — driven from the SwiftUI view, not from the view-model's init()/grade(). Listen still auto-plays; it just happens on card appearance. This builds on unit audio-autoplay-a, which already restricts autoplay to Listen only (do not re-broaden it to Trace).
