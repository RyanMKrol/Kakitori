## Overview
The deck 'Mastery %' (mature-card ratio, interval>=21 — DeckCardView.proficiencyPercentage ~213-223) legitimately stays 0% for a long time, so completing a session shows no Home movement. Per the owner's decision (2026-07-18), add a 'completed today' progress indicator (X of Y) to the deck card so a session visibly advances progress: Y = the deck's allotment for today (from the DailyAllowance helper introduced in conv-daily-cap-b), X = cards completed today for that deck. Depends on conv-daily-cap-b for the allowance helper.

## Do
Using the DailyAllowance helper from conv-daily-cap-b for the deck's today allotment (Y) plus today's completed count for the deck (X — e.g. cards graded/introduced today for the deck, or Y minus the deck's remaining allotment; clamp 0 <= X <= Y), compute a pure, unit-testable 'completed today' value (inject AppClock/now for the day boundary — never a bare Date()). Render it on BOTH the DeckCardView compact row (~L33-45) and the regular card (~L115-135) as an X/Y indicator (e.g. 'X/Y today' text and/or a small progress bar), consistent between the two layouts. KEEP the existing 'Mastery %' badge as the long-term mastery stat (it is a distinct concept, already labelled 'Mastery') — the new indicator is the session-progress surface that moves. Do NOT change the daily-allowance chips from conv-daily-cap-b, the SM-2 scheduler, or DailyStats recording.

## Done when
The deck card shows a 'completed today' X/Y indicator on both compact and regular layouts, where Y is the deck's today allotment (from the DailyAllowance helper) and X is today's completed count (clamped 0..Y). A unit test asserts the pure progress function: after completing N of today's allotment it reads N/Y; a fresh day reads 0/Y; it never exceeds Y. The existing 'Mastery %' badge remains present. Manually verified on BOTH iPad (Kakitori-Sim) and iPhone (Kakitori-Sim-Phone): completing cards visibly advances the deck's 'today' progress on Home, with no layout regression and the two form factors visually consistent. DoD green (xcodegen generate; swiftformat --lint; swiftlint --strict; xcodebuild test on Kakitori-Sim).
