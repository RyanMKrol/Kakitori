# T031 — Prompt pane: mode prompt + answer reveal block

## Overview

Build the session's left pane: the Trace-mode prompt that crossfades (<250ms) into the ANSWER
block when the user reveals — large Mincho target, reading, English, optional hint, and a disabled
play-audio placeholder.

## Do

- Create `Sources/Features/Session/PromptPaneView.swift` taking the `SessionViewModel`
  (`Sources/Features/Session/SessionViewModel.swift` — current card fields, `revealed` flag). Replace
  the `promptPane` placeholder inside `Sources/Features/Session/SessionView.swift` with it.
- **Before reveal** (Trace mode) — vertically centred, text centred (docs/02-product-spec.md §2.1):
  - Caption `TRACE MODE` — `KakitoriTheme.accent`, small-caps style (~12pt semibold, generous
    tracking; docs/06-ui-design.md §1 names the "TRACE MODE" label).
  - Headline `Write over the faded guides` (docs/02 §2.1) — bold ~22pt ink.
  - Instruction line `Follow the light strokes in each box. Aim for balance and correct stroke order.`
    (the reference prototype's Trace panel — this is the docs' "instruction text").
  - The card's reading (pronunciation) in `KakitoriTheme.japaneseDisplayFont`, faint ink
    ("The reading is shown" — docs/02 §2.1).
- **After reveal** — the ANSWER block, container accessibility identifier `"answer-block"`
  (docs/06 §2.3):
  - Caption `ANSWER` in accent small caps.
  - Target glyphs in `KakitoriTheme.japaneseDisplayFont`: 96pt when the card has ≤2 units, 64pt
    otherwise (docs/06 §1: "~64–96pt depending on unit count").
  - Reading (Mincho ~22pt, accent), English (~17pt ink), and — only when non-nil/non-empty — the
    hint in a rounded paper-tinted box.
  - A `Play audio` pill (accent-tint capsule) set `.disabled(true)` as a placeholder; leave a
    `// TODO` naming `Sources/Audio/AudioService.swift` (the audio task wires it later).
- Transition prompt ⇄ answer with a crossfade: `.transition(.opacity)` under an animation of
  duration ≤ 0.25s (docs/06 §3: "Keep under 250ms").
- Render check: `Tests/UnitTests/PromptPaneRenderTests.swift`, same recipe as
  `Tests/UnitTests/SessionScaffoldRenderTests.swift` (in-memory container, fixed clock,
  `ImageRenderer` at 1194×834 via `#filePath`-derived path): write `screenshots/T031-prompt.png`
  pre-reveal and `screenshots/T031-answer.png` after calling `showAnswer()`.
