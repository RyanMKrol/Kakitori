# T026 — SessionQueue: queue building with daily caps + ordering

## Overview

Create the pure session-queue builder per docs/03-srs-algorithm.md §5: due learning first, due
reviews capped by the daily review allowance, new cards capped by the daily new allowance, reviews
interleaved with new (reviews weighted first).

## Do

Create `Sources/Scheduler/SessionQueue.swift` (Scheduler module rules: `import Foundation` only —
no SwiftData/SwiftUI; no `Date()`/`.now`; this task needs no RNG). T022 provides
`ScheduleSnapshot`/`SRSConstants`; do NOT use `SM2Scheduler` here (it may not exist yet). Define:

```swift
struct QueueEntry: Equatable, Sendable { let id: UUID; var snapshot: ScheduleSnapshot }

struct SessionQueue: Sendable {
    private(set) var entries: [QueueEntry]
    var newCount: Int    // entries with state .new
    var learnCount: Int  // entries with state .learning or .relearning
    var dueCount: Int    // entries with state .review
    static func build(cards: [QueueEntry], now: Date, endOfToday: Date,
                      newPerDay: Int = SRSConstants.defaultNewPerDay,            // 10
                      maxReviewsPerDay: Int = SRSConstants.defaultMaxReviewsPerDay, // 100
                      newIntroducedToday: Int, reviewsDoneToday: Int) -> SessionQueue
}
```

`endOfToday` is the end of the 4 AM-adjusted day — the next 4:00 AM local time after `now`
(docs/03 §7); the **caller** computes it (this module just compares dates). Build steps, quoting
docs/03 §5: "(1) due learning/relearning (dueAt ≤ now); (2) due review (dueAt ≤ end of today),
capped by max reviews/day minus reviews already done today; (3) new cards, capped by new/day minus
new cards already introduced today; (4) queue order: learning (by dueAt) → interleave review and
new (reviews weighted first)". Concretely:

1. **Learning block**: state `.learning`/`.relearning` with `dueAt != nil && dueAt <= now`, sorted
   by `dueAt` ascending.
2. **Reviews**: state `.review` with `dueAt != nil && dueAt <= endOfToday`, sorted by `dueAt`
   ascending, then `prefix(max(0, maxReviewsPerDay - reviewsDoneToday))`.
3. **New**: state `.new`, kept in the given `cards` order, then
   `prefix(max(0, newPerDay - newIntroducedToday))`.
4. **Interleave** ("reviews weighted first"), deterministic fractional-position merge: review at
   index i (0-based, R total) gets key `(i+1)/(R+1)`; new at index j (N total) gets key
   `(j+1)/(N+1)`; sort all by key ascending, **review wins ties**. Examples: R1..R4 + N1,N2 →
   `R1 N1 R2 R3 N2 R4`; R1,R2 + N1,N2 → `R1 N1 R2 N2`. Final order = learning block + merged list.
