# T049 — iPhone compact Home, deck setup, and summary layouts

## Overview

Add the compact (iPhone) variants of Home, the deck setup sheet, and the session summary per
docs/06-ui-design.md §2.1/§2.2/§2.4: single-column deck list with a leading representative glyph,
compressed stat captions, medium-detent bottom sheet, and the iPhone summary copy variants.

## Do

All switching is via `@Environment(\.horizontalSizeClass) == .compact` — never device checks.

1. Home deck list (`Sources/Features/Home/`), compact: "a single-column list where each row leads
   with a representative glyph (あ / ア / 語)" and "the whole row is tappable" (docs/06 §2.1).
   Pick the glyph by the deck's script classification: hiragana → "あ", katakana → "ア",
   kanji/mixed → "語". Keep the `deck-row-<name>` identifiers on the rows. Regular (iPad) keeps
   the existing 2-up grid unchanged.
2. Home stats (`Sources/Features/Home/`), compact: stat captions compress to exactly "Streak" /
   "Written" / "Studied" (docs/06 §2.1), stat cards stacked to fit the compact width per the
   iPhone prototype (`reference/Kaki - iPhone.dc.html`).
3. Deck setup sheet (`Sources/Features/DeckSetup/`): "medium detent on iPhone, formSheet on iPad"
   (docs/06 §2.2) — apply `.presentationDetents([.medium])` only in the compact branch; regular
   presentation unchanged. Keep `mode-trace`, `start-writing` identifiers.
4. Session summary (`Sources/Features/Summary/`), compact copy variants (docs/06 §2.4):
   - Streak footer is just "🔥 5 day streak" — WITHOUT " · keep it going tomorrow" (regular keeps
     the full line).
   - Buttons: "Study another script" (primary) and "Back home" (secondary/quiet). Regular keeps
     "Back to decks" (primary) / "Study another deck" (quiet). Copy character-for-character.
   Both buttons keep their existing actions and identifiers; only labels/emphasis change.

## Done when

- The DEFAULT dedicated simulator is `Kakitori-Sim` (an iPad); tests/DoD run there. For the visual
  check an iPhone-width device is REQUIRED: `./build_run.sh "iPhone 16" -seedDemoData YES` (any
  iPhone 16-class simulator; deliberate exception to the Kakitori-Sim substitution because this
  task IS the iPhone layout). `screenshots/latest.png` must show: single-column deck rows led by
  あ / ア / 語, stat captions "Streak" / "Written" / "Studied", whole rows tappable (no separate
  Study-only tap target required).
- If xcui is available (`.harness/custom/visual-verify-build.md`), tap a deck row and re-screenshot:
  the setup sheet appears as a half-height (medium detent) bottom sheet.
- `./build_run.sh Kakitori-Sim -seedDemoData YES` afterwards: iPad Home still shows the 2-up deck
  grid and full stat captions, unchanged.
